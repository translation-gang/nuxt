## `nuxt.config`

Точкой входа для приложения Nuxt по-прежнему остаётся файл `nuxt.config`.

::note
Конфигурация Nuxt загружается через [`unjs/jiti`](https://github.com/unjs/jiti) и [`unjs/c12`](https://github.com/unjs/c12).
::

### Миграция

1. Перейдите на функцию `defineNuxtConfig` с типизированной схемой конфигурации.

   ::code-group

   ```ts [Nuxt 2]
   export default {
     // ...
   }
   ```

   ```ts [Nuxt 3]
   export default defineNuxtConfig({
     // ...
   })
   ```

   ::

1. Если использовался `router.extendRoutes`, перейдите на хук `pages:extend`:

   ::code-group

   ```ts [Nuxt 2]
   export default {
     router: {
       extendRoutes (routes) {
         //
       },
     },
   }
   ```

   ```ts [Nuxt 3]
   export default defineNuxtConfig({
     hooks: {
       'pages:extend' (routes) {
         //
       },
     },
   })
   ```

   ::

1. Если использовался `router.routeNameSplitter`, тот же эффект можно получить, обновив логику генерации имён маршрутов в хуке `pages:extend`:

   ::code-group

   ```ts [Nuxt 2]
   export default {
     router: {
       routeNameSplitter: '/',
     },
   }
   ```

   ```ts [Nuxt 3]
   import { createResolver } from '@nuxt/kit'

   export default defineNuxtConfig({
     hooks: {
       'pages:extend' (routes) {
         const routeNameSplitter = '/'
         const root = createResolver(import.meta.url).resolve('./pages')

         function updateName (routes) {
           if (!routes) {
             return
           }

           for (const route of routes) {
             const relativePath = route.file.substring(root.length + 1)
             route.name = relativePath.slice(0, -4).replace(/\/index$/, '').replace(/\//g, routeNameSplitter)

             updateName(route.children)
           }
         }
         updateName(routes)
       },
     },
   })
   ```

   ::

#### Синтаксис ESM

Nuxt 3 — [фреймворк на нативном ESM](/docs/4.x/guide/concepts/esm). Хотя [`unjs/jiti`](https://github.com/unjs/jiti) частично обеспечивает совместимость при загрузке `nuxt.config`, избегайте `require` и `module.exports` в этом файле.

1. Замените `module.exports` на `export default`
1. Замените `const lib = require('lib')` на `import lib from 'lib'`

#### Асинхронная конфигурация

Чтобы поведение загрузки Nuxt было предсказуемее, асинхронный синтаксис конфигурации устарел. Для асинхронных операций используйте хуки Nuxt.

#### Dotenv

В Nuxt есть встроенная загрузка файлов `.env`. Не импортируйте dotenv напрямую из `nuxt.config`.

## Модули

Nuxt и модули Nuxt теперь используются только на этапе сборки.

### Миграция

1. Перенесите все `buildModules` в `modules`.
2. Проверьте совместимость модулей с Nuxt 3.
3. Если локальные модули указывали на каталог, укажите файл входа:

```diff
  export default defineNuxtConfig({
    modules: [
-     '~/modules/my-module'
+     '~/modules/my-module/index'
    ]
  })
```

::tip
Если вы автор модуля, см. [дополнительно о совместимости модулей](/docs/4.x/migration/module-authors) и [руководство для авторов модулей](/docs/4.x/guide/modules).
::

## Изменения в структуре каталогов

Каталог `static/` (статические файлы) переименован в `public/`. Можно переименовать `static` в `public` или оставить имя, задав `dir.public` в `nuxt.config`.

:read-more{to="/docs/4.x/directory-structure/public"}

## TypeScript

Миграция проще, если использовать интеграцию TypeScript в Nuxt. Это не значит, что всё приложение нужно писать на TypeScript — Nuxt даст подсказки типов в редакторе.

Подробнее о поддержке TypeScript — [в документации](/docs/4.x/guide/concepts/typescript).

::note
Nuxt может проверять типы через [`vue-tsc`](https://github.com/vuejs/language-tools/tree/master/packages/tsc) командой `nuxt typecheck`.
::

### Миграция

1. Создайте `tsconfig.json` с таким содержимым:

   ```json
   {
     "files": [],
     "references": [
       {
         "path": "./.nuxt/tsconfig.app.json"
       },
       {
         "path": "./.nuxt/tsconfig.server.json"
       },
       {
         "path": "./.nuxt/tsconfig.shared.json"
       },
       {
         "path": "./.nuxt/tsconfig.node.json"
       }
     ]
   }
   ```

1. Выполните `npx nuxt prepare`, чтобы сгенерировать файлы tsconfig.
1. Установите Volar по [инструкции в документации](/docs/4.x/getting-started/installation#prerequisites).

## Изменения во Vue

Изменились рекомендации по Vue и есть breaking changes между Vue 2 и 3.

Имеет смысл прочитать [руководство по миграции на Vue 3](https://v3-migration.vuejs.org) и в частности [список breaking changes](https://v3-migration.vuejs.org/breaking-changes/).

Сейчас [migration build Vue 3](https://v3-migration.vuejs.org/migration-build.html) с Nuxt 3 использовать нельзя.

## Vuex

В Nuxt больше нет встроенной интеграции с Vuex. Официальная рекомендация Vue — `pinia`, с поддержкой Nuxt через [модуль Nuxt](https://pinia.vuejs.org/ssr/nuxt.html). [Подробнее о pinia](https://pinia.vuejs.org).

Простой вариант глобального состояния с pinia:

Установите модуль [`@pinia/nuxt`](/modules/pinia):

```bash [Terminal]
yarn add pinia @pinia/nuxt
```

Подключите модуль в конфигурации nuxt:

```ts [nuxt.config.ts]
import { defineNuxtConfig } from 'nuxt/config'

export default defineNuxtConfig({
  modules: ['@pinia/nuxt'],
})
```

Создайте папку `store` в корне приложения:

```ts [store/index.ts]
import { defineStore } from 'pinia'

export const useMainStore = defineStore('main', {
  state: () => ({
    counter: 0,
  }),
  actions: {
    increment () {
      // `this` is the store instance
      this.counter++
    },
  },
})
```

Создайте [плагин](/docs/4.x/directory-structure/app/plugins) для глобального доступа к store:

```ts [app/plugins/pinia.ts]
import { useMainStore } from '~/store'

export default defineNuxtPlugin(({ $pinia }) => {
  return {
    provide: {
      store: useMainStore($pinia),
    },
  }
})
```

Чтобы остаться на Vuex, можно вручную перейти на Vuex 4 по [этим шагам](https://vuex.vuejs.org/guide/migrating-to-4-0-from-3-x.html).

После этого добавьте в приложение такой плагин:

```ts [app/plugins/vuex.ts]
import store from '~/store'

export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.vueApp.use(store)
})
```

Для крупных приложений миграция может быть трудоёмкой. Если обновление Vuex тормозит, можно использовать сообщество: [nuxt3-vuex-module](https://github.com/vedmant/nuxt3-vuex#nuxt3-vuex-module) — часто работает из коробки.
