### Изоморфный fetch

В Nuxt 2 часто использовали `@nuxtjs/axios`, `@nuxt/http` или полифилленный глобальный `fetch`.

В Nuxt 3 доступен глобальный `fetch` с API как у [Fetch API](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch), или метод [`$fetch`](/docs/4.x/api/utils/dollarfetch) на базе [unjs/ofetch](https://github.com/unjs/ofetch). Плюсы:

1. [Прямые вызовы API](/docs/4.x/guide/concepts/server-engine#direct-api-calls) на сервере и клиентские запросы к вашему API на клиенте (можно и к сторонним API).

2. Удобства: автоматический разбор ответов и сериализация данных.

Подробнее: [прямые вызовы API](/docs/4.x/guide/concepts/server-engine#direct-api-calls) и [загрузка данных](/docs/4.x/getting-started/data-fetching).

### Композаблы

В Nuxt 3 для данных: [`useAsyncData`](/docs/4.x/api/composables/use-async-data) и `useFetch`. Есть «ленивые» варианты (`useLazyAsyncData` и `useLazyFetch`), не блокирующие клиентскую навигацию.

В Nuxt 2 данные в компоненте могли выглядеть так:

```ts
export default {
  async asyncData ({ params, $http }) {
    const post = await $http.$get(`https://api.nuxtjs.dev/posts/${params.id}`)
    return { post }
  },
  // or alternatively
  fetch () {
    this.post = await $http.$get(`https://api.nuxtjs.dev/posts/${params.id}`)
  },
}
```

В методах и шаблоне `post` использовался как обычные данные компонента.

В Nuxt 3 то же делают композаблами в `setup()` или `<script setup>`:

```vue
<script setup lang="ts">
// Define params wherever, through `defineProps()`, `useRoute()`, etc.
const { data: post, refresh } = await useAsyncData('post', () => $fetch(`https://api.nuxtjs.dev/posts/${params.id}`))
// Or instead - useFetch is a convenience wrapper around useAsyncData when you're just performing a simple fetch
const { data: post, refresh } = await useFetch(`https://api.nuxtjs.dev/posts/${params.id}`)
</script>
```

`post` доступен в шаблоне Nuxt 3, `refresh` обновляет данные.

::note
Несмотря на название, [`useFetch`](/docs/4.x/api/composables/use-fetch) не прямая замена хука `fetch()`. [`useAsyncData`](/docs/4.x/api/composables/use-async-data) заменяет оба хука и гибче; [`useFetch`](/docs/4.x/api/composables/use-fetch) — обёртка над [`useAsyncData`](/docs/4.x/api/composables/use-async-data) для простых запросов к эндпоинту.
::

### Миграция

1. Замените хук `asyncData` на [`useAsyncData`](/docs/4.x/api/composables/use-async-data) или [`useFetch`](/docs/4.x/api/composables/use-fetch) в странице/компоненте.
2. Замените хук `fetch` на [`useAsyncData`](/docs/4.x/api/composables/use-async-data) или [`useFetch`](/docs/4.x/api/composables/use-fetch) в компоненте.

## `key`

Ключ страницы задаётся в макросе [`definePageMeta`](/docs/4.x/api/utils/define-page-meta).

```diff [app/pages/index.vue]
- <script>
- export default {
-   key: 'index'
-   // or a method
-   // key: route => route.fullPath
- }
+ <script setup>
+ definePageMeta({
+   key: 'index'
+   // or a method
+   // key: route => route.fullPath
+ })
</script>
```

## `scrollToTop`

В Nuxt 3 пока не поддерживается. Чтобы переопределить поведение прокрутки `vue-router`, используйте `~/router.options.ts` (см. [документацию](/docs/4.x/guide/recipes/custom-routing#router-options)).
Как и для `key`, укажите в макросе [`definePageMeta`](/docs/4.x/api/utils/define-page-meta):

```diff [app/pages/index.vue]
- <script>
- export default {
-   scrollToTop: false
- }
+ <script setup>
+ definePageMeta({
+   scrollToTop: false
+ })
</script>
```

## `validate`

Хук `validate` в Nuxt 3 принимает один аргумент — `route`. Как в Nuxt 2, можно вернуть boolean; при `false` и отсутствии другого совпадения — 404. Можно вернуть объект с `status`/`statusText` для немедленной ошибки (другие совпадения не проверяются).

```diff [app/pages/users/[id\\].vue]
- <script>
- export default {
-   async validate({ params }) {
-     return /^\d+$/.test(params.id)
-   }
- }
+ <script setup>
+ definePageMeta({
+   validate: async (route) => {
+     const nuxtApp = useNuxtApp()
+     return /^\d+$/.test(route.params.id)
+   }
+ })
  </script>
```

## `watchQuery`

В Nuxt 3 не поддерживается. Вместо этого используйте watcher и повторный запрос данных.

```vue [app/pages/users/[id\\].vue]
<script setup lang="ts">
const route = useRoute()
const { data, refresh } = await useFetch('/api/user')
watch(() => route.query, () => refresh())
</script>
```
