## `app.vue`

В Nuxt 3 точка входа приложения — файл `~/app.vue`.

::note
Если в исходниках нет `app.vue`, Nuxt использует встроенный вариант по умолчанию.
::

Сюда удобно помещать код, который выполняется один раз при старте, и компоненты, общие для всех страниц. Например, при одном layout его можно перенести в `app.vue`.

:read-more{to="/docs/4.x/directory-structure/app/app"}

:link-example{to="/docs/4.x/examples/hello-world"}

### Миграция

Создайте `app.vue` и перенесите в него логику верхнего уровня. Пример — [в документации](/docs/4.x/directory-structure/app/app).

## Layouts

Если для нескольких страниц используются layouts, изменения небольшие.

В Nuxt 2 в layout для текущей страницы служил компонент `<Nuxt>`. В Nuxt 3 layouts используют слоты: замените `<Nuxt>` на `<slot />`. Так можно использовать именованные и scoped-слоты. [Подробнее о layouts](/docs/4.x/directory-structure/app/layouts).

Layout для страницы задаётся макросом `definePageMeta`. Имена в kebab-case: `app/layouts/customLayout.vue` → `custom-layout` в странице.

### Миграция

1. Замените `<Nuxt />` на `<slot />`
    ```diff [app/layouts/custom.vue]
      <template>
        <div id="app-layout">
          <main>
    -       <Nuxt />
    +       <slot />
          </main>
        </div>
      </template>
    ```
2. Используйте [`definePageMeta`](/docs/4.x/api/utils/define-page-meta), чтобы указать layout страницы.
    ```diff [app/pages/index.vue]
    + <script setup>
    + definePageMeta({
    +   layout: 'custom'
    + })
    - <script>
    - export default {
    -   layout: 'custom'
    - }
      </script>
    ```
3. Перенесите `~/layouts/_error.vue` в `~/error.vue`. См. [документацию по обработке ошибок](/docs/4.x/getting-started/error-handling). Чтобы страница ошибки использовала layout, оберните содержимое в [`<NuxtLayout>`](/docs/4.x/directory-structure/app/layouts) внутри `error.vue`:
    ```vue [error.vue]
    <template>
      <div>
        <NuxtLayout name="default">
          <!-- -->
        </NuxtLayout>
      </div>
    </template>
    ```

## Страницы

В Nuxt 3 опционально подключается `vue-router`, если в проекте есть каталог [`app/pages/`](/docs/4.x/directory-structure/app/pages). Если страница одна, её можно перенести в `app.vue` для более лёгкой сборки.

### Динамические маршруты

В Nuxt 3 формат динамических сегментов отличается от Nuxt 2 — возможно, понадобится переименовать файлы в `app/pages/`.

1. Вместо `_id` для параметра используйте `[id]`.
2. Вместо `_.vue` для catch-all используйте `[...slug].vue`.

### Вложенные маршруты

В Nuxt 2 вложенные маршруты (родитель и дочерние компоненты) задавались через `<Nuxt>` и `<NuxtChild>`. В Nuxt 3 это заменено одним компонентом `<NuxtPage>`.

### Ключ страницы и keep-alive

Кастомный ключ страницы или пропсы keep-alive, раньше передававшиеся в `<Nuxt>`, задаются через `definePageMeta`.

:read-more{to="/docs/4.x/directory-structure/app/pages#special-metadata"}

### Переходы страницы и layout

Если переходы для страницы или layout задавались в опциях компонента, используйте `definePageMeta`. Во Vue 3 [-enter и -leave классы переименованы](https://v3-migration.vuejs.org/breaking-changes/transition.html). Проп `style` у `<Nuxt>` больше не влияет на переход при `<slot>` — перенесите стили в класс `-active`.

:read-more{to="/docs/4.x/getting-started/transitions"}

### Миграция

1. Переименуйте файлы страниц с динамическими параметрами под новый формат.
2. Замените `<Nuxt>` и `<NuxtChild>` на `<NuxtPage>`.
3. При Composition API замените `this.$route` и `this.$router` на композаблы [`useRoute`](/docs/4.x/api/composables/use-route) и [`useRouter`](/docs/4.x/api/composables/use-router).

#### Пример: динамические маршруты

::code-group

``` [Nuxt 2]
- URL: /users
- Page: /pages/users/index.vue

- URL: /users/some-user-name
- Page: /pages/users/_user.vue
- Usage: params.user

- URL: /users/some-user-name/edit
- Page: /pages/users/_user/edit.vue
- Usage: params.user

- URL: /users/anything-else
- Page: /pages/users/_.vue
- Usage: params.pathMatch
```

``` [Nuxt 3]
- URL: /users
- Page: /pages/users/index.vue

- URL: /users/some-user-name
- Page: /pages/users/[user].vue
- Usage: params.user

- URL: /users/some-user-name/edit
- Page: /pages/users/[user]/edit.vue
- Usage: params.user

- URL: /users/anything-else
- Page: /pages/users/[...slug].vue
- Usage: params.slug
```

::

#### Пример: вложенные маршруты и `definePageMeta`

::code-group

```vue [Nuxt 2]
<template>
  <div>
    <NuxtChild
      keep-alive
      :keep-alive-props="{ exclude: ['modal'] }"
      :nuxt-child-key="$route.slug"
    />
  </div>
</template>

<script>
export default {
  transition: 'page', // or { name: 'page' }
}
</script>
```

```vue [Nuxt 3]
<template>
  <div>
    <NuxtPage />
  </div>
</template>

<script setup lang="ts">
// This compiler macro works in both <script> and <script setup>
definePageMeta({
  // you can also pass a string or a computed property
  key: route => route.slug,
  transition: {
    name: 'page',
  },
  keepalive: {
    exclude: ['modal'],
  },
})
</script>
```

::

## Компонент `<NuxtLink>`

Синтаксис и поведение глобального компонента [NuxtLink](/docs/4.x/api/components/nuxt-link) в целом те же. Сокращение `<NLink>` замените на `<NuxtLink>`.

`<NuxtLink>` теперь подходит для _всех_ ссылок, в том числе внешних. Подробнее — в документации и о расширении своего компонента ссылки.

:read-more{to="/docs/4.x/api/components/nuxt-link"}

## Программная навигация

При миграции с Nuxt 2 на Nuxt 3 обновите программную навигацию. В Nuxt 2 был `this.$router`. В Nuxt 3 используйте [`navigateTo()`](/docs/4.x/api/utils/navigate-to) с маршрутом и параметрами для Vue Router.

::warning
Всегда `await` для [`navigateTo`](/docs/4.x/api/utils/navigate-to) или возвращайте результат из функций.
::

::code-group

```vue [Nuxt 2]
<script>
export default {
  methods: {
    navigate () {
      this.$router.push({
        path: '/search',
        query: {
          name: 'first name',
          type: '1',
        },
      })
    },
  },
}
</script>
```

```vue [Nuxt 3]
<script setup lang="ts">
function navigate () {
  return navigateTo({
    path: '/search',
    query: {
      name: 'first name',
      type: '1',
    },
  })
}
</script>
```

::
