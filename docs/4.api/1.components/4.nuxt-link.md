::note
`<NuxtLink>` заменяет и `<RouterLink>` из Vue Router, и HTML-тег `<a>`. Он определяет, внутренняя ссылка или внешняя, и рендерит её соответствующим образом с доступными оптимизациями (prefetch, атрибуты по умолчанию и т.д.)
::

## Внутренняя маршрутизация

В этом примере `<NuxtLink>` ведёт на другую страницу приложения.

::code-group
```vue [app/pages/index.vue]
<template>
  <NuxtLink to="/about">About page</NuxtLink>
</template>
```

```html [(Renders as) index.html]
<!-- (Vue Router & Smart Prefetching) -->
<a href="/about">About page</a>
```
::

### Параметры динамических маршрутов

Здесь в маршрут `~/pages/posts/[id].vue` передаётся параметр `id`.

::code-group
```vue [app/pages/index.vue]
<template>
  <NuxtLink :to="{ name: 'posts-id', params: { id: 123 } }">
    Post 123
  </NuxtLink>
</template>
```

```html [(Renders as) index.html]
<a href="/posts/123">Post 123</a>
```
::

::tip
В панели Pages в Nuxt DevTools видны имя маршрута и параметры.
::

::tip
При передаче объекта в проп `to` `<NuxtLink>` наследует обработку query-параметров Vue Router: ключи и значения кодируются автоматически, вызывать `encodeURI` или `encodeURIComponent` не нужно.
::

### Статические файлы и ссылки между приложениями

По умолчанию `<NuxtLink>` использует клиентскую навигацию Vue Router для относительных путей. Ссылки на статические файлы в `/public` или на другое приложение на том же домене могут давать неожиданные 404 — они не входят в клиентские маршруты. В таких случаях используйте проп `external`, чтобы обойти внутренний роутинг Vue Router.

Проп `external` явно помечает ссылку как внешнюю: `<NuxtLink>` рендерит обычный `<a>`, обходя логику Vue Router и указывая напрямую на ресурс.

#### Ссылки на статические файлы

Для файлов в `/public` (PDF, изображения и т.д.) задайте `external`, чтобы ссылка разрешалась корректно.

```vue [app/pages/index.vue]
<template>
  <NuxtLink
    to="/example-report.pdf"
    external
  >
    Download Report
  </NuxtLink>
</template>
```

#### URL другого приложения

Для другого приложения на том же домене `external` обеспечивает ожидаемое поведение.

```vue [app/pages/index.vue]
<template>
  <NuxtLink
    to="/another-app"
    external
  >
    Go to Another App
  </NuxtLink>
</template>
```

Явный `external` или автоматическая обработка дают корректную навигацию, избегают сюрпризов роутинга и лучше работают со статикой и кросс-приложенческими сценариями.

## Внешняя маршрутизация

Пример ссылки на внешний сайт.

```vue [app/app.vue]
<template>
  <NuxtLink to="https://nuxtjs.org">
    Nuxt website
  </NuxtLink>
  <!-- <a href="https://nuxtjs.org" rel="noopener noreferrer">...</a> -->
</template>
```

## Атрибуты `rel` и `noRel`

По умолчанию для ссылок с `target` или для абсолютных (`http://`, `https://`, `//`) добавляется `rel="noopener noreferrer"`.
- `noopener` закрывает [уязвимость](https://mathiasbynens.github.io/rel-noopener/) в старых браузерах.
- `noreferrer` не отправляет заголовок `Referer` на целевой сайт.

Это не вредит SEO и считается [хорошей практикой](https://developer.chrome.com/docs/lighthouse/best-practices/external-anchors-use-rel-noopener).

Чтобы переопределить поведение, используйте пропы `rel` или `noRel`.

```vue [app/app.vue]
<template>
  <NuxtLink to="https://twitter.com/nuxt_js">
    Nuxt Twitter
  </NuxtLink>
  <!-- <a href="https://twitter.com/nuxt_js" rel="noopener noreferrer">...</a> -->

  <NuxtLink
    to="https://discord.nuxtjs.org"
    rel="noopener"
  >
    Nuxt Discord
  </NuxtLink>
  <!-- <a href="https://discord.nuxtjs.org" rel="noopener">...</a> -->

  <NuxtLink
    to="/about"
    target="_blank"
  >About page</NuxtLink>
  <!-- <a href="/about" target="_blank" rel="noopener noreferrer">...</a> -->
</template>
```

Проп `noRel` отключает добавление `rel` по умолчанию для абсолютных ссылок.

```vue [app/app.vue]
<template>
  <NuxtLink
    to="https://github.com/nuxt"
    no-rel
  >
    Nuxt GitHub
  </NuxtLink>
  <!-- <a href="https://github.com/nuxt">...</a> -->
</template>
```

::note
`noRel` и `rel` нельзя использовать вместе: `rel` будет проигнорирован.
::

## Prefetch ссылок

Nuxt включает умный prefetch: когда ссылка видна (по умолчанию — во вьюпорте или при прокрутке), подгружается JS целевых страниц, чтобы по клику они были готовы. Ресурсы грузятся, когда браузер не занят; prefetch пропускается офлайн или при соединении 2G.

```vue [app/pages/index.vue]
<NuxtLink to="/about" no-prefetch>
About page not pre-fetched
</NuxtLink>

<NuxtLink to="/about" :prefetch="false">
About page not pre-fetched
</NuxtLink>
```

### Пользовательские триггеры prefetch

С версии `v3.13.0` для `<NuxtLink>` доступны пользовательские триггеры prefetch через проп `prefetchOn`.

```vue
<template>
  <NuxtLink prefetch-on="visibility">
    This will prefetch when it becomes visible (default)
  </NuxtLink>

  <NuxtLink prefetch-on="interaction">
    This will prefetch when hovered or when it gains focus
  </NuxtLink>
</template>
```

- `visibility`: prefetch при появлении ссылки во вьюпорте. Используется [Intersection Observer API](https://developer.mozilla.org/en-US/docs/Web/API/Intersection_Observer_API).
- `interaction`: prefetch при наведении или фокусе (`pointerenter`, `focus`).

Можно передать объект для `prefetchOn`:

```vue
<template>
  <NuxtLink :prefetch-on="{ interaction: true }">
    This will prefetch when hovered or when it gains focus
  </NuxtLink>
</template>
```

Обычно не нужно включать оба сразу.

```vue
<template>
  <NuxtLink :prefetch-on="{ visibility: true, interaction: true }">
    This will prefetch when hovered/focus - or when it becomes visible
  </NuxtLink>
</template>
```

Так отслеживается и видимость, и hover/focus — возможны лишние запросы и дублирующий prefetch.

### Кросс-ориджин prefetch

Кросс-ориджин prefetch: опция `crossOriginPrefetch` в `nuxt.config` и [Speculation Rules API](https://developer.mozilla.org/en-US/docs/Web/API/Speculation_Rules_API).

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  experimental: {
    crossOriginPrefetch: true,
  },
})
```

### Глобальное отключение prefetch

Prefetch для всех ссылок можно включить или отключить глобально.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  experimental: {
    defaults: {
      nuxtLink: {
        prefetch: false,
      },
    },
  },
})
```

## Пропы

### RouterLink

Без `external` `<NuxtLink>` поддерживает все пропы [`RouterLink`](https://router.vuejs.org/api/interfaces/routerlinkprops) из Vue Router:

- `to`: любой URL или [объект местоположения маршрута](https://router.vuejs.org/api/type-aliases/routelocation)
- `custom`: не оборачивать контент в `<a>`; полный контроль над разметкой и навигацией по клику — как [проп `custom` в Vue Router](https://router.vuejs.org/api/interfaces/routerlinkprops#custom-)
- `exactActiveClass`: класс для точного совпадения активного маршрута — как [`exactActiveClass`](https://router.vuejs.org/api/interfaces/routerlinkprops#exactActiveClass-); по умолчанию `"router-link-exact-active"`
- `activeClass`: класс для активной ссылки — как [`activeClass`](https://router.vuejs.org/api/interfaces/routerlinkprops#activeClass-); по умолчанию `"router-link-active"`
- `replace`: как [`replace`](https://router.vuejs.org/api/interfaces/routelocationoptions#replace-) у внутренних ссылок
- `ariaCurrentValue`: значение `aria-current` для точного совпадения — как [`ariaCurrentValue`](https://router.vuejs.org/api/interfaces/routerlinkprops#ariaCurrentValue-)

### NuxtLink

- `href`: синоним `to`; при наличии обоих учитывается только `to`
- `noRel`: при `true` не добавляется `rel` для внешней ссылки
- `external`: принудительно рендер как `<a>`, а не `RouterLink`
- `prefetch`: при включении — prefetch middleware, layouts и payload (с [payloadExtraction](/docs/4.x/guide/going-further/experimental-features#payloadextraction)) для ссылок во вьюпорте; используется с экспериментальным [crossOriginPrefetch](/docs/4.x/guide/going-further/experimental-features#crossoriginprefetch)
- `prefetchOn`: когда prefetch’ить: `interaction`, `visibility` (по умолчанию) или объект, например `{ interaction: true, visibility: true }`. Учитывается при включённом `prefetch` и без `noPrefetch`
- `noPrefetch`: отключает prefetch
- `prefetchedClass`: класс для уже prefetch’нутых ссылок

### Якорь (`<a>`)

- `target`: значение атрибута `target`
- `rel`: значение `rel`; для внешних ссылок по умолчанию `"noopener noreferrer"`

::tip
Значения по умолчанию можно переопределить — см. [overwriting defaults](/docs/4.x/api/components/nuxt-link#overwriting-defaults).
::

## Переопределение значений по умолчанию

### В nuxt.config

Часть настроек `<NuxtLink>` задаётся в [`nuxt.config`](/docs/4.x/guide/going-further/experimental-features#defaults)

::important
В будущем эти опции могут переехать, например в `app.config` или каталог `app/`.
::

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  experimental: {
    defaults: {
      nuxtLink: {
        // default values
        componentName: 'NuxtLink',
        externalRelAttribute: 'noopener noreferrer',
        activeClass: 'router-link-active',
        exactActiveClass: 'router-link-exact-active',
        prefetchedClass: undefined, // can be any valid string class name
        trailingSlash: undefined, // can be 'append' or 'remove'
        prefetch: true,
        prefetchOn: { visibility: true },
      },
    },
  },
})
```

### Свой компонент ссылки

Свой компонент ссылок через `defineNuxtLink` и новые значения по умолчанию.

```ts [app/components/MyNuxtLink.ts]
export default defineNuxtLink({
  componentName: 'MyNuxtLink',
  /* see signature below for more */
})
```

Далее используйте `<MyNuxtLink />` как обычно.

### Сигнатура `defineNuxtLink`

```ts
interface NuxtLinkOptions {
  componentName?: string
  externalRelAttribute?: string
  activeClass?: string
  exactActiveClass?: string
  trailingSlash?: 'append' | 'remove'
  prefetch?: boolean
  prefetchedClass?: string
  prefetchOn?: Partial<{
    visibility: boolean
    interaction: boolean
  }>
}
function defineNuxtLink (options: NuxtLinkOptions): Component {}
```

- `componentName`: имя компонента; по умолчанию `NuxtLink`
- `externalRelAttribute`: значение `rel` для внешних ссылок; по умолчанию `"noopener noreferrer"`; `""` — отключить
- `activeClass`: класс активной ссылки — как [`linkActiveClass`](https://router.vuejs.org/api/interfaces/routeroptions#linkActiveClass-) в Vue Router; по умолчанию `"router-link-active"`
- `exactActiveClass`: класс при точном совпадении — как [`linkExactActiveClass`](https://router.vuejs.org/api/interfaces/routeroptions#linkExactActiveClass-); по умолчанию `"router-link-exact-active"`
- `trailingSlash`: добавить или убрать завершающий слэш в `href`; иначе игнорируется
- `prefetch`: prefetch ссылок по умолчанию
- `prefetchOn`: какие стратегии prefetch применять по умолчанию
- `prefetchedClass`: класс для prefetch’нутых ссылок
