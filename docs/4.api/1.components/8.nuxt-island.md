При рендере island-содержимое статично, клиентский JS для него не загружается.

Изменение пропов island-компонента вызывает повторный запрос и перерисовку.

::note
Глобальные стили приложения приходят в ответе вместе с island.
::

::tip
Server-only компоненты под капотом используют `<NuxtIsland>`
::

## Пропы

- `name` : имя рендеримого компонента.
  - **тип**: `string`
  - **обязательный**
- `lazy`: неблокирующий рендер.
  - **тип**: `boolean`
  - **по умолчанию**: `false`
- `props`: пропы для целевого компонента.
  - **тип**: `Record<string, any>`
- `source`: удалённый источник для рендера island.
  - **тип**: `string`
- **dangerouslyLoadClientComponents**: нужно для загрузки клиентских компонентов с удалённого источника.
  - **тип**: `boolean`
  - **по умолчанию**: `false`

::note
Удалённым island’ам нужен `experimental.componentIslands: 'local+remote'` в `nuxt.config`.
::

::warning{icon="i-ph-warning-duotone"}
Проп `source` для контента с удалённого сервера по сути доверяет этому серверу безопасный HTML. Это сопоставимо с `v-html` для внешнего контента — сервер может подставить произвольный HTML, в том числе вредоносный. **Используйте `source` только для серверов, которым полностью доверяете.**

Проп `dangerouslyLoadClientComponents` добавляет риск: загрузка и выполнение клиентских компонентов с удалённого источника. Даже при `dangerouslyLoadClientComponents: false` вы доверяете HTML с удалённого сервера.
::

::note
По умолчанию island-компоненты ищутся в `~/components/islands/`. Например, `~/components/islands/MyIsland.vue` рендерится как `<NuxtIsland name="MyIsland" />`.
::

## Слоты

Слоты можно передать island-компоненту, если они объявлены.

Каждый слот интерактивен, так как его задаёт родитель.

Некоторые слоты зарезервированы у `NuxtIsland` для особых случаев.

- `#fallback`: содержимое до загрузки island (при `lazy`) или при ошибке fetch у `NuxtIsland`.

## Ref

- `refresh()`
  - **тип**: `() => Promise<void>`
  - **описание**: принудительно перезапросить и перерисовать server-компонент.

## События

- `error`
  - **параметры**:
    - **error**:
      - **тип**: `unknown`
  - **описание**: при неудачной загрузке нового island.
