`<NuxtPage>` — встроенный компонент Nuxt. Он отображает страницы верхнего уровня или вложенные страницы из каталога [`app/pages/`](/docs/4.x/directory-structure/app/pages).

::note
`<NuxtPage>` — обёртка над [`<RouterView>`](https://router.vuejs.org/api/interfaces/routerviewprops) из Vue Router. Его следует использовать вместо `<RouterView>`, потому что Nuxt дополнительно учитывает внутреннее состояние. Иначе `useRoute()` может возвращать неверные пути.
::

`<NuxtPage>` включает следующие компоненты:

```vue
<template>
  <RouterView v-slot="{ Component }">
    <!-- Optional, when using transitions -->
    <Transition>
      <!-- Optional, when using keep-alive -->
      <KeepAlive>
        <Suspense>
          <component :is="Component" />
        </Suspense>
      </KeepAlive>
    </Transition>
  </RouterView>
</template>
```

По умолчанию Nuxt не включает `<Transition>` и `<KeepAlive>`. Их можно включить в `nuxt.config` или задав свойства `transition` и `keepalive` у `<NuxtPage>`. Для конкретной страницы — через `definePageMeta` в компоненте страницы.

::warning
Если включаете `<Transition>` в компоненте страницы, убедитесь, что у страницы один корневой элемент.
::

Так как `<NuxtPage>` внутри использует `<Suspense>`, жизненный цикл при смене страниц отличается от обычного Vue-приложения.

В типичном Vue новый компонент страницы монтируется **только после** полного размонтирования предыдущего. В Nuxt из-за реализации Vue `<Suspense>` новый компонент страницы монтируется **до** размонтирования предыдущего.

## Пропы

- `name`: указывает `<RouterView>` рендерить компонент с соответствующим именем в `components` совпавшего маршрута.
  - тип: `string`
- `route`: объект маршрута со всеми разрешёнными компонентами.
  - тип: `RouteLocationNormalized`
- `pageKey`: управляет тем, когда компонент `NuxtPage` перерисовывается.
  - тип: `string` или `function`
- `transition`: глобальные переходы для всех страниц, рендеримых через `NuxtPage`.
  - тип: `boolean` или [`TransitionProps`](https://vuejs.org/api/built-in-components#transition)
- `keepalive`: сохранение состояния страниц, рендеримых через `NuxtPage`.
  - тип: `boolean` или [`KeepAliveProps`](https://vuejs.org/api/built-in-components#keepalive)

::tip
Nuxt сам подставляет `name` и `route`, сканируя и рендеря все Vue-файлы в каталоге `/pages`.
::

## Пример

Например, если передать ключ, который никогда не меняется, `<NuxtPage>` отрендерится один раз — при первом монтировании.

```vue [app/app.vue]
<template>
  <NuxtPage page-key="static" />
</template>
```

Можно использовать динамический ключ от текущего маршрута:

```html
<NuxtPage :page-key="route => route.fullPath" />
```

::warning
Не используйте здесь объект `$route` — это может ломать рендер страниц с `<Suspense>` у `<NuxtPage>`.
::

Альтернатива: передать `pageKey` как `key` через [`definePageMeta`](/docs/4.x/api/utils/define-page-meta) в `<script>` компонента из `/pages`.

```vue [app/pages/my-page.vue]
<script setup lang="ts">
definePageMeta({
  key: route => route.fullPath,
})
</script>
```

## Ref страницы

Чтобы получить `ref` компонента страницы, обращайтесь через `ref.value.pageRef`

````vue [app/app.vue]
<script setup lang="ts">
const page = ref()

function logFoo () {
  page.value.pageRef.foo()
}
</script>

<template>
  <NuxtPage ref="page" />
</template>
````

````vue [my-page.vue]
<script setup lang="ts">
const foo = () => {
  console.log('foo method called')
}

defineExpose({
  foo,
})
</script>
````

## Пользовательские пропы

`<NuxtPage>` принимает произвольные пропы для передачи дальше по иерархии.

Например, значение `foobar` попадёт в `NuxtPage` и затем в компоненты страниц.

```vue [app/app.vue]
<template>
  <NuxtPage :foobar="123" />
</template>
```

В компоненте страницы к пропу `foobar` можно обратиться так:

```vue [app/pages/page.vue]
<script setup lang="ts">
const props = defineProps<{ foobar: number }>()

console.log(props.foobar) // Outputs: 123
```

Если проп не объявлен через `defineProps`, переданные в `NuxtPage` атрибуты всё равно доступны через `attrs` страницы:

```vue [app/pages/page.vue]
<script setup lang="ts">
const attrs = useAttrs()
console.log(attrs.foobar) // Outputs: 123
</script>
```
