## Использование

Добавьте `<NuxtAnnouncer/>` в [`app.vue`](/docs/4.x/directory-structure/app/app) или [`app/layouts/`](/docs/4.x/directory-structure/app/layouts), чтобы объявлять скринридерам смену контента на странице: валидация, тосты, состояния загрузки и т.п.

```vue [app/app.vue]
<template>
  <NuxtAnnouncer />
  <NuxtRouteAnnouncer />
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>
```

Дальше в приложении вызывайте [`useAnnouncer`](/docs/4.x/api/composables/use-announcer):

```vue [app/pages/contact.vue]
<script setup lang="ts">
const { polite, assertive } = useAnnouncer()

async function submitForm () {
  try {
    await $fetch('/api/contact', { method: 'POST', body: formData })
    polite('Message sent successfully')
  } catch (error) {
    assertive('Error: Failed to send message')
  }
}
</script>
```

## Слоты

В слот по умолчанию можно передать свой HTML или компоненты.

```vue
<template>
  <NuxtAnnouncer>
    <template #default="{ message }">
      <p>{{ message }}</p>
    </template>
  </NuxtAnnouncer>
</template>
```

## Пропы

- `atomic`: при `true` зачитывается весь контент при обновлении, при `false` — только изменения. (по умолчанию `true`)
- `politeness`: срочность по умолчанию: `off`, `polite`, `assertive`. (по умолчанию `polite`)

## Отличия от `<NuxtRouteAnnouncer>`

| Аспект | `<NuxtRouteAnnouncer>` | `<NuxtAnnouncer>` |
|--------|------------------------|-------------------|
| **Назначение** | Смена маршрута/страницы | Любой динамический контент |
| **Триггер** | Автоматически при навигации | Вручную через `polite()` / `assertive()` |
| **Источник сообщения** | `<title>` страницы | Задаёт разработчик |
| **atomic по умолчанию** | `false` | `true` |
