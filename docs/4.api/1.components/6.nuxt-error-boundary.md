::tip
`<NuxtErrorBoundary>` внутри использует хук Vue [`onErrorCaptured`](https://vuejs.org/api/composition-api-lifecycle#onerrorcaptured).
::

## События

- `@error`: при ошибке в слоте по умолчанию.

  ```vue
  <template>
    <NuxtErrorBoundary @error="logSomeError">
      <!-- ... -->
    </NuxtErrorBoundary>
  </template>
  ```

## Слоты

- `#error`: запасной контент при ошибке.

  ```vue
  <template>
    <NuxtErrorBoundary>
      <!-- ... -->
      <template #error="{ error, clearError }">
        <p>An error occurred: {{ error }}</p>

        <button @click="clearError">
          Clear error
        </button>
      </template>
    </NuxtErrorBoundary>
  </template>
  ```

:read-more{to="/docs/4.x/getting-started/error-handling"}

## Примеры

### Доступ к `error` и `clearError` в script

Свойства `error` и `clearError` доступны из скрипта так:

```vue
<template>
  <NuxtErrorBoundary ref="errorBoundary">
    <!-- ... -->
  </NuxtErrorBoundary>
</template>

<script setup lang="ts">
const errorBoundary = useTemplateRef('errorBoundary')

// errorBoundary.value?.error
// errorBoundary.value?.clearError()
</script>
```
