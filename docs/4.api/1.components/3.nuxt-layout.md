Компонент `<NuxtLayout />` подключает макет по умолчанию в `app.vue` или `error.vue`.

```vue [app/app.vue]
<template>
  <NuxtLayout>
    контент страницы
  </NuxtLayout>
</template>
```

:read-more{to="/docs/4.x/directory-structure/app/layouts"}

## Пропсы

- `name`: имя макета для отображения (строка, ref или computed). Должно совпадать с именем файла в [`app/layouts/`](/docs/4.x/directory-structure/app/layouts) или `false`, чтобы отключить макет.
  - **тип**: `string | false`
  - **по умолчанию**: `default`

```vue [app/pages/index.vue]
<script setup lang="ts">
// layouts/custom.vue
const layout = 'custom'
</script>

<template>
  <NuxtLayout :name="layout">
    <NuxtPage />
  </NuxtLayout>
</template>
```

::note
Имя макета нормализуется в kebab-case: файл `errorLayout.vue` соответствует свойству `name="error-layout"` в `<NuxtLayout />`.
::

```vue [error.vue]
<template>
  <NuxtLayout name="error-layout">
    <NuxtPage />
  </NuxtLayout>
</template>
```

::read-more{to="/docs/4.x/directory-structure/app/layouts"}
Подробнее о динамических макетах.
::

- `fallback`: макет, который рендерится, если в `name` передан несуществующий. Должен совпадать с именем файла в [`app/layouts/`](/docs/4.x/directory-structure/app/layouts).
  - **тип**: `string`
  - **по умолчанию**: `null`

## Дополнительные пропсы

В `<NuxtLayout>` можно передать любые пропсы — они будут доступны в макете как атрибуты.

```vue [app/pages/some-page.vue]
<template>
  <div>
    <NuxtLayout
      name="custom"
      title="I am a custom layout"
    >
      <!-- ... -->
    </NuxtLayout>
  </div>
</template>
```

В примере выше `title` доступен в макете как `$attrs.title` в шаблоне или `useAttrs().title` в `<script setup>`.

```vue [app/layouts/custom.vue]
<script setup lang="ts">
const layoutCustomProps = useAttrs()

console.log(layoutCustomProps.title) // I am a custom layout
</script>
```

## Переходы

`<NuxtLayout />` renders incoming content via `<slot />`, which is then wrapped around Vue’s `<Transition />` component to activate layout transition. Чтобы переходы работали, `<NuxtLayout />` не должен быть корневым элементом страницы.

::code-group

```vue [app/pages/index.vue]
<template>
  <div>
    <NuxtLayout name="custom">
      <template #header>
        Контент шапки макета.
      </template>
    </NuxtLayout>
  </div>
</template>
```

```vue [app/layouts/custom.vue]
<template>
  <div>
    <!-- именованный слот -->
    <slot name="header" />
    <slot />
  </div>
</template>
```

::

:read-more{to="/docs/4.x/getting-started/transitions"}

## Ref макета

Чтобы получить ref компонента макета, используйте `ref.value.layoutRef`.

::code-group

```vue [app/app.vue]
<script setup lang="ts">
const layout = ref()

function logFoo () {
  layout.value.layoutRef.foo()
}
</script>

<template>
  <NuxtLayout ref="layout">
    default layout
  </NuxtLayout>
</template>
```

```vue [app/layouts/default.vue]
<script setup lang="ts">
const foo = () => console.log('foo')
defineExpose({
  foo,
})
</script>

<template>
  <div>
    default layout
    <slot />
  </div>
</template>
```
