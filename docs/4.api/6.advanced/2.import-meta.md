## Объект `import.meta`

С ES-модулями можно читать метаданные из кода, который импортирует или компилирует ваш модуль.
Это доступно как `import.meta` — объект, в котором среда передаёт эту информацию в ваш код.
В документации Nuxt эти поля часто используют, чтобы отличать выполнение на клиенте и на сервере.

## Свойства времени выполнения (приложение)

Эти значения подставляются статически и допускают вырезание неиспользуемого кода (tree-shaking) в коде времени выполнения.

Свойство                | Тип       | Описание
------------------------|-----------|----------------------------------------------------------------------
`import.meta.client`    | `boolean` | `true`, если вычисляется на клиенте.
`import.meta.browser`   | `boolean` | `true`, если вычисляется на клиенте.
`import.meta.server`    | `boolean` | `true`, если вычисляется на сервере.
`import.meta.nitro`     | `boolean` | `true`, если вычисляется на сервере.
`import.meta.dev`       | `boolean` | `true` при запуске dev-сервера Nuxt.
`import.meta.test`      | `boolean` | `true` в контексте тестов.
`import.meta.prerender` | `boolean` | `true` при пререндере HTML во время сборки.

## Свойства сборщика

Эти значения доступны в модулях и в `nuxt.config`.

Свойство          | Тип      | Описание
------------------|-----------|------------------------------------
`import.meta.env` | `object`  | То же, что `process.env`.
`import.meta.url` | `string`  | Абсолютный URL текущего модуля (файла).

## Примеры

### Разрешение файлов внутри модуля с `import.meta.url`

```ts [modules/my-module/index.ts]
import { createResolver } from 'nuxt/kit'

// Относительно этого файла
const resolver = createResolver(import.meta.url)

export default defineNuxtModule({
  meta: { name: 'myModule' },
  setup () {
    addComponent({
      name: 'MyModuleComponent',
      // Разрешается в '/modules/my-module/components/MyModuleComponent.vue'
      filePath: resolver.resolve('./components/MyModuleComponent.vue'),
    })
  },
})
```
