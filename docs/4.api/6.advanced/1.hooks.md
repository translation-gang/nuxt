## Хуки приложения (runtime)

Полный список хуков см. в [исходном коде приложения](https://github.com/nuxt/nuxt/blob/main/packages/nuxt/src/app/nuxt.ts#L37).

| Хук                         | Аргументы           | Окружение     | Описание                                                                                                                                                                           |
|------------------------------|---------------------|-----------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `app:created`                | `vueApp`            | Сервер и клиент | Вызывается при создании начального экземпляра `vueApp`.                                                                                                                                     |
| `app:error`                  | `err`               | Сервер и клиент | Вызывается при фатальной ошибке.                                                                                                                                                     |
| `app:error:cleared`          | `{ redirect? }`     | Сервер и клиент | Вызывается при сбросе фатальной ошибки.                                                                                                                                                     |
| `vue:setup`                  | -                   | Сервер и клиент | Вызывается при инициализации setup корня Nuxt. Колбэк должен быть синхронным.                                                                                                 |
| `vue:error`                  | `err, target, info` | Сервер и клиент | Вызывается, когда ошибка Vue доходит до корневого компонента. [Подробнее](https://vuejs.org/api/composition-api-lifecycle#onerrorcaptured).                                              |
| `app:rendered`               | `renderContext`     | Сервер          | Вызывается по завершении SSR-рендеринга.                                                                                                                                                    |
| `app:redirected`             | -                   | Сервер          | Вызывается перед редиректом при SSR.                                                                                                                                                        |
| `app:beforeMount`            | `vueApp`            | Клиент          | Вызывается перед монтированием приложения, только на клиенте.                                                                                                                           |
| `app:mounted`                | `vueApp`            | Клиент          | Вызывается, когда приложение Vue инициализировано и смонтировано в браузере.                                                                                                                            |
| `app:suspense:resolve`       | `appComponent`      | Клиент          | Событие разрешения [Suspense](https://vuejs.org/guide/built-ins/suspense#suspense).                                                                                                    |
| `app:manifest:update`        | `{ id, timestamp }` | Клиент          | Вызывается при обнаружении более новой версии приложения.                                                                                                                            |
| `app:data:refresh`           | `keys?`             | Клиент          | Вызывается при вызове `refreshNuxtData`.                                                                                                                                              |
| `link:prefetch`              | `to`                | Клиент          | Вызывается, когда `<NuxtLink>` попадает в зону префетча.                                                                                                                              |
| `page:start`                 | `pageComponent?`    | Клиент          | Вызывается при событии pending [Suspense](https://vuejs.org/guide/built-ins/suspense#suspense) внутри `NuxtPage`.                                                                         |
| `page:finish`                | `pageComponent?`    | Клиент          | Вызывается при событии resolved [Suspense](https://vuejs.org/guide/built-ins/suspense#suspense) внутри `NuxtPage`.                                                                        |
| `page:loading:start`         | -                   | Клиент          | Вызывается, когда выполняется `setup()` новой страницы.                                                                                                                                 |
| `page:loading:end`           | -                   | Клиент          | Вызывается после `page:finish`.                                                                                                                                                            |
| `page:transition:finish`     | `pageComponent?`    | Клиент          | После события [onAfterLeave](https://vuejs.org/guide/built-ins/transition#javascript-hooks) перехода страницы.                                                                            |
| `dev:ssr-logs`               | `logs`              | Клиент          | Вызывается с массивом серверных логов, переданных на клиент (если включено `features.devLogs`).                                                                      |
| `page:view-transition:start` | `transition`        | Клиент          | Вызывается после `document.startViewTransition`, когда [включена экспериментальная поддержка viewTransition](/docs/4.x/getting-started/transitions#view-transitions-api-experimental). Аргумент `transition` — объект [`ViewTransition`](https://developer.mozilla.org/en-US/docs/Web/API/ViewTransition) со свойством `types` ([`ViewTransitionTypeSet`](https://developer.mozilla.org/en-US/docs/Web/API/ViewTransitionTypeSet)), которое можно читать или менять. |

## Хуки Nuxt (время сборки)

Полный список хуков см. в [исходниках схемы](https://github.com/nuxt/nuxt/blob/main/packages/schema/src/types/hooks.ts#L83).

| Хук                        | Аргументы                                                | Описание                                                                                                                                                                                                                                                                                  |
|-----------------------------|----------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `kit:compatibility`         | `compatibility, issues`                                  | Позволяет расширять проверки совместимости.                                                                                                                                                                                                                                                       |
| `ready`                     | `nuxt`                                                   | Вызывается после инициализации Nuxt, когда экземпляр готов к работе.                                                                                                                                                                                                                   |
| `close`                     | `nuxt`                                                   | Вызывается при корректном завершении экземпляра Nuxt.                                                                                                                                                                                                                                             |
| `restart`                   | `{ hard?: boolean }`                                     | Следует вызывать для перезапуска текущего экземпляра Nuxt.                                                                                                                                                                                                                                           |
| `modules:before`            | -                                                        | Вызывается при инициализации Nuxt до установки пользовательских модулей.                                                                                                                                                                                                                           |
| `modules:done`              | -                                                        | Вызывается при инициализации Nuxt после установки пользовательских модулей.                                                                                                                                                                                                                            |
| `app:resolve`               | `app`                                                    | Вызывается после разрешения экземпляра `app`.                                                                                                                                                                                                                                                   |
| `app:templates`             | `app`                                                    | Вызывается при генерации `NuxtApp`, чтобы настраивать, менять или добавлять файлы в каталог сборки (виртуально или записывая в `.nuxt`).                                                                                                                                  |
| `app:templatesGenerated`    | `app`                                                    | Вызывается после компиляции шаблонов в [виртуальную файловую систему](/docs/4.x/directory-structure/nuxt) (vfs).                                                                                                                                                                                |
| `build:before`              | -                                                        | Вызывается перед сборщиком бандла Nuxt.                                                                                                                                                                                                                                                           |
| `build:done`                | -                                                        | Вызывается после завершения сборщика бандла Nuxt.                                                                                                                                                                                                                                                |
| `build:manifest`            | `manifest`                                               | Вызывается при сборке манифеста Vite и webpack. Позволяет настроить манифест, который Nitro использует для тегов `<script>` и `<link>` в итоговом HTML.                                                                                                                     |
| `builder:generateApp`       | `options`                                                | Вызывается перед генерацией приложения.                                                                                                                                                                                                                                                            |
| `builder:watch`             | `event, path`                                            | Вызывается при сборке в режиме разработки, когда наблюдатель замечает изменение файла или каталога проекта.                                                                                                                                                                                   |
| `pages:extend`              | `pages`                                                  | Вызывается после сканирования маршрутов страниц с диска.                                                                                                                                                                                                                                   |
| `pages:resolved`            | `pages`                                                  | Вызывается после обогащения маршрутов страниц просканированными метаданными.                                                                                                                                                                                                                          |
| `pages:routerOptions`       | `{ files: Array<{ path: string, optional?: boolean }> }` | Вызывается при разрешении файлов `router.options`. Более поздние элементы массива переопределяют более ранние.                                                                                                                                                                                                |
| `server:devHandler`         | `handler`                                                | Вызывается при регистрации dev-middleware на dev-сервере Nitro.                                                                                                                                                                                                                  |
| `imports:sources`           | `presets`                                                | Вызывается при настройке и позволяет модулям расширять источники.                                                                                                                                                                                                                                          |
| `imports:extend`            | `imports`                                                | Вызывается при настройке и позволяет модулям расширять импорты.                                                                                                                                                                                                                                          |
| `imports:context`           | `context`                                                | Вызывается при создании контекста [unimport](https://github.com/unjs/unimport).                                                                                                                                                                                                             |
| `imports:dirs`              | `dirs`                                                   | Позволяет расширять каталоги импорта.                                                                                                                                                                                                                                                         |
| `components:dirs`           | `dirs`                                                   | Вызывается внутри `app:resolve` и позволяет расширять каталоги сканирования автоимпортируемых компонентов.                                                                                                                                                                              |
| `components:extend`         | `components`                                             | Позволяет добавлять новые компоненты.                                                                                                                                                                                                                                                             |
| `nitro:config`              | `nitroConfig`                                            | Вызывается перед инициализацией Nitro и позволяет настроить конфигурацию Nitro.                                                                                                                                                                                                           |
| `nitro:init`                | `nitro`                                                  | Вызывается после инициализации Nitro; можно регистрировать хуки Nitro и работать с Nitro напрямую.                                                                                                                                                                                 |
| `nitro:build:before`        | `nitro`                                                  | Вызывается перед сборкой экземпляра Nitro.                                                                                                                                                                                                                                                   |
| `nitro:build:public-assets` | `nitro`                                                  | Вызывается после копирования публичных ассетов. Позволяет изменить публичные ассеты до сборки сервера Nitro.                                                                                                                                                                                             |
| `prerender:routes`          | `ctx`                                                    | Позволяет расширить список маршрутов для пререндера.                                                                                                                                                                                                                                              |
| `build:error`               | `error`                                                  | Вызывается при ошибке на этапе сборки.                                                                                                                                                                                                                                                   |
| `prepare:types`             | `options`                                                | Вызывается перед тем, как `@nuxt/cli` записывает файлы конфигурации TypeScript (`.nuxt/tsconfig.app.json`, `.nuxt/tsconfig.server.json` и т.д.) и `.nuxt/nuxt.d.ts`; можно добавить ссылки и объявления в `nuxt.d.ts` или изменить опции генерируемых конфигураций |
| `listen`                    | `listenerServer, listener`                               | Вызывается при загрузке dev-сервера.                                                                                                                                                                                                                                                       |
| `schema:extend`             | `schemas`                                                | Позволяет расширять схемы по умолчанию.                                                                                                                                                                                                                                                            |
| `schema:resolved`           | `schema`                                                 | Позволяет расширять разрешённую схему.                                                                                                                                                                                                                                                            |
| `schema:beforeWrite`        | `schema`                                                 | Вызывается перед записью указанной схемы.                                                                                                                                                                                                                                                      |
| `schema:written`            | -                                                        | Вызывается после записи схемы.                                                                                                                                                                                                                                                          |
| `vite:extend`               | `viteBuildContext`                                       | Позволяет расширять контекст Vite по умолчанию.                                                                                                                                                                                                                                                       |
| `vite:extendConfig`         | `viteInlineConfig, env`                                  | Позволяет расширять конфигурацию Vite по умолчанию. **Устарело в Nuxt 5+.** В Nuxt 5 работает с общей конфигурацией, а не с отдельными конфигами клиента/сервера.                                                                                                                              |
| `vite:configResolved`       | `viteInlineConfig, env`                                  | Позволяет читать разрешённую конфигурацию Vite. **Устарело в Nuxt 5+.** В Nuxt 5 работает с общей конфигурацией, а не с отдельными конфигами клиента/сервера.                                                                                                                           |
| `vite:serverCreated`        | `viteServer, env`                                        | Вызывается при создании сервера Vite.                                                                                                                                                                                                                                                      |
| `vite:compiled`             | -                                                        | Вызывается после компиляции сервера Vite.                                                                                                                                                                                                                                                        |
| `webpack:config`            | `webpackConfigs`                                         | Вызывается перед настройкой компилятора webpack.                                                                                                                                                                                                                                              |
| `webpack:configResolved`    | `webpackConfigs`                                         | Позволяет читать разрешённую конфигурацию webpack.                                                                                                                                                                                                                                                  |
| `webpack:compile`           | `options`                                                | Вызывается непосредственно перед компиляцией.                                                                                                                                                                                                                                                             |
| `webpack:compiled`          | `options`                                                | Вызывается после загрузки ресурсов.                                                                                                                                                                                                                                                           |
| `webpack:change`            | `shortPath`                                              | Вызывается при `change` в WebpackBar.                                                                                                                                                                                                                                                            |
| `webpack:error`             | -                                                        | Вызывается при `done`, если в WebpackBar есть ошибки.                                                                                                                                                                                                                                                |
| `webpack:done`              | -                                                        | Вызывается при `allDone` в WebpackBar.                                                                                                                                                                                                                                                           |
| `webpack:progress`          | `statesArray`                                            | Вызывается при `progress` в WebpackBar.                                                                                                                                                                                                                                                          |
