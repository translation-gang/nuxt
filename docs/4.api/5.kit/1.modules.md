Модули — строительные блоки Nuxt. Kit даёт утилиты для их создания и использования. Например, `defineNuxtModule` описывает модуль, а опция `moduleDependencies` задаёт зависимости от других модулей.

## `defineNuxtModule`

Описывает модуль Nuxt: сливает значения по умолчанию с опциями пользователя, подключает переданные хуки и при необходимости вызывает `setup` для полного контроля.

### Использование

```ts twoslash
import { defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  meta: {
    name: 'my-module',
    configKey: 'myModule',
  },
  defaults: {
    enabled: true,
  },
  setup (options) {
    if (options.enabled) {
      console.log('My Nuxt module is enabled!')
    }
  },
})
```

### Тип

```ts twoslash
// @errors: 2391
import type { ModuleDefinition, ModuleOptions, NuxtModule } from '@nuxt/schema'
// ---cut---
export function defineNuxtModule<TOptions extends ModuleOptions> (
  definition?: ModuleDefinition<TOptions, Partial<TOptions>, false> | NuxtModule<TOptions, Partial<TOptions>, false>,
): NuxtModule<TOptions, TOptions, false>

export function defineNuxtModule<TOptions extends ModuleOptions> (): {
  with: <TOptionsDefaults extends Partial<TOptions>> (
    definition: ModuleDefinition<TOptions, TOptionsDefaults, true> | NuxtModule<TOptions, TOptionsDefaults, true>,
  ) => NuxtModule<TOptions, TOptionsDefaults, true>
}
```

### Параметры

**definition**: объект описания модуля или функция-модуль. В объекте могут быть свойства:

| Property             | Type                                                                                                              | Required | Description                                                                                                                                                                                                        |
|----------------------|-------------------------------------------------------------------------------------------------------------------|----------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `meta`               | `ModuleMeta`                                                                                                      | `false`  | Метаданные: имя, версия, ключ конфига, совместимость.                                                                                                                         |
| `defaults`           | `T \| ((nuxt: Nuxt) => T)`{lang="ts"}                                                                             | `false`  | Значения по умолчанию. Если функция — первый аргумент экземпляр Nuxt.                                                                                         |
| `schema`             | `T`                                                                                                               | `false`  | Схема опций модуля; опции накладываются на схему.                                                                                                                                 |
| `hooks`              | `Partial<NuxtHooks>`{lang="ts"}                                                                                   | `false`  | Хуки, которые установит модуль.                                                                                                                              |
| `moduleDependencies` | `Record<string, ModuleDependency> \| ((nuxt: Nuxt) => Record<string, ModuleDependency>)`{lang="ts"}               | `false`  | Зависимости от других модулей (версии и конфиг). Объект или функция от `nuxt`. См. [пример](/docs/4.x/api/kit/modules#specifying-module-dependencies). |
| `onInstall`          | `(nuxt: Nuxt) => Awaitable<void>`{lang="ts"}                                                                      | `false`  | При первой установке модуля. Нужны `meta.name` и `meta.version`.                                                                                                   |
| `onUpgrade`          | `(nuxt: Nuxt, options: T, previousVersion: string) => Awaitable<void>`{lang="ts"}                                 | `false`  | При обновлении версии модуля. Нужны `meta.name` и `meta.version`.                                                                                       |
| `setup`              | `(this: void, resolvedOptions: T, nuxt: Nuxt) => Awaitable<void \| false \| ModuleSetupInstallResult>`{lang="ts"} | `false`  | Основная логика модуля.                                                                                                                               |

### Примеры

#### `configKey` для настройки из `nuxt.config`

```ts
import { defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  meta: {
    name: 'my-module',
    configKey: 'myModule',
  },
  defaults: {
    // Module options
    enabled: true,
  },
  setup (options) {
    if (options.enabled) {
      console.log('My Nuxt module is enabled!')
    }
  },
})
```

Опции задаются под этим ключом в `nuxt.config`:

```ts
export default defineNuxtConfig({
  myModule: {
    enabled: false,
  },
})
```

Полное отключение — ключ `false`: `setup` не выполняется, типы для опций модуля всё равно генерируются.

```ts
export default defineNuxtConfig({
  // Disable the module entirely
  myModule: false,
})
```

::tip
Удобно отключать модули, унаследованные из [слоёв Nuxt](/docs/4.x/guide/going-further/layers#disabling-modules-from-layers).
::

#### Требования совместимости

Если модуль опирается на API конкретных версий Nuxt, укажите `compatibility.nuxt`:

```ts
export default defineNuxtModule({
  meta: {
    name: '@nuxt/icon',
    configKey: 'icon',
    compatibility: {
      // Required nuxt version in semver format.
      nuxt: '>=3.0.0', // or use '^3.0.0'
    },
  },
  setup () {
    const resolver = createResolver(import.meta.url)
    // Implement
  },
})
```

При несовместимой версии в консоли будет предупреждение.

```terminal
 WARN  Module @nuxt/icon is disabled due to incompatibility issues:
 - [nuxt] Nuxt version ^3.1.0 is required but currently using 3.0.0
```

#### Типобезопасность через `.with()`

Для корректного вывода типов у смерженных опций используйте `.with()`:

```ts
import { defineNuxtModule } from '@nuxt/kit'

// Define your module options interface
interface ModuleOptions {
  apiKey: string
  baseURL: string
  timeout?: number
  retries?: number
}

export default defineNuxtModule<ModuleOptions>().with({
  meta: {
    name: '@nuxtjs/my-api',
    configKey: 'myApi',
  },
  defaults: {
    baseURL: 'https://api.example.com',
    timeout: 5000,
    retries: 3,
  },
  setup (resolvedOptions, nuxt) {
    // resolvedOptions is properly typed as:
    // {
    //   apiKey: string          // Required, no default provided
    //   baseURL: string         // Required, has default value
    //   timeout: number         // Optional, has default value
    //   retries: number         // Optional, has default value
    // }

    console.log(resolvedOptions.baseURL) // ✅ TypeScript knows this is always defined
    console.log(resolvedOptions.timeout) // ✅ TypeScript knows this is always defined
    console.log(resolvedOptions.retries) // ✅ TypeScript knows this is always defined
  },
})
```

Без `.with()` `resolvedOptions` соответствует «сырой» `ModuleOptions`, где поля с дефолтами могли бы оставаться `undefined`. `.with()` связывает дефолты с обязательностью полей в итоговом типе.

#### Хуки установки и обновления

`onInstall` и `onUpgrade` для разовых задач, миграций и очистки.

::important
Для хуков нужны **оба** поля: `meta.name` и `meta.version`. Состояние хранится в `.nuxtrc` проекта.
::

Хуки выполняются до `setup`; ошибка логируется и не останавливает сборку.

**`onInstall`** — один раз при первом добавлении модуля.

**`onUpgrade`** — при повышении версии по semver, один раз на шаг версии.

##### Пример

```ts
import { defineNuxtModule } from '@nuxt/kit'
import semver from 'semver'

export default defineNuxtModule({
  meta: {
    name: 'my-awesome-module',
    version: '1.2.0', // Required for lifecycle hooks
    configKey: 'myAwesomeModule',
  },
  defaults: {
    apiKey: '',
    enabled: true,
  },

  onInstall (nuxt) {
    // This runs only when the module is first installed
    console.log('Setting up my-awesome-module for the first time!')

    // You might want to:
    // - Create initial configuration files
    // - Set up database schemas
    // - Display welcome messages
    // - Perform initial data migration
  },

  onUpgrade (nuxt, options, previousVersion) {
    // This runs when the module is upgraded to a newer version
    console.log(`Upgrading my-awesome-module from ${previousVersion} to 1.2.0`)

    // You might want to:
    // - Migrate configuration files
    // - Update database schemas
    // - Clean up deprecated files
    // - Display upgrade notes

    if (semver.lt(previousVersion, '1.1.0')) {
      console.log('⚠️  Breaking changes in 1.1.0 - please check the migration guide')
    }
  },

  setup (options, nuxt) {
    // Regular setup logic runs on every build
    if (options.enabled) {
      // Configure the module
    }
  },
})
```

#### Зависимости между модулями

`moduleDependencies` объявляет зависимости: порядок инициализации, версии и конфигурация.

Объект или функция `(nuxt) => ...`:

##### Пример

```ts
import { defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  meta: {
    name: 'my-module',
  },
  moduleDependencies: {
    '@nuxtjs/tailwindcss': {
      // Specify a version constraint (semver format)
      version: '>=6.0.0',
      // Configuration that overrides user settings
      overrides: {
        exposeConfig: true,
      },
      // Configuration that sets defaults but respects user settings
      defaults: {
        config: {
          darkMode: 'class',
        },
      },
    },
    '@nuxtjs/fontaine': {
      // Optional dependencies won't be installed but ensure that options
      // can be set if they _are_ installed
      optional: true,
      defaults: {
        fonts: [
          {
            family: 'Roboto',
            fallbacks: ['Impact'],
          },
        ],
      },
    },
  },
  setup (options, nuxt) {

  },
})
```

Динамически от конфигурации Nuxt:

```ts
import { defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  meta: {
    name: 'my-module',
  },
  moduleDependencies (nuxt) {
    const dependencies: Record<string, any> = {
      '@nuxtjs/tailwindcss': {
        version: '>=6.0.0',
      },
    }

    // Conditionally add dependencies based on Nuxt config
    if (nuxt.options.experimental?.someFeature) {
      dependencies['@nuxtjs/fontaine'] = {
        optional: true,
      }
    }

    return dependencies
  },
  setup (options, nuxt) {
    // Your setup logic runs after all dependencies are initialized
  },
})
```

## `installModule`

::callout{type="warning"}
**Устарело:** используйте опцию [`moduleDependencies`](/docs/4.x/api/kit/modules#specifying-module-dependencies) в `defineNuxtModule`. `installModule` может быть удалён или стать неблокирующим в будущих версиях.
::

Программная установка модуля. Удобно, если модуль зависит от других. Опции передаются в `inlineOptions` и попадают в `setup` целевого модуля.

### Использование

```ts twoslash
import { defineNuxtModule, installModule } from '@nuxt/kit'

export default defineNuxtModule({
  async setup () {
    // will install @nuxtjs/fontaine with Roboto font and Impact fallback
    await installModule('@nuxtjs/fontaine', {
      // module configuration
      fonts: [
        {
          family: 'Roboto',
          fallbacks: ['Impact'],
          fallbackName: 'fallback-a',
        },
      ],
    })
  },
})
```

### Тип

```ts
async function installModule (moduleToInstall: string | NuxtModule, inlineOptions?: any, nuxt?: Nuxt)
```

### Параметры

| Property          | Type                              | Required | Description                                                                                   |
|-------------------|-----------------------------------|----------|-----------------------------------------------------------------------------------------------|
| `moduleToInstall` | `string \| NuxtModule`{lang="ts"} | `true`   | Имя модуля (строка) или объект модуля. |
| `inlineOptions`   | `any`                             | `false`  | Опции для `setup` модуля.              |
| `nuxt`            | `Nuxt`                            | `false`  | Экземпляр Nuxt; иначе из контекста `useNuxt()`.   |

### Примеры

```ts
import { defineNuxtModule, installModule } from '@nuxt/kit'

export default defineNuxtModule({
  async setup (options, nuxt) {
    // will install @nuxtjs/fontaine with Roboto font and Impact fallback
    await installModule('@nuxtjs/fontaine', {
      // module configuration
      fonts: [
        {
          family: 'Roboto',
          fallbacks: ['Impact'],
          fallbackName: 'fallback-a',
        },
      ],
    })
  },
})
```
