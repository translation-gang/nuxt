Компоненты — основа UI в Nuxt: переиспользуемые экземпляры Vue. По умолчанию компоненты из каталога `components` подключаются автоматически. Для другого каталога или выборочной регистрации используйте `addComponentsDir` и `addComponent` из `@nuxt/kit`.

::tip{icon="i-lucide-video" to="https://vueschool.io/lessons/injecting-components-and-component-directories?friend=nuxt" target="_blank"}
Видео Vue School о подключении компонентов и каталогов.
::

## `addComponentsDir`

Регистрирует каталог для сканирования компонентов; импорт при использовании. Глобальная регистрация только при `global: true`.

### Использование

```ts
export default defineNuxtModule({
  meta: {
    name: '@nuxt/ui',
    configKey: 'ui',
  },
  setup () {
    addComponentsDir({
      path: resolve('./runtime/components'),
      prefix: 'U',
      pathPrefix: false,
    })
  },
})
```

### Тип

```ts
function addComponentsDir (dir: ComponentsDir, opts: { prepend?: boolean } = {}): void
```

### Параметры

`dir` — объект со свойствами:

| Property          | Type                                                                                     | Required | Description                                                                                                                                                                                                                                             |
|-------------------|------------------------------------------------------------------------------------------|----------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `path`            | `string`                                                                                 | `true`   | Путь (абсолютный или относительный) к каталогу компонентов. Алиасы Nuxt (~ или @) или путь npm-пакета как в require.                                               |
| `pattern`         | `string \| string[]`{lang="ts"}                                                          | `false`  | Паттерн(ы) относительно `path`.                                                                                                                                                                                                 |
| `ignore`          | `string[]`                                                                               | `false`  | Паттерны игнорирования для `path`.                                                                                                                                                                                                |
| `prefix`          | `string`                                                                                 | `false`  | Префикс имён совпавших компонентов.                                                                                                                                                                                                         |
| `pathPrefix`      | `boolean`                                                                                | `false`  | Префикс имени компонента из пути.                                                                                                                                                                                                                      |
| `prefetch`        | `boolean`                                                                                | `false`  | В продакшене настраивает Lazy-компоненты через magic comments webpack. См. [документацию webpack](https://webpack.js.org/api/module-methods/#magic-comments) |
| `preload`         | `boolean`                                                                                | `false`  | То же для preload. См. [webpack](https://webpack.js.org/api/module-methods/#magic-comments) |
| `isAsync`         | `boolean`                                                                                | `false`  | Загружать компонент асинхронно (отдельный чанк) независимо от префикса Lazy.                                                                                                                                   |
| `extendComponent` | `(component: Component) => Promise<Component \| void> \| (Component \| void)`{lang="ts"} | `false`  | Вызывается для каждого найденного компонента; возвращает компонент или Promise.                                                          |
| `global`          | `boolean`                                                                                | `false`  | Глобальная регистрация.                                                                                                                                                                                              |
| `island`          | `boolean`                                                                                | `false`  | Регистрация как island. См. [`<NuxtIsland/>`](/docs/4.x/api/components/nuxt-island).                                                                                          |
| `watch`           | `boolean`                                                                                | `false`  | Следить за каталогом: добавление и удаление файлов.                                                                                                                                                                          |
| `extensions`      | `string[]`                                                                               | `false`  | Расширения, поддерживаемые сборщиком Nuxt.                                                                                                                                                                                                                   |
| `transpile`       | `'auto' \| boolean`{lang="ts"}                                                           | `false`  | Транспиляция через `build.transpile`. `'auto'` → `transpile: true`, если в пути есть `node_modules/`.                                                                                                                        |

`opts`

| Property  | Type      | Required | Description                                                                                          |
|-----------|-----------|----------|------------------------------------------------------------------------------------------------------|
| `prepend` | `boolean` | `false`  | При `true` каталог добавляется через `unshift()` вместо `push()`. |

## `addComponent`

Регистрирует компонент для автоимпорта.

### Использование

```ts
import { addComponent, createResolver, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  meta: {
    name: '@nuxt/image',
    configKey: 'image',
  },
  setup () {
    const resolver = createResolver(import.meta.url)

    addComponent({
      name: 'NuxtImg',
      filePath: resolver.resolve('./runtime/components/NuxtImg.vue'),
    })

    addComponent({
      name: 'NuxtPicture',
      filePath: resolver.resolve('./runtime/components/NuxtPicture.vue'),
    })
  },
})
```

### Тип

```ts
function addComponent (options: AddComponentOptions): void
```

### Параметры

`options`:

| Property          | Type                                       | Required | Description                                                                                                                                                                                                                                             |
|-------------------|--------------------------------------------|----------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|  
| `name`            | `string`                                   | `true`   | Имя компонента.                                                                                                                                                                                                                                         |
| `filePath`        | `string`                                   | `true`   | Путь к компоненту.                                                                                                                                                                                                                                  |
| `declarationPath` | `string`                                   | `false`  | Путь к файлу объявлений для [шаблонов типов](/docs/4.x/api/kit/templates#addtypetemplate); иначе используется `filePath`.                                                                    |
| `pascalName`      | `string`                                   | `false`  | PascalCase; иначе из `name`.                                                                                                                                                              |
| `kebabName`       | `string`                                   | `false`  | kebab-case; иначе из `name`.                                                                                                                                                               |
| `export`          | `string`                                   | `false`  | Именованный или default export; по умолчанию `'default'`.                                                                                                                                                                        |
| `shortPath`       | `string`                                   | `false`  | Короткий путь; иначе из пути компонента.                                                                                                                                                             |
| `chunkName`       | `string`                                   | `false`  | Имя чанка; иначе из имени компонента.                                                                                                                                                            |
| `prefetch`        | `boolean`                                  | `false`  | Lazy + webpack magic comments. См. [webpack](https://webpack.js.org/api/module-methods/#magic-comments) |  
| `preload`         | `boolean`                                  | `false`  | То же. См. [webpack](https://webpack.js.org/api/module-methods/#magic-comments) |
| `global`          | `boolean`                                  | `false`  | Глобальная регистрация.                                                                                                                                                                                               |
| `island`          | `boolean`                                  | `false`  | Island. См. [`<NuxtIsland/>`](/docs/4.x/api/components/nuxt-island).                                                                                            |
| `mode`            | `'client' \| 'server' \| 'all'`{lang="ts"} | `false`  | Клиент, сервер или оба. По умолчанию оба.                                                                                                                      |
| `priority`        | `number`                                   | `false`  | При совпадении имён выбирается больший приоритет.                                                                                                                                   |

### Примеры

Именованный экспорт из npm-пакета — укажите `export`:

```ts
import { addComponent, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  setup () {
    // import { MyComponent as MyAutoImportedComponent } from 'my-npm-package'
    addComponent({
      name: 'MyAutoImportedComponent',
      export: 'MyComponent',
      filePath: 'my-npm-package',
    })
  },
})
```
