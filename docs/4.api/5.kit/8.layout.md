Лейаут оборачивает страницы общими блоками (например, шапка и подвал). Регистрация — через `addLayout`.

## `addLayout`

Регистрирует шаблон как лейаут и добавляет его в набор лейаутов.

::note
В Nuxt 2 лейаут `error` тоже можно было зарегистрировать так. В Nuxt 3+ лейаут ошибки [заменён](/docs/4.x/getting-started/error-handling#error-page) на страницу `error.vue` в корне проекта.
::

### Использование

```ts twoslash
import { addLayout, createResolver, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  setup () {
    const { resolve } = createResolver(import.meta.url)

    addLayout({
      src: resolve('templates/custom-layout.ts'),
      filename: 'custom-layout.ts',
    }, 'custom')
  },
})
```

### Тип

```ts
function addLayout (layout: NuxtTemplate | string, name: string): void
```

### Параметры

**`layout`**: объект шаблона или строка с путём. Строка превращается в объект с `src`. У объекта шаблона свойства:

| Property      | Type                                             | Required | Description                                                                                                                                                                      |
| ------------- | ------------------------------------------------ | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `src`         | `string`                                         | `false`  | Путь к шаблону. Без `src` нужен `getContents`.                                                                                          |
| `filename`    | `string`                                         | `false`  | Имя файла. Если не задано — из `src` (тогда `src` обязателен).                                   |
| `dst`         | `string`                                         | `false`  | Путь назначения. Если не задан — из `filename` и `buildDir`.                                                |
| `options`     | `Record<string, any>`{lang="ts"}                 | `false`  | Опции для шаблона.                                                                                                                                                 |
| `getContents` | `(data) => string \| Promise<string>`{lang="ts"} | `false`  | Функция с объектом `options`, возвращает строку или Promise. При наличии `src` игнорируется. |
| `write`       | `boolean`                                        | `false`  | При `true` шаблон пишется на диск; иначе только виртуальная ФС.                                         |

**`name`**: имя лейаута (`default`, `custom` и т.д.).

### Пример

Регистрируется лейаут `custom` с шапкой и подвалом.

```ts twoslash
import { addLayout, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  setup () {
    addLayout({
      write: true,
      filename: 'my-layout.vue',
      getContents: () => `<template>
  <div>
    <header>My Header</header>
    <slot />
    <footer>My Footer</footer>
  </div>
</template>`,
    }, 'custom')
  },
})
```

Использование на странице:

```vue [app/pages/about.vue]
<script setup lang="ts">
definePageMeta({
  layout: 'custom',
})
</script>

<template>
  <div>About Page</div>
</template>
```

::warning
`@vitejs/plugin-vue` не поддерживает виртуальные `.vue` — передайте `write: true` в первый аргумент `addLayout`.
::
