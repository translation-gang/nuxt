## `extendPages`

В Nuxt маршруты строятся по файлам в `app/pages`. Иногда нужно кастомизировать маршруты: добавить динамический маршрут, удалить существующий или изменить конфигурацию. Для этого есть `extendPages` — расширение и изменение конфигурации страниц.

::tip{icon="i-lucide-video" to="https://vueschool.io/lessons/extend-and-alter-nuxt-pages?friend=nuxt" target="_blank"}
Видео Vue School о `extendPages`.
::

### Использование

```ts twoslash
import { createResolver, defineNuxtModule, extendPages } from '@nuxt/kit'

export default defineNuxtModule({
  setup (options) {
    const { resolve } = createResolver(import.meta.url)

    extendPages((pages) => {
      pages.unshift({
        name: 'prismic-preview',
        path: '/preview',
        file: resolve('runtime/preview.vue'),
      })
    })
  },
})
```

### Тип

```ts
function extendPages (callback: (pages: NuxtPage[]) => void): void
```

### Параметры

**callback**: функция с массивом страниц. Меняйте этот массив напрямую (добавление, удаление, правка). Копия массива в конфигурацию не попадёт.

| Property   | Type                               | Required | Description                                                                                  |
| ---------- | ---------------------------------- | -------- | -------------------------------------------------------------------------------------------- |
| `name`     | `string`                           | `false`  | Имя маршрута: навигация и идентификация.            |
| `path`     | `string`                           | `false`  | URL-путь. Если не задан, выводится из расположения файла.                   |
| `file`     | `string`                           | `false`  | Путь к Vue-файлу компонента маршрута.                     |
| `meta`     | `Record<string, any>`{lang="ts"}   | `false`  | Метаданные маршрута: лейауты, middleware, guards.    |
| `alias`    | `string[] \| string`{lang="ts"}    | `false`  | Алиасы путей.                  |
| `redirect` | `RouteLocationRaw`{lang="ts"}      | `false`  | Правило редиректа: именованные маршруты, объекты, строки.                |
| `children` | `NuxtPage[]`{lang="ts"}            | `false`  | Вложенные дочерние маршруты.                             |

## `extendRouteRules`

Nuxt использует серверный движок [Nitro](https://nitro.build/). В конфигурации можно задавать редиректы, прокси, кэш и заголовки для шаблонов маршрутов.

::tip
Подробнее о правилах маршрутов Nitro — в [документации Nitro](https://nitro.build/guide/routing#route-rules).
::

::tip{icon="i-lucide-video" to="https://vueschool.io/lessons/adding-route-rules-and-route-middlewares?friend=nuxt" target="_blank"}
Видео Vue School о правилах маршрутов и route middleware.
::

### Использование

```ts twoslash
import { createResolver, defineNuxtModule, extendPages, extendRouteRules } from '@nuxt/kit'

export default defineNuxtModule({
  setup (options) {
    const { resolve } = createResolver(import.meta.url)

    extendPages((pages) => {
      pages.unshift({
        name: 'preview-new',
        path: '/preview-new',
        file: resolve('runtime/preview.vue'),
      })
    })

    extendRouteRules('/preview', {
      redirect: {
        to: '/preview-new',
        statusCode: 302,
      },
    })

    extendRouteRules('/preview-new', {
      cache: {
        maxAge: 60 * 60 * 24 * 7,
      },
    })
  },
})
```

### Тип

```ts
function extendRouteRules (route: string, rule: NitroRouteConfig, options?: ExtendRouteRulesOptions): void
```

### Параметры

**route**: шаблон маршрута для сопоставления.\
**rule**: конфигурация правила для совпавшего маршрута.

::tip
Опции правил маршрутов — в [Гибридный рендеринг > Правила маршрутов](/docs/4.x/guide/concepts/rendering#route-rules).
::

**options**: если `override: true`, существующая конфигурация правила перезаписывается.

| Name       | Type      | Default | Description                                  |
| ---------- | --------- | ------- | -------------------------------------------- |
| `override` | `boolean` | `false` | Перезаписать правило маршрута; по умолчанию `false` |

## `addRouteMiddleware`

Регистрирует route middleware для всех или выбранных маршрутов.

Middleware также можно объявлять в плагинах через композабл [`addRouteMiddleware`](/docs/4.x/api/utils/add-route-middleware).

::tip
Подробнее — [документация по route middleware](/docs/4.x/getting-started/routing#route-middleware).
::

::tip{icon="i-lucide-video" to="https://vueschool.io/lessons/adding-route-rules-and-route-middlewares?friend=nuxt" target="_blank"}
Видео Vue School о правилах маршрутов и middleware.
::

### Использование

::code-group

```ts twoslash [module.ts]
import { addRouteMiddleware, createResolver, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  setup () {
    const { resolve } = createResolver(import.meta.url)

    addRouteMiddleware({
      name: 'auth',
      path: resolve('runtime/auth'),
      global: true,
    }, { prepend: true })
  },
})
```

```ts twoslash [runtime/auth.ts]
function isAuthenticated (): boolean { return false }
// ---cut---
export default defineNuxtRouteMiddleware((to, from) => {
  // isAuthenticated() is an example method verifying if a user is authenticated
  if (to.path !== '/login' && isAuthenticated() === false) {
    return navigateTo('/login')
  }
})
```

::

### Тип

```ts
function addRouteMiddleware (input: NuxtMiddleware | NuxtMiddleware[], options?: AddRouteMiddlewareOptions): void
```

### Параметры

**input**: объект middleware или массив объектов:

| Property | Type      | Required | Description                                         |
| -------- | --------- | -------- | --------------------------------------------------- |
| `name`   | `string`  | `true`   | Имя middleware.                         |
| `path`   | `string`  | `true`   | Путь к файлу middleware.                    |
| `global` | `boolean` | `false`  | При `true` — для всех маршрутов. |

**options**:

| Property   | Type      | Default | Description                                                 |
| ---------- | --------- | ------- | ----------------------------------------------------------- |
| `override` | `boolean` | `false` | При `true` заменяет middleware с тем же именем.          |
| `prepend`  | `boolean` | `false` | При `true` добавляет перед существующими middleware. |
