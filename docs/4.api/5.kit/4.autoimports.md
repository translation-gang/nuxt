Nuxt автоматически импортирует вспомогательные функции, композаблы и API Vue по всему приложению без явных `import`. По структуре каталогов приложение также может автоимпортировать свои композаблы и плагины.

Через Nuxt Kit можно добавлять собственные автоимпорты: `addImports` и `addImportsDir` — импорты в приложение Nuxt; `addImportsSources` — перечисленные импорты из сторонних пакетов.

Под капотом используется [`unimport`](https://github.com/unjs/unimport) — механизм автоимпорта в Nuxt.

::note
Эти функции предназначены для регистрации утилит, композаблов и API Vue. Для страниц, компонентов и плагинов см. разделы [Pages](/docs/4.x/api/kit/pages), [Components](/docs/4.x/api/kit/components), [Plugins](/docs/4.x/api/kit/plugins).
::

## `addImports`

Добавляет импорты в приложение Nuxt. Они доступны в контексте приложения без ручного импорта.

::tip
Для контекста сервера Nitro используйте [`addServerImports`](/docs/4.x/api/kit/nitro#addserverimports).
::

### Использование

```ts twoslash
import { addImports, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  setup (options, nuxt) {
    const names = [
      'useStoryblok',
      'useStoryblokApi',
      'useStoryblokBridge',
      'renderRichText',
      'RichTextSchema',
    ]

    names.forEach(name =>
      addImports({ name, as: name, from: '@storyblok/vue' }),
    )
  },
})
```

### Тип

```ts
function addImports (imports: Import | Import[]): void
```

### Параметры

`imports`: объект или массив объектов со свойствами:

| Property   | Type                  | Required | Description                                                                                                     |
|------------|-----------------------|----------|-----------------------------------------------------------------------------------------------------------------|
| `name`     | `string`              | `true`   | Имя импорта для обнаружения.                                                                                     |
| `from`     | `string`              | `true`   | Спецификатор модуля.                                                                                |
| `priority` | `number`              | `false`  | Приоритет; при совпадении имён используется импорт с большим приоритетом. |
| `disabled` | `boolean`             | `false`  | Отключён ли этот импорт.                                                                                     |
| `meta`     | `Record<string, any>` | `false`  | Метаданные импорта.                                                                                         |
| `type`     | `boolean`             | `false`  | Чистый type-only импорт.                                                                           |
| `typeFrom` | `string`              | `false`  | Значение `from` при генерации объявлений типов.                                                 |
| `as`       | `string`              | `false`  | Импортировать под этим именем.                                                                                            |

## `addImportsDir`

Добавляет импорты из каталога: все файлы каталога подхватываются и доступны без ручного импорта.

### Использование

```ts twoslash
import { addImportsDir, createResolver, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  meta: {
    name: '@vueuse/motion',
    configKey: 'motion',
  },
  setup (options, nuxt) {
    const resolver = createResolver(import.meta.url)
    addImportsDir(resolver.resolve('./runtime/composables'))
  },
})
```

### Тип

```ts
function addImportsDir (dirs: string | string[], options?: { prepend?: boolean }): void
```

### Параметры

| Property  | Type                               | Required | Description                                                                                                         |
|-----------|------------------------------------|----------|---------------------------------------------------------------------------------------------------------------------|
| `dirs`    | `string \| string[]`{lang="ts"}    | `true`   | Путь или массив путей к каталогу.                                      |
| `options` | `{ prepend?: boolean }`{lang="ts"} | `false`  | Если `prepend: true`, импорты добавляются в начало списка. |

## `addImportsSources`

Добавляет перечисленные импорты в приложение Nuxt.

### Использование

```ts twoslash
import { addImportsSources, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  setup () {
    addImportsSources({
      from: 'h3',
      imports: [
        'defineEventHandler',
        'getQuery',
        'getRouterParams',
        'readBody',
        'sendRedirect',
      ],
    })
  },
})
```

### Тип

```ts
function addImportsSources (importSources: ImportSource | ImportSource[]): void
```

### Параметры

**importSources**: объект или массив объектов со свойствами:

| Property  | Type                                        | Required | Description                                                                                    |
|-----------|---------------------------------------------|----------|------------------------------------------------------------------------------------------------|
| `from`    | `string`                                    | `true`   | Спецификатор модуля.                                                               |
| `imports` | `PresetImport \| ImportSource[]`{lang="ts"} | `true`   | Имена импортов, объекты импортов или источники импортов. |
