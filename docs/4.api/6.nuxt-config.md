## alias

Можно улучшить DX, задав дополнительные алиасы для доступа к своим каталогам в JavaScript и CSS.

- **Тип**: `object`
- **По умолчанию**
```json
{
  "~": "/<rootDir>/app",
  "@": "/<rootDir>/app",
  "~~": "/<rootDir>",
  "@@": "/<rootDir>",
  "#shared": "/<rootDir>/shared",
  "#server": "/<rootDir>/server",
  "assets": "/<rootDir>/app/assets",
  "public": "/<rootDir>/public",
  "#build": "/<rootDir>/.nuxt",
  "#internal/nuxt/paths": "/<rootDir>/.nuxt/paths.mjs"
}
```

::callout
**Примечание**: в контексте webpack (источники изображений, CSS — но не JavaScript) алиас
нужно обязательно указывать с префиксом `~`.
::

::callout
<!-- @case-police-ignore tsConfig -->
**Примечание**: эти алиасы автоматически попадут в сгенерированные конфигурации TypeScript (`.nuxt/tsconfig.app.json`, `.nuxt/tsconfig.server.json` и т.д.), чтобы были типы и автодополнение путей. Если нужно донастроить опции сгенерированных конфигов, добавьте их здесь или в свойстве `typescript.tsConfig` в `nuxt.config`.
::

**Пример**:
```ts
import { fileURLToPath } from 'node:url'

export default defineNuxtConfig({
  alias: {
    'images': fileURLToPath(new URL('./assets/images', import.meta.url)),
    'style': fileURLToPath(new URL('./assets/style', import.meta.url)),
    'data': fileURLToPath(new URL('./assets/other/data', import.meta.url)),
  },
})
```

```html
<template>
  <img src="~images/main-bg.jpg">
</template>

<script>
import data from 'data/test.json'
</script>

<style>
// Uncomment the below
//@import '~style/variables.scss';
//@import '~style/utils.scss';
//@import '~style/base.scss';
body {
  background-image: url('~images/main-bg.jpg');
}
</style>
```

## analyzeDir

Каталог, куда Nuxt сохраняет сгенерированные файлы при запуске `nuxt analyze`.

Относительный путь считается от `rootDir`.

- **Тип**: `string`
- **По умолчанию:** `"/<rootDir>/.nuxt/analyze"`

## app

Конфигурация приложения Nuxt.

### `baseURL`

Базовый путь приложения Nuxt.

Например:

- **Тип**: `string`
- **По умолчанию:** `"/"`

**Пример**:
```ts
export default defineNuxtConfig({
  app: {
    baseURL: '/prefix/',
  },
})
```

Также можно задать во время выполнения через переменную окружения NUXT_APP_BASE_URL.

**Пример**:
```bash
NUXT_APP_BASE_URL=/prefix/ node .output/server/index.mjs
```

### `buildAssetsDir`

Имя каталога со встроенными ассетами сайта, относительно `baseURL` (или `cdnURL`, если задан). Задаётся на этапе сборки и не должно меняться в runtime.

- **Тип**: `string`
- **По умолчанию:** `"/_nuxt/"`

### `cdnURL`

Абсолютный URL для раздачи публичной папки (только production).

Например:

- **Тип**: `string`
- **По умолчанию:** `""`

**Пример**:
```ts
export default defineNuxtConfig({
  app: {
    cdnURL: 'https://mycdn.org/',
  },
})
```

В runtime значение можно переопределить переменной окружения `NUXT_APP_CDN_URL`.

**Пример**:
```bash
NUXT_APP_CDN_URL=https://mycdn.org/ node .output/server/index.mjs
```

### `head`

Значения по умолчанию для `<head>` на каждой странице.

- **Тип**: `object`
- **По умолчанию**
```json
{
  "meta": [
    {
      "name": "viewport",
      "content": "width=device-width, initial-scale=1"
    },
    {
      "charset": "utf-8"
    }
  ],
  "link": [],
  "style": [],
  "script": [],
  "noscript": []
}
```

**Пример**:
```ts
export default defineNuxtConfig({
  app: {
    head: {
      meta: [
      // <meta name="viewport" content="width=device-width, initial-scale=1">
        { name: 'viewport', content: 'width=device-width, initial-scale=1' },
      ],
      script: [
      // <script src="https://myawesome-lib.js"></script>
        { src: 'https://awesome-lib.js' },
      ],
      link: [
      // <link rel="stylesheet" href="https://myawesome-lib.css">
        { rel: 'stylesheet', href: 'https://awesome-lib.css' },
      ],
      // please note that this is an area that is likely to change
      style: [
      // <style>:root { color: red }</style>
        { textContent: ':root { color: red }' },
      ],
      noscript: [
      // <noscript>JavaScript is required</noscript>
        { textContent: 'JavaScript is required' },
      ],
    },
  },
})
```

### `keepalive`

Значения по умолчанию для KeepAlive между страницами.

Можно переопределить через `definePageMeta` на отдельной странице. Допускаются только значения, сериализуемые в JSON.

- **Тип**: `boolean`
- **По умолчанию:** `false`

**См.**: [Vue KeepAlive](https://vuejs.org/api/built-in-components#keepalive)

### `layoutTransition`

Значения по умолчанию для переходов между макетами.

Можно переопределить через `definePageMeta` на отдельной странице. Допускаются только значения, сериализуемые в JSON.

- **Тип**: `boolean | TransitionProps`
- **По умолчанию:** `false`

**См.**: [Vue Transition docs](https://vuejs.org/api/built-in-components#transition)

### `pageTransition`

Значения по умолчанию для переходов между страницами.

Можно переопределить через `definePageMeta` на отдельной странице. Допускаются только значения, сериализуемые в JSON.

- **Тип**: `boolean | TransitionProps`
- **По умолчанию:** `false`

**См.**: [Vue Transition docs](https://vuejs.org/api/built-in-components#transition)

### `rootAttrs`

Настройка атрибутов корневого элемента Nuxt.

- **Тип**: `object`
- **По умолчанию**
```json
{
  "id": "__nuxt"
}
```

### `rootId`

Настройка id корневого элемента Nuxt.

- **Тип**: `string`
- **По умолчанию:** `"__nuxt"`

### `rootTag`

### `spaLoaderAttrs`

Настройка атрибутов элемента шаблона загрузки SPA в Nuxt.

- **Тип**: `object`
- **По умолчанию:**
```json

}
```

- **Тип**: `string`
- **По умолчанию:** `"__nuxt-loader"`

### `spaLoaderTag`

Настройка тега элемента SpaLoader в Nuxt.

- **Тип**: `string`
- **По умолчанию:** `"div"`

### `teleportAttrs`

Настройка атрибутов элемента Teleport в Nuxt.

- **Тип**: `object`
- **По умолчанию**
```json
{
  "id": "teleports"
}
```

### `teleportId`

Настройка id элемента Teleport в Nuxt.

- **Тип**: `string`
- **По умолчанию:** `"teleports"`

### `teleportTag`

Настройка тега элемента Teleport в Nuxt.

- **Тип**: `string`
- **По умолчанию:** `"div"`

### `viewTransition`

Значения по умолчанию для view transitions.

Учитывается только если **экспериментальная** поддержка View Transitions [включена в nuxt.config](/docs/4.x/getting-started/transitions#view-transitions-api-experimental).
Можно переопределить через `definePageMeta` на отдельной странице.

- **Тип**: `boolean`
- **По умолчанию:** `false`

**См.**: [Nuxt View Transition API docs](https://nuxt.com/docs/4.x/getting-started/transitions#view-transitions-api-experimental)

## appConfig

Дополнительная конфигурация приложения

Для программного доступа и поддержки типов можно задать конфиг приложения этой опцией. Он объединится с файлом `app.config` как значение по умолчанию.

### `nuxt`

## appId

Для проектов с несколькими приложениями — уникальный id приложения Nuxt.

По умолчанию `nuxt-app`.

- **Тип**: `string`
- **По умолчанию:** `"nuxt-app"`

## build

Общие настройки сборки.

### `analyze`

Nuxt позволяет визуализировать бандлы и понимать, как их оптимизировать.

Укажите `true`, чтобы включить анализ бандла, или передайте объект с опциями: [для webpack](https://github.com/webpack/webpack-bundle-analyzer#options-for-plugin) или [для vite](https://github.com/btd/rollup-plugin-visualizer#options).

- **Тип**: `object`
- **По умолчанию**
```json
{
  "template": "treemap",
  "projectRoot": "/<rootDir>",
  "filename": "/<rootDir>/.nuxt/analyze/{name}.html"
}
```

**Пример**:
```ts
export default defineNuxtConfig({
  analyze: {
    analyzerMode: 'static',
  },
})
```

### `templates`

Рекомендуется использовать `addTemplate` из `@nuxt/kit` вместо этой опции.

- **Тип**: `array`

**Пример**:
```ts
export default defineNuxtConfig({
  build: {
    templates: [
      {
        src: '~~/modules/support/plugin.js', // `src` can be absolute or relative
        dst: 'support.js', // `dst` is relative to project `.nuxt` dir
      },
    ],
  },
})
```

### `transpile`

Чтобы транспилировать отдельные зависимости через Babel, добавьте их сюда. Каждый элемент может быть именем пакета, функцией, строкой или RegExp, совпадающим с именем файла зависимости.

Можно передать функцию для условной транспиляции. Ей передаётся объект `{ isDev, isServer, isClient, isModern, isLegacy }`.

- **Тип**: `array`

**Пример**:
```ts
export default defineNuxtConfig({
  build: {
    transpile: [({ isLegacy }) => isLegacy && 'ky'],
  },
})
```

## buildDir

Каталог, куда помещаются собранные файлы Nuxt.

Многие инструменты считают `.nuxt` скрытым каталогом (из‑за точки в начале). Если это мешает, используйте эту опцию.

- **Тип**: `string`
- **По умолчанию:** `"/<rootDir>/.nuxt"`

**Пример**:
```ts
export default defineNuxtConfig({
  buildDir: 'nuxt-build',
})
```

## buildId

Уникальный идентификатор сборки. Может включать хэш текущего состояния проекта.

- **Тип**: `string`
- **По умолчанию:** `"4a2e2d30-418f-41df-8e58-ed5df06de7fd"`

## builder

Сборщик для бандлинга клиентской части приложения на Vue.

Nuxt поддерживает несколько сборщиков. По умолчанию используется Vite; можно переключиться на webpack, Rspack или подключить свой сборщик.

- **Тип**: `'vite' | 'webpack' | 'rspack' | string | { bundle: (nuxt: Nuxt) => Promise<void> }`
- **По умолчанию:** `"@nuxt/vite-builder"`

**Поддерживаемые сборщики:**

```ts
export default defineNuxtConfig({
  // default - uses @nuxt/vite-builder
  // builder: 'vite',

  // uses @nuxt/webpack-builder
  // builder: 'webpack',

  // uses @nuxt/rspack-builder
  builder: 'rspack',
})
```

При использовании `webpack` или `rspack` убедитесь, что в проекте явно установлены `@nuxt/webpack-builder` или `@nuxt/rspack-builder`.

**Объект со своим сборщиком:**

Передайте объект с функцией `bundle`:

```ts
export default defineNuxtConfig({
  builder: {
    async bundle (nuxt) {
      const entry = await resolvePath(resolve(nuxt.options.appDir, 'entry'))

      // Build client and server bundles
      await buildClient(nuxt, entry)
      if (nuxt.options.ssr) {
        await buildServer(nuxt, entry)
      }

      // ... it's a bit more complicated than that, of course!
    },
  },
})
```

**Отдельный пакет со сборщиком:**

Пакет должен экспортировать функцию `bundle`. Имя пакета указывается в `nuxt.config.ts`:

```ts
export default defineNuxtConfig({
  builder: 'my-custom-builder',
})
```

Дата совместимости для приложения.

Используется для управления поведением пресетов в Nitro, Nuxt Image и других модулях, которые могут менять поведение без мажорного релиза.

## components

Настройка авто-регистрации компонентов Nuxt.

Компоненты из указанных каталогов доступны на страницах, в макетах и других компонентах без явного импорта.

- **Тип**: `object`
- **По умолчанию**
```json
{
  "dirs": [
    {
      "path": "~/components/global",
      "global": true
    },
    "~/components"
  ]
}
```

**См.**: [документация каталога `app/components/`](https://nuxt.com/docs/4.x/directory-structure/app/components)

## css

Глобальные CSS-файлы, модули и библиотеки (подключаются на каждой странице).

Nuxt определяет тип файла по расширению и подключает нужный препроцессор. Соответствующие лоадеры нужно установить самостоятельно.

- **Тип**: `array`

**Пример**:
```ts
export default defineNuxtConfig({
  css: [
  // Load a Node.js module directly (here it's a Sass file).
    'bulma',
    // CSS file in the project
    '~/assets/css/main.css',
    // SCSS file in the project
    '~/assets/css/main.scss',
  ],
})
```

- **Тип**: `boolean`
- **По умолчанию:** `false`

## hooks

Хуки — подписчики на события Nuxt; обычно используются в модулях, но доступны и в `nuxt.config`.

Внутри имена хуков с двоеточием (например, `build:done`).
Для удобства в `nuxt.config` их можно описать вложенным объектом (см. ниже).

**Пример**:
```ts
import fs from 'node:fs'
import path from 'node:path'

export default defineNuxtConfig({
  hooks: {
    build: {
      done (builder) {
        const extraFilePath = path.join(
          builder.nuxt.options.buildDir,
          'extra-file',
        )
        fs.writeFileSync(extraFilePath, 'Something extra')
      },
    },
  },
})
```

## ignore

- **Тип**: `array`
- **По умолчанию**
```json
[
  "**/*.stories.{js,cts,mts,ts,jsx,tsx}",
  "**/*.{spec,test}.{js,cts,mts,ts,jsx,tsx}",
  "**/*.d.{cts,mts,ts}",
  "**/*.d.vue.{cts,mts,ts}",
  "**/.{pnpm-store,vercel,netlify,output,git,cache,data}",
  "**/*.sock",
  ".nuxt/analyze",
  ".nuxt",
  "**/-*.*"
]
```

## modules

Модули расширяют ядро Nuxt и добавляют интеграции.

Элемент массива — строка (пакет или путь к файлу), кортеж `[модуль, опции]` или inline-функция модуля.

- **Тип**: `array`

```ts
    // Relative to your project rootDir

    // Providing options

    // Inline definition
    function () {},
  ],
})
```

## plugins

### `extractCSS`

Общее извлечение CSS.

Через [mini-css-extract-plugin](https://github.com/webpack/mini-css-extract-plugin) CSS выносится в отдельные файлы (часто по компоненту), что упрощает кэширование CSS и JS отдельно.

- **Тип**: `boolean`
- **По умолчанию:** `true`

**Пример**:
```ts
export default defineNuxtConfig({
  webpack: {
    extractCSS: true,
    // or
    extractCSS: {
      ignoreOrder: true,
    },
  },
})
```

Обходной путь для выноса всего CSS в один файл существует, но один общий файл не рекомендуется:
несколько CSS-файлов лучше для кэша и изоляции preload и позволяют подгружать только нужные ресурсы.

**Пример**:
```ts
export default defineNuxtConfig({
  webpack: {
    extractCSS: true,
    optimization: {
      splitChunks: {
        cacheGroups: {
          styles: {
            name: 'styles',
            test: /\.(css|vue)$/,
            chunks: 'all',
            enforce: true,
          },
        },
      },
    },
  },
})
```

### `filenames`

Имена файлов бандла.

Про манифесты см. [документацию webpack](https://webpack.js.org/guides/code-splitting/).

::callout
**Примечание**: в production без хэша в имени браузер может долго кэшировать ассет и не увидеть обновление с первого захода.
::

Пример заменяет «красивые» имена чанков на числовые id:

**Пример**:
```ts
export default defineNuxtConfig({

    },
  },
})
```
