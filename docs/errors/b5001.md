# B5001

В конфигурации Nuxt не задан `compatibilityDate`. Nuxt использует эту дату, чтобы выбрать набор поведения по умолчанию: проект остаётся стабильным при обновлениях Nuxt и Nitro и не подхватывает новые значения по умолчанию незаметно.

## Решение

Добавьте `compatibilityDate` в `nuxt.config`, указав сегодняшнюю дату — так вы явно соглашаетесь с текущими значениями по умолчанию:

```ts
export default defineNuxtConfig({
  // use the current date, e.g. the day you add this line
  compatibilityDate: 'YYYY-MM-DD',
})
```
