# E2004

Route middleware указан (обычно через `definePageMeta({ middleware: [...] })`), но middleware с таким именем не существует. Частые причины: опечатка в имени или файл middleware переименовали/удалили, не обновив ссылки.

## Решение

Убедитесь, что имя совпадает с файлом в `middleware/`. Имя берётся из имени файла: `middleware/auth.ts` подключается как `auth`.
