Файл `.nuxtrc` задаёт конфигурацию Nuxt в плоском формате. Основан на [`unjs/rc9`](https://github.com/unjs/rc9).

::tip
Для более сложной конфигурации используйте [`nuxt.config`](/docs/4.x/directory-structure/nuxt-config).
::

## Использование

```bash [.nuxtrc]
# Отключить SSR
ssr=false

# Конфигурация @nuxt/devtools
devtools.enabled=true

# Модули Nuxt
modules[]=@nuxt/image
modules[]=nuxt-security

# Настройки модулей (добавляются Nuxt автоматически)
setups.@nuxt/test-utils="3.23.0"
```

Если задано и там и там, значения из `nuxt.config` переопределяют значения из `.nuxtrc`.

::note
Nuxt автоматически добавляет секцию `setups` для отслеживания установки и обновления модулей. Она используется [хуками жизненного цикла модулей](/docs/4.x/api/kit/modules#using-lifecycle-hooks-for-module-installation-and-upgrade), вручную её менять не нужно.
::

::read-more{to="/docs/4.x/api/configuration/nuxt-config"}
Все доступные опции — в документации **конфигурации Nuxt**.
::

## Глобальный .nuxtrc

Можно создать глобальный `.nuxtrc` в домашней директории — он применится ко всем проектам Nuxt на системе.

- macOS/Linux:

  ```md
  ~/.nuxtrc
  ```

- Windows:

  ```md
  C:\Users\{username}\.nuxtrc
  ```

Настройки из глобального файла переопределяются настройками в проекте, а `nuxt.config` имеет приоритет над обоими.
