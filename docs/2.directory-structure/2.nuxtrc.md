Файл `.nuxtrc` настраивает Nuxt плоским синтаксисом. Основан на [`unjs/rc9`](https://github.com/unjs/rc9).

::tip
Для сложной конфигурации используйте [`nuxt.config`](/docs/4.x/directory-structure/nuxt-config).
::

## Использование

```bash [.nuxtrc]
# Disable SSR
ssr=false

# Configuration for `@nuxt/devtools`
devtools.enabled=true

# Add Nuxt modules
modules[]=@nuxt/image
modules[]=nuxt-security

# Module setups (automatically added by Nuxt)
setups.@nuxt/test-utils="3.23.0"
```

Если есть `nuxt.config`, его свойства перезаписывают значения из `.nuxtrc`.

::note
Nuxt автоматически добавляет секцию `setups` для отслеживания установки и обновления модулей. Это внутреннее использование для [хуков жизненного цикла модулей](/docs/4.x/api/kit/modules#using-lifecycle-hooks-for-module-installation-and-upgrade); вручную не меняйте.
::

::read-more{to="/docs/4.x/api/configuration/nuxt-config"}
Все опции — в документации **конфигурации Nuxt**.
::

## Глобальный файл `.nuxtrc`

Можно создать глобальный `.nuxtrc` в домашнем каталоге для настроек по умолчанию.

- В macOS/Linux:

  ```md
  ~/.nuxtrc
  ```

- В Windows:

  ```md
  C:\Users\{username}\.nuxtrc
  ```

Глобальный `.nuxtrc` задаёт значения по умолчанию для всех проектов Nuxt на машине. Проектный `.nuxtrc` важнее, а `nuxt.config` важнее обоих.
