Каталог `layers/` помогает организовать и переиспользовать код, компоненты, composables и конфигурации. Слои в `layers/` в проекте регистрируются автоматически.

::note
Авторегистрация каталога `layers/` доступна в Nuxt v3.12.0+.
::

::tip{icon="i-lucide-lightbulb"}
Слои удобны для крупных кодовых баз с **Domain-Driven Design (DDD)**, переиспользуемых **UI-библиотек** и **тем**, общих **пресетов конфигурации** между проектами и разделения зон вроде **админок** или **функциональных модулей**.
::

## Структура

Каждый подкаталог в `layers/` — отдельный слой. Структура слоя такая же, как у обычного приложения Nuxt.

::important
У каждого слоя **обязан** быть файл `nuxt.config.ts`, даже пустой — иначе слой не считается валидным.
::

```bash [Directory structure]
-| layers/
---| base/
-----| nuxt.config.ts
-----| app/
-------| components/
---------| BaseButton.vue
-------| composables/
---------| useBase.ts
-----| server/
-------| api/
---------| hello.ts
---| admin/
-----| nuxt.config.ts
-----| app/
-------| pages/
---------| admin.vue
-------| layouts/
---------| admin.vue
```

## Автоматические алиасы

Для каждого слоя создаётся именованный алиас к его `srcDir`. Доступ через `#layers/[name]`:

```ts
// Access the base layer
import something from '#layers/base/path/to/file'

// Access the admin layer
import { useAdmin } from '#layers/admin/composables/useAdmin'
```

::note
Именованные алиасы слоёв появились в Nuxt v3.16.0.
::

## Содержимое слоя

В слое могут быть:

- [`nuxt.config.ts`](/docs/4.x/directory-structure/nuxt-config) — конфигурация слоя, сливается с основной
- [`app.config.ts`](/docs/4.x/directory-structure/app/app-config) — реактивная конфигурация приложения
- [`app/components/`](/docs/4.x/directory-structure/app/components) — Vue-компоненты (автоимпорт)
- [`app/composables/`](/docs/4.x/directory-structure/app/composables) — composables (автоимпорт)
- [`app/utils/`](/docs/4.x/directory-structure/app/utils) — утилиты (автоимпорт)
- [`app/pages/`](/docs/4.x/directory-structure/app/pages) — страницы
- [`app/layouts/`](/docs/4.x/directory-structure/app/layouts) — макеты
- [`app/middleware/`](/docs/4.x/directory-structure/app/middleware) — middleware маршрутов
- [`app/plugins/`](/docs/4.x/directory-structure/app/plugins) — плагины Nuxt
- [`server/`](/docs/4.x/directory-structure/server) — серверные маршруты, middleware и утилиты
- [`shared/`](/docs/4.x/directory-structure/shared) — общий код приложения и сервера

## Порядок приоритета

Если несколько слоёв задают один и тот же ресурс (компонент, composable, страницу и т.д.), побеждает слой с **большим приоритетом**. Слои сортируются по алфавиту: позже в алфавите — выше приоритет (Z > A).

Порядок задаётся префиксами в имени: `1.base/`, `2.features/`, `3.admin/`.
