## Использование

**Способ 1:** именованный экспорт

```ts [app/composables/useFoo.ts]
export const useFoo = () => {
  return useState('foo', () => 'bar')
}
```

**Способ 2:** экспорт по умолчанию

```ts [app/composables/use-foo.ts or composables/useFoo.ts]
// It will be available as useFoo() (camelCase of file name without extension)
export default function () {
  return useState('foo', () => 'bar')
}
```

**Использование:** автоимпортированный composable доступен в `.js`, `.ts` и `.vue`.

```vue [app/app.vue]
<script setup lang="ts">
const foo = useFoo()
</script>

<template>
  <div>
    {{ foo }}
  </div>
</template>
```

::note
Каталог `app/composables/` в Nuxt сам по себе не добавляет «магической» реактивности. Реактивность в composables строится на Composition API Vue (`ref`, `reactive` и т.д.). Реактивный код не ограничен только `app/composables/` — используйте его там, где нужно.
::

:read-more{to="/docs/4.x/guide/concepts/auto-imports"}

:link-example{to="/docs/4.x/examples/features/auto-imports"}

## Типы

Nuxt генерирует `.nuxt/imports.d.ts` с объявлениями типов.

Чтобы типы появились, выполните [`nuxt prepare`](/docs/4.x/api/commands/prepare), [`nuxt dev`](/docs/4.x/api/commands/dev) или [`nuxt build`](/docs/4.x/api/commands/build).

::note
Если создать composable без запущенного dev-сервера, TypeScript может выдать ошибку вроде `Cannot find name 'useBar'.`
::

## Примеры

### Вложенные composables

Один composable может вызывать другой через автоимпорт:

```ts [app/composables/test.ts]
export const useFoo = () => {
  const nuxtApp = useNuxtApp()
  const bar = useBar()
}
```

### Инъекции из плагинов

Из composables доступны [инъекции плагинов](/docs/4.x/directory-structure/app/plugins#providing-helpers):

```ts [app/composables/test.ts]
export const useHello = () => {
  const nuxtApp = useNuxtApp()
  return nuxtApp.$hello
}
```

## Как сканируются файлы

Сканируются только файлы **на верхнем уровне** [`app/composables/`](/docs/4.x/directory-structure/app/composables):

```bash [Directory Structure]
-| composables/
---| index.ts     // scanned
---| useFoo.ts    // scanned
---| nested/
-----| utils.ts   // not scanned
```

В автоимпорт попадут только `app/composables/index.ts` и `app/composables/useFoo.ts`.

Для вложенных модулей — реэкспорт (предпочтительно) или настройка сканера:

**Пример:** реэкспорт из `app/composables/index.ts`:

```ts [app/composables/index.ts]
// Enables auto import for this export
export { utils } from './nested/utils.ts'
```

**Пример:** сканирование вложенных каталогов:

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  imports: {
    dirs: [
      // Scan top-level composables
      '~/composables',
      // ... or scan composables nested one level deep with a specific name and file extension
      '~/composables/*/index.{ts,js,mjs,mts}',
      // ... or scan all composables within given directory
      '~/composables/**',
    ],
  },
})
```
