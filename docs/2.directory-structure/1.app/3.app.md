::tip
Если есть каталог `app/pages/`, файл `app.vue` необязателен. Nuxt подставит `app.vue` по умолчанию, но свой файл можно добавить для своей разметки и структуры.
::

## Использование

### Минимальный вариант

Каталог [`app/pages/`](/docs/4.x/directory-structure/app/pages) необязателен. Без него Nuxt не подключает зависимость [vue-router](https://router.vuejs.org) — удобно для лендинга или приложения без маршрутизации.

```vue [app/app.vue]
<template>
  <h1>Hello World!</h1>
</template>
```

:link-example{to="/docs/4.x/examples/hello-world"}

### Со страницами

При наличии [`app/pages/`](/docs/4.x/directory-structure/app/pages) для текущей страницы нужен компонент [`<NuxtPage>`](/docs/4.x/api/components/nuxt-page):

```vue [app/app.vue]
<template>
  <NuxtPage />
</template>
```

Общую оболочку (шапка, подвал) можно задать прямо в `app.vue`:

```vue [app/app.vue]
<template>
  <header>
    Header content
  </header>
  <NuxtPage />
  <footer>
    Footer content
  </footer>
</template>
```

::note
`app.vue` — корневой компонент: всё, что в нём (JS и CSS), глобально на всех страницах.
::

::read-more{to="/docs/4.x/directory-structure/app/pages"}
Как устроить страницы в `app/pages/`.
::

### С макетами

Разные макеты для разных страниц — каталог `app/layouts/` и компонент [`<NuxtLayout>`](/docs/4.x/api/components/nuxt-layout):

```vue [app/app.vue]
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>
```
