Файл `app/app.config.ts` задаёт реактивную конфигурацию внутри приложения: её можно менять в рантайме в хуках жизненного цикла, через плагин Nuxt и с HMR.

Расширение может быть `.ts`, `.js` или `.mjs`.

```ts twoslash [app/app.config.ts]
export default defineAppConfig({
  foo: 'bar',
})
```

::caution
Не кладите секреты в `app.config` — он попадает в клиентский бандл.
::

::note
При своём [`srcDir`](/docs/4.x/api/nuxt-config#srcdir) положите `app.config` в корень нового `srcDir`.
::

## Использование

Чтобы пробросить настройки и переменные окружения в приложение, опишите их в `app.config`:

```ts twoslash [app/app.config.ts]
export default defineAppConfig({
  theme: {
    primaryColor: '#ababab',
  },
})
```

К `theme` можно обращаться и при SSR, и в браузере через composable [`useAppConfig`](/docs/4.x/api/composables/use-app-config).

```vue [app/pages/index.vue]
<script setup lang="ts">
const appConfig = useAppConfig()

console.log(appConfig.theme)
</script>
```

Утилита [`updateAppConfig`](/docs/4.x/api/utils/update-app-config) обновляет `app.config` в рантайме.

```vue [app/pages/index.vue]
<script setup>
const appConfig = useAppConfig() // { foo: 'bar' }

const newAppConfig = { foo: 'baz' }

updateAppConfig(newAppConfig)

console.log(appConfig) // { foo: 'baz' }
</script>
```

::read-more{to="/docs/4.x/api/utils/update-app-config"}
Подробнее об утилите `updateAppConfig`.
::

## Типизация App Config

Nuxt старается сгенерировать интерфейс TypeScript из вашего app config.

Иногда типы задают вручную. Возможны два случая.

### Входные данные App Config

`AppConfigInput` подходит авторам модулей: какие опции допустимы при настройке app config. На тип `useAppConfig()` не влияет.

```ts [index.d.ts]
declare module 'nuxt/schema' {
  interface AppConfigInput {
    /** Theme configuration */
    theme?: {
      /** Primary app color */
      primaryColor?: string
    }
  }
}

// It is always important to ensure you import/export something when augmenting a type
export {}
```

### Результат App Config

Чтобы типизировать результат [`useAppConfig()`](/docs/4.x/api/composables/use-app-config), расширяют `AppConfig`.

::warning
Будьте осторожны: расширение `AppConfig` может полностью заменить выводимые Nuxt типы.
::

```ts [index.d.ts]
declare module 'nuxt/schema' {
  interface AppConfig {
    // This will entirely replace the existing inferred `theme` property
    theme: {
      // You might want to type this value to add more specific types than Nuxt can infer,
      // such as string literal types
      primaryColor?: 'red' | 'blue'
    }
  }
}

// It is always important to ensure you import/export something when augmenting a type
export {}
```

## Стратегия слияния

Для `AppConfig` между [слоями](/docs/4.x/getting-started/layers) Nuxt использует особое слияние.

Реализовано через [Function Merger](https://github.com/unjs/defu#function-merger): для каждого ключа в `app.config`, значение которого — массив, можно задать свою функцию слияния.

::note
Function merger доступен в расширяемых слоях, не в основном `app.config` проекта.
::

Пример:

::code-group

```ts twoslash [layer/app/app.config.ts]
export default defineAppConfig({
  // Default array value
  array: ['hello'],
})
```

```ts twoslash [app/app.config.ts]
export default defineAppConfig({
  // Overwrite default array value by using a merger function
  array: () => ['bonjour'],
})
```

::

## Известные ограничения

С Nuxt v3.3 файл `app.config.ts` разделяется с Nitro, отсюда ограничения:

1. Нельзя напрямую импортировать Vue-компоненты в `app.config.ts`.
2. В контексте Nitro часть автоимпортов недоступна.

Это из-за того, что Nitro обрабатывает app config без полной поддержки Vue.

Обход через Vite-плагины в конфиге Nitro возможен, но не рекомендуется:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  nitro: {
    vite: {
      plugins: [vue()],
    },
  },
})
```

::warning
Такой обход может давать непредсказуемое поведение. Многие плагины, в том числе Vue, в контексте Nitro недоступны.
::

Связанные issue:
- [Issue #19858](https://github.com/nuxt/nuxt/issues/19858)
- [Issue #19854](https://github.com/nuxt/nuxt/issues/19854)

::note
В Nitro v3 эти ограничения снимут за счёт отказа от app config.
Прогресс: [этот pull request](https://github.com/nitrojs/nitro/pull/2521).
::
