## Включение макетов

Макеты включаются, добавив [`<NuxtLayout>`](/docs/4.x/api/components/nuxt-layout) в [`app.vue`](/docs/4.x/directory-structure/app/app):

```vue [app/app.vue]
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>
```

Использование макета:
- свойство `layout` на странице через [definePageMeta](/docs/4.x/api/utils/define-page-meta)
- проп `name` у `<NuxtLayout>`
- свойство `appLayout` в route rules

::note
Имя макета приводится к kebab-case: `someLayout` → `some-layout`.
::

::note
Если макет не указан, используется `app/layouts/default.vue`.
::

::important
Если в приложении один макет, проще обойтись [`app.vue`](/docs/4.x/directory-structure/app/app).
::

::important
В отличие от других компонентов, у макета должен быть **один корневой элемент**, чтобы Nuxt мог применять переходы между макетами — и этим корнем не может быть `<slot />`.
::

## Макет по умолчанию

Файл `~/layouts/default.vue`:

```vue [app/layouts/default.vue]
<template>
  <div>
    <p>Some default layout content shared across all pages</p>
    <slot />
  </div>
</template>
```

В файле макета содержимое страницы рендерится в `<slot />`.

## Именованный макет

```bash [Directory Structure]
-| layouts/
---| default.vue
---| custom.vue
```

На странице укажите макет `custom`:

```vue twoslash [pages/about.vue]
<script setup lang="ts">
declare module 'nuxt/app' {
  interface NuxtLayouts {
    'custom': unknown
  }
}
// ---cut---
definePageMeta({
  layout: 'custom',
})
</script>
```

::read-more{to="/docs/4.x/directory-structure/app/pages#page-metadata"}
Подробнее о `definePageMeta`.
::

Макет по умолчанию для всех страниц можно переопределить пропом `name` у [`<NuxtLayout>`](/docs/4.x/api/components/nuxt-layout):

```vue [app/app.vue]
<script setup lang="ts">
// You might choose this based on an API call or logged-in status
const layout = 'custom'
</script>

<template>
  <NuxtLayout :name="layout">
    <NuxtPage />
  </NuxtLayout>
</template>
```

Во вложенных каталогах имя макета строится из пути и имени файла с удалением дублирующихся сегментов.

Файл | Имя макета
-- | --
`~/layouts/desktop/default.vue` | `desktop-default`
`~/layouts/desktop-base/base.vue` | `desktop-base`
`~/layouts/desktop/index.vue` | `desktop`

Для ясности имя файла лучше совпадало с именем макета:

Файл | Имя макета
-- | --
`~/layouts/desktop/DesktopDefault.vue` | `desktop-default`
`~/layouts/desktop-base/DesktopBase.vue` | `desktop-base`
`~/layouts/desktop/Desktop.vue` | `desktop`

## Смена макета динамически

Хелпер [`setPageLayout`](/docs/4.x/api/utils/set-page-layout):

```vue twoslash [app/pages/index.vue]
<script setup lang="ts">
declare module 'nuxt/app' {
  interface NuxtLayouts {
    'custom': unknown
  }
}
// ---cut---
function enableCustomLayout () {
  setPageLayout('custom')
}
definePageMeta({
  layout: false,
})
</script>

<template>
  <div>
    <button @click="enableCustomLayout">
      Update layout
    </button>
  </div>
</template>
```

Макеты для маршрутов можно задать через `appLayout` в route rules:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  routeRules: {
    // Set layout for specific route
    '/admin': { appLayout: 'admin' },
    // Set layout for multiple routes
    '/dashboard/**': { appLayout: 'dashboard' },
    // Disable layout for a route
    '/landing': { appLayout: false },
  },
})
```

::tip
Удобно, когда макеты задаются централизованно в конфиге, а не в каждой странице, или для маршрутов без отдельного компонента страницы (например catch-all).
::

## Свой макет на уровне страницы

При использовании страниц можно задать `layout: false` и вставить `<NuxtLayout>` внутри страницы.

::code-group

```vue [app/pages/index.vue]
<script setup lang="ts">
definePageMeta({
  layout: false,
})
</script>

<template>
  <div>
    <NuxtLayout name="custom">
      <template #header>
        Some header template content.
      </template>

      The rest of the page
    </NuxtLayout>
  </div>
</template>
```

```vue [app/layouts/custom.vue]
<template>
  <div>
    <header>
      <slot name="header">
        Default header content
      </slot>
    </header>
    <main>
      <slot />
    </main>
  </div>
</template>
```

::

::important
Если `<NuxtLayout>` внутри страницы, он не должен быть корневым элементом (или [отключите переходы макета/страницы](/docs/4.x/getting-started/transitions#disable-transitions)).
::
