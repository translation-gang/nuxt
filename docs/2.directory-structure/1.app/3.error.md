Во время работы приложения возможны непредвиденные ошибки. Файл `error.vue` заменяет стандартную страницу ошибки и показывает её аккуратно.

```vue [error.vue]
<script setup lang="ts">
import type { NuxtError } from '#app'

const props = defineProps<{ error: NuxtError }>()
</script>

<template>
  <div>
    <h1>{{ error.status }}</h1>
    <NuxtLink to="/">На главную</NuxtLink>
  </div>
</template>
```

::note
Несмотря на название «страница ошибки», это не маршрут: файл не кладут в `~/pages`, и `definePageMeta` здесь не используют. Макеты можно подключить через [`NuxtLayout`](/docs/4.x/api/components/nuxt-layout), указав имя макета.
::

У страницы ошибки один проп — `error` с данными об ошибке.

Объект `error` содержит поля:
```ts
interface NuxtError {
  status: number
  fatal: boolean
  unhandled: boolean
  statusText?: string
  data?: unknown
  cause?: unknown
  // legacy/deprecated equivalent of `status`
  statusCode: number
  // legacy/deprecated equivalent of `statusText`
  statusMessage?: string
}
```

Произвольные поля ошибки теряются; передавайте их в `data`:

```ts
throw createError({
  status: 404,
  statusText: 'Страница не найдена',
  data: {
    myCustomField: true,
  },
})
```
