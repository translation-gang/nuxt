Nuxt автоматически читает файлы в `app/plugins/` и подключает их при создании приложения Vue.

::note
Плагины из каталога регистрируются автоматически, в `nuxt.config` их отдельно добавлять не нужно.
::

::note
Суффиксы `.server` или `.client` в имени файла ограничивают плагин только сервером или только клиентом.
::

## Регистрация плагинов

Авторегистрация только для файлов на верхнем уровне каталога (или `index` в подкаталогах).

```bash [Directory structure]
-| plugins/
---| foo.ts      // scanned
---| bar/
-----| baz.ts    // not scanned
-----| foz.vue   // not scanned
-----| index.ts  // currently scanned but deprecated
```

Зарегистрируются только `foo.ts` и `bar/index.ts`.

Плагины из подкаталогов укажите в [`app/plugins`](/docs/4.x/api/nuxt-config#plugins-1) в `nuxt.config.ts`:

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  plugins: [
    '~/plugins/bar/baz',
    '~/plugins/bar/foz',
  ],
})
```

## Создание плагинов

Единственный аргумент плагина — [`nuxtApp`](/docs/4.x/api/composables/use-nuxt-app).

```ts twoslash [plugins/hello.ts]
export default defineNuxtPlugin((nuxtApp) => {
  // Doing something with nuxtApp
})
```

### Плагины в виде объекта

Для сложных сценариев плагин можно описать объектом:

```ts twoslash [plugins/hello.ts]
export default defineNuxtPlugin({
  name: 'my-plugin',
  enforce: 'pre', // or 'post'
  async setup (nuxtApp) {
    // this is the equivalent of a normal functional plugin
  },
  hooks: {
    // You can directly register Nuxt app runtime hooks here
    'app:created' () {
      const nuxtApp = useNuxtApp()
      // do something in the hook
    },
  },
  env: {
    // Set this value to `false` if you don't want the plugin to run when rendering server-only or island components.
    islands: true,
  },
})
```

:video-accordion{title="Видео Александра Лихтера об объектном синтаксисе плагинов Nuxt" videoId="2aXZyXB1QGQ"}

::note
При объектном синтаксисе свойства анализируются статически для более оптимальной сборки — не задавайте их в рантайме. :br
Например, `enforce: import.meta.server ? 'pre' : 'post'` ломает будущие оптимизации Nuxt.
С object-syntax Nuxt заранее подгружает слушатели хуков, порядок регистрации плагинов для хуков не важен.
::

## Порядок регистрации

Порядок задаётся префиксами с «алфавитной» нумерацией в имени файла.

```bash [Directory structure]
plugins/
 | - 01.myPlugin.ts
 | - 02.myOtherPlugin.ts
```

Тогда `02.myOtherPlugin.ts` выполнится после `01.myPlugin.ts` и увидит его инъекции.

Нужно, когда один плагин зависит от другого.

::note
Имена сортируются как строки: `10.myPlugin.ts` раньше `2.myOtherPlugin.ts`. Однозначные номера дополняют `0`.
::

## Стратегия загрузки

### Параллельные плагины

По умолчанию плагины выполняются по очереди. Флаг `parallel: true` — следующий плагин стартует, не дожидаясь завершения текущего.

```ts twoslash [plugins/my-plugin.ts]
export default defineNuxtPlugin({
  name: 'my-plugin',
  parallel: true,
  async setup (nuxtApp) {
    // the next plugin will be executed immediately
  },
})
```

### Зависимости между плагинами

Чтобы дождаться другого плагина, укажите его имя в `dependsOn`.

```ts twoslash [plugins/depending-on-my-plugin.ts]
export default defineNuxtPlugin({
  name: 'depends-on-my-plugin',
  dependsOn: ['my-plugin'],
  async setup (nuxtApp) {
    // this plugin will wait for the end of `my-plugin`'s execution before it runs
  },
})
```

## Использование composables

В плагинах Nuxt можно вызывать [composables](/docs/4.x/directory-structure/app/composables) и [utils](/docs/4.x/directory-structure/app/utils):

```ts [app/plugins/hello.ts]
export default defineNuxtPlugin((nuxtApp) => {
  const foo = useFoo()
})
```

Ограничения:

::important
**Composable, зависящий от плагина, зарегистрированного позже, может не сработать.** :br

Плагины вызываются по порядку и раньше остального. Composable может требовать плагин, который ещё не выполнился.
::

::important
**Composable, завязанный на жизненный цикл Vue, в плагине не сработает.** :br

Composables Vue привязаны к экземпляру компонента, плагины — к [`nuxtApp`](/docs/4.x/api/composables/use-nuxt-app).
::

## Предоставление хелперов

Чтобы добавить хелпер в [`NuxtApp`](/docs/4.x/api/composables/use-nuxt-app), верните из плагина объект с ключом `provide`.

::code-group
```ts twoslash [plugins/hello.ts]
export default defineNuxtPlugin(() => {
  return {
    provide: {
      hello: (msg: string) => `Hello ${msg}!`,
    },
  }
})
```
```ts twoslash [plugins/hello-object-syntax.ts]
export default defineNuxtPlugin({
  name: 'hello',
  setup () {
    return {
      provide: {
        hello: (msg: string) => `Hello ${msg}!`,
      },
    }
  },
})
```
::

В компонентах:

```vue [app/components/Hello.vue]
<script setup lang="ts">
// alternatively, you can also use it here
const { $hello } = useNuxtApp()
</script>

<template>
  <div>
    {{ $hello('world') }}
  </div>
</template>
```

::important
Лучше использовать [`composables`](/docs/4.x/directory-structure/app/composables), чем засорять глобальное пространство имён через `provide`.
::

::warning
**Если плагин отдаёт `ref` или `computed`, в `<template>` компонента они не разворачиваются автоматически.** :br
Так устроен Vue для ref не верхнего уровня шаблона. Подробнее [в документации Vue](https://vuejs.org/guide/essentials/reactivity-fundamentals#caveat-when-unwrapping-in-templates).
::

## Типизация плагинов

Хелперы из `provide` типизируются автоматически — в `useNuxtApp()` и в шаблонах.

::note
Чтобы вызвать хелпер _из другого плагина_, можно взять типизированный экземпляр через [`useNuxtApp()`](/docs/4.x/api/composables/use-nuxt-app), но так лучше не делать без уверенности в порядке плагинов.
::

Расширение типов вручную:

```ts [index.d.ts]
declare module '#app' {
  interface NuxtApp {
    $hello (msg: string): string
  }
}

declare module 'vue' {
  interface ComponentCustomProperties {
    $hello (msg: string): string
  }
}

export {}
```

## Плагины Vue

Плагины Vue, например [vue-gtag](https://github.com/MatteoGabriele/vue-gtag) для Google Analytics, подключаются через плагин Nuxt.

Сначала установите зависимость:

::code-group{sync="pm"}
```bash [npm]
npm install --save-dev vue-gtag-next
```
```bash [yarn]
yarn add --dev vue-gtag-next
```
```bash [pnpm]
pnpm add -D vue-gtag-next
```
```bash [bun]
bun add -D vue-gtag-next
```
```bash [deno]
deno add -D npm:vue-gtag-next
```
::

Then create a plugin file:

```ts [app/plugins/vue-gtag.client.ts]
import VueGtag, { trackRouter } from 'vue-gtag-next'

export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.vueApp.use(VueGtag, {
    property: {
      id: 'GA_MEASUREMENT_ID',
    },
  })
  trackRouter(useRouter())
})
```

## Директивы Vue

Пользовательские директивы Vue регистрируются в плагине так же.

```ts twoslash [plugins/my-directive.ts]
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.vueApp.directive('focus', {
    mounted (el) {
      el.focus()
    },
    getSSRProps (binding, vnode) {
      // you can provide SSR-specific props here
      return {}
    },
  })
})
```

::warning
Директиву нужно регистрировать и на клиенте, и на сервере, если она используется с обеих сторон. Если только на клиенте — `~/plugins/my-directive.client.ts` и заглушка в `~/plugins/my-directive.server.ts`.
::
