::note
Чтобы уменьшить бандл, этот каталог **необязателен**: [`vue-router`](https://router.vuejs.org) не подключится, если используется только [`app.vue`](/docs/4.x/directory-structure/app/app). Чтобы принудительно включить страницы, задайте `pages: true` в `nuxt.config` или добавьте [`router.options.ts`](/docs/4.x/guide/recipes/custom-routing#using-routeroptions).
::

## Использование

Страницы — это Vue-компоненты с любым [допустимым расширением](/docs/4.x/api/nuxt-config#extensions), которое поддерживает Nuxt (по умолчанию `.vue`, `.js`, `.jsx`, `.mjs`, `.ts` или `.tsx`).

Nuxt создаёт маршрут для каждой страницы в `~/pages/`.

::code-group

```vue [app/pages/index.vue]
<template>
  <h1>Index page</h1>
</template>
```

```ts twoslash [pages/index.ts]
// https://vuejs.org/guide/extras/render-function.html
export default defineComponent({
  render () {
    return h('h1', 'Index page')
  },
})
```

```tsx twoslash [pages/index.tsx]
// https://nuxt.com/docs/4.x/examples/advanced/jsx
// https://vuejs.org/guide/extras/render-function.html#jsx-tsx
export default defineComponent({
  render () {
    return <h1>Index page</h1>
  },
})
```

::

Файл `app/pages/index.vue` соответствует маршруту `/`.

Если используется [`app.vue`](/docs/4.x/directory-structure/app/app), для текущей страницы нужен [`<NuxtPage/>`](/docs/4.x/api/components/nuxt-page):

```vue [app/app.vue]
<template>
  <div>
    <!-- Markup shared across all pages, ex: NavBar -->
    <NuxtPage />
  </div>
</template>
```

У страницы **должен быть один корневой элемент**, иначе [переходы между маршрутами](/docs/4.x/getting-started/transitions) не сработают. HTML-комментарии тоже считаются узлами.

При SSR или статической генерации содержимое видно, но при клиентской навигации переход сломается и маршрут не отрендерится.

Примеры корректной и некорректной разметки:

::code-group

```vue [app/pages/working.vue]
<template>
  <div>
    <!-- This page correctly has only one single root element -->
    Page content
  </div>
</template>
```

```vue [app/pages/bad-1.vue]
<template>
  <!-- This page will not render when route changes during client side navigation, because of this comment -->
  <div>Page content</div>
</template>
```

```vue [app/pages/bad-2.vue]
<template>
  <div>This page</div>
  <div>Has more than one root element</div>
  <div>And will not render when route changes during client side navigation</div>
</template>
```

::

## Динамические маршруты

Содержимое в квадратных скобках становится [динамическим параметром](https://router.vuejs.org/guide/essentials/dynamic-matching) маршрута. Можно комбинировать несколько параметров и обычный текст в имени файла или каталога.

Чтобы параметр был _необязательным_, используйте двойные скобки — например `~/pages/[[slug]]/index.vue` или `~/pages/[[slug]].vue` совпадут и с `/`, и с `/test`.

```bash [Directory Structure]
-| pages/
---| index.vue
---| users-[group]/
-----| [id].vue
```

В примере выше `group` и `id` доступны в компоненте через объект `$route`:

```vue [app/pages/users-[group\\]/[id\\].vue]
<template>
  <p>{{ $route.params.group }} - {{ $route.params.id }}</p>
</template>
```

Переход на `/users-admins/123` даст:

```html
<p>admins - 123</p>
```

В Composition API для маршрута есть глобальная [`useRoute`](/docs/4.x/api/composables/use-route) — аналог `this.$route` в Options API.

```vue twoslash
<script setup lang="ts">
const route = useRoute()

if (route.params.group === 'admins' && !route.params.id) {
  console.log('Warning! Make sure user is authenticated!')
}
</script>
```

::note
Именованный родительский маршрут важнее вложенного динамического: для `/foo/hello` сработает `~/pages/foo.vue` раньше `~/pages/foo/[slug].vue`. :br Для `/foo` и `/foo/hello` с разными страницами используйте `~/pages/foo/index.vue` и `~/pages/foo/[slug].vue`.
::

## Catch-all маршрут

Catch-all создаётся файлом вида `[...slug].vue` — совпадёт со _всеми_ путями под этим префиксом.

```vue [app/pages/[...slug\\].vue]
<template>
  <p>{{ $route.params.slug }}</p>
</template>
```

Переход на `/hello/world` даст:

```html
<p>["hello", "world"]</p>
```

## Вложенные маршруты

[Вложенные маршруты](https://router.vuejs.org/guide/essentials/nested-routes) отображаются через `<NuxtPage>`.

Пример:

```bash [Directory Structure]
-| pages/
---| parent/
-----| child.vue
---| parent.vue
```

Дерево файлов даст такие маршруты:

```js
[
  {
    path: '/parent',
    component: '~/pages/parent.vue',
    name: 'parent',
    children: [
      {
        path: 'child',
        component: '~/pages/parent/child.vue',
        name: 'parent-child',
      },
    ],
  },
]
```

Чтобы показать `child.vue`, вставьте `<NuxtPage>` в `app/pages/parent.vue`:

```vue {}[pages/parent.vue]
<template>
  <div>
    <h1>I am the parent view</h1>
    <NuxtPage :foobar="123" />
  </div>
</template>
```

```vue {}[pages/parent/child.vue]
<script setup lang="ts">
const props = defineProps({
  foobar: String,
})

console.log(props.foobar)
</script>
```

### Ключи дочерних маршрутов

Чтобы управлять перерисовкой `<NuxtPage>` (например для переходов), передайте строку или функцию в проп `pageKey` или задайте `key` через `definePageMeta`:

```vue {}[pages/parent.vue]
<template>
  <div>
    <h1>I am the parent view</h1>
    <NuxtPage :page-key="route => route.fullPath" />
  </div>
</template>
```

Или так:

```vue twoslash {}[pages/parent/child.vue]
<script setup lang="ts">
definePageMeta({
  key: route => route.fullPath,
})
</script>
```

## Группы маршрутов

Иногда нужно сгруппировать страницы без влияния на URL. Тогда каталог оборачивают в скобки `(` и `)`.

Пример:

```bash [Directory structure]
-| pages/
---| index.vue
---| (marketing)/
-----| about.vue
-----| contact.vue
```

Получатся маршруты `/`, `/about` и `/contact`. Группа `marketing` не входит в URL.

### Доступ к группам маршрутов

Группы попадают в метаданные маршрута как `route.meta.groups`.
Так можно использовать группу в компонентах для условий, стилей и т.д.

```vue {}[pages/(marketing)/about.vue]
<script setup lang="ts">
const route = useRoute()

console.log(route.meta.groups) // Output: ['marketing']
</script>

<template>
  <div>
    <p v-if="route.meta.groups?.includes('marketing')">
      This is a marketing page
    </p>
  </div>
</template>
```

## Метаданные страницы

Метаданные маршрута задаются макросом `definePageMeta` в `<script>` и `<script setup>`:

```vue twoslash
<script setup lang="ts">
definePageMeta({
  title: 'My home page',
})
</script>
```

Данные доступны в приложении через `route.meta`.

```vue twoslash
<script setup lang="ts">
const route = useRoute()

console.log(route.meta.title) // My home page
</script>
```

При вложенных маршрутах метаданные со всех уровней сливаются в один объект. Подробнее — [документация vue-router](https://router.vuejs.org/guide/advanced/meta).

Как `defineEmits` и `defineProps` (см. [документацию Vue](https://vuejs.org/api/sfc-script-setup#defineprops-defineemits)), `definePageMeta` — **макрос компилятора**: он исчезает при сборке, на него нельзя ссылаться из компонента. Переданные метаданные выносятся из компонента.
Объект метаданных не может ссылаться на сам компонент, но может ссылаться на импорты и локальные **чистые функции**.

::warning
Не ссылайтесь на реактивные данные и функции с побочными эффектами — возможно непредсказуемое поведение.
::

```vue
<script setup lang="ts">
import { someData } from '~/utils/example'

function validateIdParam (route) {
  return route.params.id && !Number.isNaN(Number(route.params.id))
}

const title = ref('')

definePageMeta({
  validate: validateIdParam,
  someData,
  title, // do not do this, the ref will be hoisted out of the component
})
</script>
```

### Особые поля метаданных

Свои поля для приложения можно добавлять свободно. Ниже — поля `definePageMeta` со специальным значением в Nuxt:

#### `alias`

Псевдонимы страницы: один и тот же компонент с разных путей. Строка или массив строк — см. [vue-router](https://router.vuejs.org/guide/essentials/redirect-and-alias#Alias).

#### `keepalive`

При `keepalive: true` Nuxt оборачивает страницу в [Vue `<KeepAlive>`](https://vuejs.org/guide/built-ins/keep-alive#keepalive) — удобно для родительских маршрутов с динамическими дочерними, чтобы сохранять состояние.

Для родительских маршрутов: `<NuxtPage keepalive />`. Пропсы для `<KeepAlive>` — [список](https://vuejs.org/api/built-in-components#keepalive).

Значение по умолчанию — [в `nuxt.config`](/docs/4.x/api/nuxt-config#keepalive).

#### `key`

[См. выше](/docs/4.x/directory-structure/app/pages#child-route-keys).

#### `layout`

Макет для маршрута: `false` (без макета), строка или `ref`/`computed` для реактивности. [Подробнее о макетах](/docs/4.x/directory-structure/app/layouts).

#### `layoutTransition` и `pageTransition`

Параметры для `<transition>` вокруг страниц и макетов или `false`, чтобы отключить обёртку. [Опции](https://vuejs.org/api/built-in-components#transition), [как работают переходы](https://vuejs.org/guide/built-ins/transition#transition).

По умолчанию — [в `nuxt.config`](/docs/4.x/api/nuxt-config#layouttransition).

#### `middleware`

Middleware до загрузки страницы; сливается с middleware родительских/дочерних маршрутов. Строка, функция (по образцу [global before guard](https://router.vuejs.org/guide/advanced/navigation-guards#Global-Before-Guards)) или массив. [Именованный middleware](/docs/4.x/directory-structure/app/middleware).

#### `name`

Имя маршрута страницы.

#### `path`

Сопоставление пути, если файлового имени недостаточно. См. [vue-router](https://router.vuejs.org/guide/essentials/route-matching-syntax#Custom-regex-in-params).

#### `props`

`params` маршрута как пропсы страницы. См. [vue-router](https://router.vuejs.org/guide/essentials/passing-props).

### Типизация своих метаданных

Чтобы типизировать кастомные поля `definePageMeta`, расширьте принимаемый тип:

```ts [index.d.ts]
declare module '#app' {
  interface PageMeta {
    pageType?: string
  }
}

// It is always important to ensure you import/export something when augmenting a type
export {}
```

## Навигация

Для переходов между страницами используйте [`<NuxtLink>`](/docs/4.x/api/components/nuxt-link).

Компонент встроен в Nuxt, отдельно импортировать не нужно.

Ссылка на `index.vue` в `app/pages`:

```vue
<template>
  <NuxtLink to="/">Home page</NuxtLink>
</template>
```

::read-more{to="/docs/4.x/api/components/nuxt-link"}
Подробнее о `<NuxtLink>`.

## Программная навигация

Переходы можно выполнять через `navigateTo()` — например после отправки формы поиска.

::note
Всегда `await` для `navigateTo` или возвращайте его результат из функций.
::

```vue twoslash
<script setup lang="ts">
const name = ref('')
const type = ref(1)

function navigate () {
  return navigateTo({
    path: '/search',
    query: {
      name: name.value,
      type: type.value,
    },
  })
}
</script>
```

## Только клиент

Страница [только на клиенте](/docs/4.x/directory-structure/app/components#client-components) — суффикс `.client.vue`. Контент не рендерится на сервере.

## Только сервер

Страница [только на сервере](/docs/4.x/directory-structure/app/components#server-components) — суффикс `.server.vue`. Клиентская навигация через `vue-router` возможна, но рендер идёт серверным компонентом, код страницы не попадает в клиентский бандл.

::warning
У server-only страницы один корневой элемент (HTML-комментарии считаются узлами).
::

## Несколько каталогов страниц

По умолчанию все страницы в одном `app/pages` в корне проекта.

Группировка через [Nuxt Layers](/docs/4.x/getting-started/layers):

```bash [Directory Structure]
-| some-app/
---| nuxt.config.ts
---| pages/
-----| app-page.vue
-| nuxt.config.ts
```

```ts twoslash [some-app/nuxt.config.ts]
// some-app/nuxt.config.ts
export default defineNuxtConfig({
})
```

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['./some-app'],
})
```
