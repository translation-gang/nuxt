Главная цель [директории `app/utils/`](/docs/4.x/directory-structure/app/utils) — отделить Vue-композаблы от остальных автоимпортируемых утилит.

## Использование

**Способ 1:** именованный экспорт

```ts twoslash [utils/index.ts]
export const { format: formatNumber } = Intl.NumberFormat('en-GB', {
  notation: 'compact',
  maximumFractionDigits: 1,
})
```

**Способ 2:** экспорт по умолчанию

```ts twoslash [utils/random-entry.ts or utils/randomEntry.ts]
// Будет доступна как randomEntry() (camelCase от имени файла без расширения)
export default function (arr: Array<any>) {
  return arr[Math.floor(Math.random() * arr.length)]
}
```

Утилиты можно использовать в файлах `.js`, `.ts` и `.vue` без явного импорта:

```vue [app/app.vue]
<template>
  <p>{{ formatNumber(1234) }}</p>
</template>
```

:link-example{to="/docs/4.x/examples/features/auto-imports"}

::tip
Сканирование и автоимпорт в `app/utils/` устроены так же, как в [директории `app/composables/`](/docs/4.x/directory-structure/app/composables).
::

::important
Эти утилиты доступны только во Vue-части приложения. В [директории `server/`](/docs/4.x/directory-structure/server#server-utilities) автоимпортируются только `server/utils`.
::
