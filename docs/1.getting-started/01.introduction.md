Nuxt — бесплатный [open-source фреймворк](https://github.com/nuxt/nuxt) с интуитивным и расширяемым способом создавать типобезопасные, производительные и готовые к продакшену full-stack веб-приложения и сайты на [Vue.js](https://vuejs.org).

Мы сделали всё так, чтобы вы могли сразу писать `.vue` файлы, пользоваться горячей заменой модулей (HMR) в разработке и получать производительное приложение в продакшене с серверным рендерингом по умолчанию.

У Nuxt нет привязки к одному вендору: приложение можно развернуть [**где угодно, в том числе на edge**](/blog/nuxt-on-the-edge).

::tip
Если хотите поиграться с Nuxt в браузере, воспользуйтесь [одной из онлайн-песочниц](/docs/4.x/getting-started/installation#play-online).
::

## Автоматизация и соглашения

Nuxt опирается на соглашения и продуманную структуру каталогов, чтобы автоматизировать рутину и позволить разработчикам сосредоточиться на фичах. Поведение по умолчанию по-прежнему можно настроить и переопределить через конфигурационный файл.

- **Маршрутизация по файлам:** маршруты задаются структурой каталога [`app/pages/`](/docs/4.x/directory-structure/app/pages). Так проще организовать приложение и не писать маршруты вручную.
- **Разбиение кода:** Nuxt автоматически делит код на чанки, что снижает объём JavaScript при первой загрузке.
- **SSR из коробки:** встроенный серверный рендеринг — отдельный сервер настраивать не нужно.
- **Автоимпорты:** пишите Vue-композаблы и компоненты в своих каталогах и используйте их без ручных импортов — с tree-shaking и оптимизированными бандлами.
- **Утилиты для данных:** композаблы для SSR-совместимой загрузки данных и разных стратегий.
- **TypeScript без настройки:** типобезопасный код без глубокого изучения TypeScript — с авто-сгенерированными типами и `tsconfig.json`.
- **Сборка по умолчанию:** по умолчанию [Vite](https://vite.dev) для HMR в разработке и сборки в продакшен с встроенными лучшими практиками.

Nuxt берёт это на себя и даёт и фронтенд, и бэкенд, чтобы вы могли сфокусироваться на главном: **на самом веб-приложении**.

## Серверный рендеринг (SSR)

Nuxt по умолчанию включает SSR без ручной настройки сервера. Это даёт приложению ряд преимуществ:

- **Быстрее первая отрисовка:** в браузер уходит уже готовый HTML, его можно показать сразу. Лучше воспринимаемая скорость и UX, особенно на медленных сетях и устройствах.
- **Лучше SEO:** поисковикам проще индексировать SSR-страницы, так как контент есть в HTML сразу, а не только после выполнения JavaScript на клиенте.
- **Лучше на слабых устройствах:** меньше JS нужно скачать и выполнить на клиенте — это важно для слабого железа.
- **Доступность:** контент доступен с первой загрузки — удобнее для скринридеров и других вспомогательных технологий.
- **Проще кэширование:** страницы можно кэшировать на сервере и ещё ускорить отдачу.

В целом SSR даёт более быстрый и предсказуемый UX, помогает SEO и доступности.

Как универсальный фреймворк, Nuxt позволяет статически сгенерировать всё приложение для статического хостинга командой `nuxt generate`, отключить SSR глобально опцией `ssr: false` или использовать гибридный рендеринг через `routeRules`.

### Серверный движок

Серверный движок Nuxt — [Nitro](https://nitro.build/) — открывает новые full-stack возможности.

В разработке он использует Rollup и worker'ы Node.js для серверного кода и изоляции контекста. По файлам в `server/api/` строится API, из `server/middleware/` — серверный middleware.

В продакшене Nitro собирает приложение и сервер в единый каталог `.output`. Выход лёгкий: минифицированный, без лишних модулей Node.js (кроме полифиллов). Его можно развернуть на любой платформе с JavaScript: Node.js, serverless, Workers, edge или чисто статически.

### Архитектура

Nuxt состоит из нескольких [основных пакетов](https://github.com/nuxt/nuxt/tree/main/packages):

- Ядро: [nuxt](https://github.com/nuxt/nuxt/tree/main/packages/nuxt)
- Сборщики: [@nuxt/vite-builder](https://github.com/nuxt/nuxt/tree/main/packages/vite), [@nuxt/rspack-builder](https://github.com/nuxt/nuxt/tree/main/packages/rspack) и [@nuxt/webpack-builder](https://github.com/nuxt/nuxt/tree/main/packages/webpack)
- CLI: [@nuxt/cli](https://github.com/nuxt/cli)
- Серверный движок: [nitro](https://github.com/nitrojs/nitro)
- DevKit: [@nuxt/kit](https://github.com/nuxt/nuxt/tree/main/packages/kit)

Имеет смысл прочитать соответствующие разделы документации, чтобы понять возможности Nuxt и зону ответственности каждого пакета.
