## Node.js-сервер

Пресет Node.js в Nitro — деплой на любой Node-хостинг.

- **Формат по умолчанию**, если пресет не задан и не определён автоматически <br>
- Подгружаются только нужные чанки для запроса — быстрый cold start <br>
- Подходит для типичного Node-хостинга

### Точка входа

При `nuxt build` с пресетом Node получается точка входа готового сервера.

```bash [Terminal]
node .output/server/index.mjs
```

По умолчанию слушает порт 3000.

Переменные окружения:

- `NITRO_PORT` или `PORT` (по умолчанию `3000`)
- `NITRO_HOST` или `HOST` (по умолчанию `'0.0.0.0'`)
- `NITRO_SSL_CERT` и `NITRO_SSL_KEY` — если заданы оба, сервер в режиме HTTPS (обычно только для тестов; в проде TLS лучше на reverse proxy вроде nginx или Cloudflare)

### PM2

[PM2](https://pm2.keymetrics.io/) — простой вариант для сервера или VM.

Пример `ecosystem.config.cjs`:

```ts [ecosystem.config.cjs]
module.exports = {
  apps: [
    {
      name: 'NuxtAppName',
      port: '3000',
      exec_mode: 'cluster',
      instances: 'max',
      script: './.output/server/index.mjs',
    },
  ],
}
```

### Cluster mode

`NITRO_PRESET=node_cluster` — несколько процессов через модуль [cluster](https://nodejs.org/dist/latest/docs/api/cluster.html) Node.js.

По умолчанию распределение round robin.

## Статический хостинг

Два подхода к static hosting:

- SSG при `ssr: true`: маршруты пререндерятся при сборке (типично для `nuxt generate`). Генерируются fallback `/200.html` и `/404.html` для SPA на клиенте (на хосте может понадобиться настройка).
- Либо пререндер при `ssr: false` (чистый SPA): HTML с пустым `<div id="__nuxt"></div>`. SEO хуже, чем у полноценного пререндера; чаще лучше `ssr: true` и [`<ClientOnly>`](/docs/4.x/api/components/client-only) для частей без SSR.

:read-more{title="Пререндеринг в Nuxt" to="/docs/4.x/getting-started/prerendering"}

### Только клиентский рендер

Без пререндера маршрутов: `ssr: false` в `nuxt.config`. Тогда `nuxt generate` даёт `.output/public/index.html` и бандлы как у классического клиентского Vue.

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  ssr: false,
})
```

## Пресеты

Кроме Node и статики есть проверенные пресеты Nitro с небольшой конфигурацией.

Явно в [`nuxt.config.ts`](/docs/4.x/directory-structure/nuxt-config):

```ts twoslash [nuxt.config.ts]
// @errors: 2353
export default defineNuxtConfig({
  nitro: {
    preset: 'node-server',
  },
})
```

…или переменная `NITRO_PRESET` при `nuxt build`:

```bash [Terminal]
NITRO_PRESET=node-server nuxt build
```

🔎 Все пресеты и провайдеры — [деплой Nitro](https://nitro.build/deploy).

## CDN и прокси

Nuxt обычно сосуществует со сторонним контентом, но опции вроде минификации и «безопасности» Cloudflare иногда ломают гидратацию.

В Cloudflare лучше отключить:

1. Speed > Settings > Content Optimization — отключить «Rocket Loader™»
2. Security > Settings — отключить «Email Address Obfuscation»

Так Cloudflare не внедрит скрипты, которые дают лишние перерисовки или ошибки гидратации.

::tip
Разделы дашборда Cloudflare иногда переезжают — при необходимости поищите аналоги.
::
