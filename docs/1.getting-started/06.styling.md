В Nuxt гибко настраивается стилизация: свои стили или локальные/внешние таблицы стилей.
Доступны препроцессоры CSS, CSS-фреймворки, UI-библиотеки и модули Nuxt.

## Локальные таблицы стилей

Логичное место для локальных стилей — [`app/assets/`](/docs/4.x/directory-structure/app/assets).

### Импорт в компонентах

Стили можно подключать в страницах, layouts и компонентах.
Используйте статический `import` в JavaScript или CSS [`@import`](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/At-rules/@import).

```vue [app/pages/index.vue]
<script>
// Use a static import for server-side compatibility
import '~/assets/css/first.css'

// Caution: Dynamic imports are not server-side compatible
import('~/assets/css/first.css')
</script>

<style>
@import url("~/assets/css/second.css");
</style>
```

::tip
Таблицы стилей попадут в HTML, который отдаёт Nuxt.
::

### Свойство `css`

В конфигурации Nuxt есть свойство `css`.
Стили обычно лежат в [`app/assets/`](/docs/4.x/directory-structure/app/assets); укажите путь — Nuxt подключит их ко всем страницам.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  css: ['~/assets/css/main.css'],
})
```

::tip
Стили встраиваются в HTML от Nuxt, подключаются глобально и действуют на всех страницах.
::

### Шрифты

Локальные шрифты кладите в `public/`, например `public/fonts`, и подключайте в CSS через `url()`.

```css [assets/css/main.css]
@font-face {
  font-family: 'FarAwayGalaxy';
  src: url('/fonts/FarAwayGalaxy.woff') format('woff');
  font-weight: normal;
  font-style: normal;
  font-display: swap;
}
```

Далее используйте имя шрифта в стилях, страницах или компонентах:

```vue
<style>
h1 {
  font-family: 'FarAwayGalaxy', sans-serif;
}
</style>
```

### Стили из npm

Можно подключать пакеты со стилями из npm. Пример — `animate.css`.

::code-group{sync="pm"}

```bash [npm]
npm install animate.css
```

```bash [yarn]
yarn add animate.css
```

```bash [pnpm]
pnpm install animate.css
```

```bash [bun]
bun install animate.css
```

```bash [deno]
deno install npm:animate.css
```

::

Подключение в страницах, layouts и компонентах:

```vue [app/app.vue]
<script>
import 'animate.css'
</script>

<style>
@import url("animate.css");
</style>
```

Пакет можно указать строкой в свойстве `css` в конфигурации Nuxt.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  css: ['animate.css'],
})
```

## Внешние таблицы стилей

Внешние CSS подключают через `<link>` в `<head>` — в том числе из `nuxt.config`. Локальные стили тоже можно так добавить.

Через [`app.head`](/docs/4.x/api/nuxt-config#head):

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  app: {
    head: {
      link: [{ rel: 'stylesheet', href: 'https://cdnjs.cloudflare.com/ajax/libs/animate.css/4.1.1/animate.min.css' }],
    },
  },
})
```

### Динамическое подключение стилей

Композабл `useHead` позволяет задавать ссылки на стили в коде.

:read-more{to="/docs/4.x/api/composables/use-head"}

```ts twoslash
useHead({
  link: [{ rel: 'stylesheet', href: 'https://cdnjs.cloudflare.com/ajax/libs/animate.css/4.1.1/animate.min.css' }],
})
```

Под капотом используется `unhead` — полная документация: [unhead.unjs.io](https://unhead.unjs.io).

### Изменение отрендеренного `<head>` через плагин Nitro

Для тонкого контроля можно перехватить HTML хуком и править `<head>` программно.

Создайте плагин `~~/server/plugins/my-plugin.ts`:

<!-- TODO: figure out how to use twoslash to inject types for a different context -->

```ts [server/plugins/my-plugin.ts]
export default defineNitroPlugin((nitro) => {
  nitro.hooks.hook('render:html', (html) => {
    html.head.push('<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/animate.css/4.1.1/animate.min.css">')
  })
})
```

Внешние стили блокируют рендер: пока они не загружены и не обработаны, страница не рисуется. Слишком тяжёлые стили замедляют отрисовку. Подробнее — на [web.dev](https://web.dev/articles/defer-non-critical-css).

## Препроцессоры

Для SCSS, Sass, Less или Stylus сначала установите пакет.

::code-group

```bash [Sass & SCSS]
npm install -D sass
```

```bash [Less]
npm install -D less
```

```bash [Stylus]
npm install -D stylus
```

::

Исходники удобно держать в `app/assets` и импортировать в `app.vue` (или layouts) синтаксисом препроцессора.

```vue [app/pages/app.vue]
<style lang="scss">
@use "~/assets/scss/main.scss";
</style>
```

Либо через свойство `css` в `nuxt.config`.

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  css: ['~/assets/scss/main.scss'],
})
```

::tip
В обоих случаях скомпилированные стили встраиваются в HTML от Nuxt.
::

Чтобы подмешивать код в препроцессированные файлы (например [Sass partial](https://sass-lang.com/documentation/at-rules/use/#partials) с переменными цвета), используйте [опции препроцессоров](https://vite.dev/config/shared-options#css-preprocessoroptions) в Vite.

Частичные файлы в `app/assets`:

::code-group{sync="preprocessor"}

```scss [assets/_colors.scss]
$primary: #49240F;
$secondary: #E4A79D;
```

```sass [assets/_colors.sass]
$primary: #49240F
$secondary: #E4A79D
```

::

В `nuxt.config`:

::code-group

```ts twoslash [SCSS]
export default defineNuxtConfig({
  vite: {
    css: {
      preprocessorOptions: {
        scss: {
          additionalData: '@use "~/assets/_colors.scss" as *;',
        },
      },
    },
  },
})
```

```ts twoslash [SASS]
export default defineNuxtConfig({
  vite: {
    css: {
      preprocessorOptions: {
        sass: {
          additionalData: '@use "~/assets/_colors.sass" as *\n',
        },
      },
    },
  },
})
```

::

По умолчанию сборщик — Vite. Для webpack смотрите документацию [sass-loader](https://webpack.js.org/loaders/sass-loader/).

### Preprocessor Workers (экспериментально)

В Vite есть [экспериментальная опция](https://vite.dev/config/shared-options#css-preprocessormaxworkers), ускоряющая препроцессоры.

Включение в `nuxt.config`:

```ts
export default defineNuxtConfig({
  vite: {
    css: {
      preprocessorMaxWorkers: true, // number of CPUs minus 1
    },
  },
})
```

::note
Опция экспериментальная; смотрите документацию Vite и [оставляйте отзывы](https://github.com/vitejs/vite/discussions/15835).
::

## Стили в Single File Components (SFC)

Сильная сторона Vue и SFC — естественная работа со стилями: CSS или препроцессор прямо в `<style>` компонента, без обязательного CSS-in-JS. Если нужен CSS-in-JS, есть сторонние библиотеки и модули, например [pinceau](https://github.com/Tahul/pinceau).

Полный справочник — в [документации Vue](https://vuejs.org/api/sfc-css-features).

### Привязка class и style

Используйте возможности SFC для `class` и `style`.

::code-group

```vue [Ref and Reactive]
<script setup lang="ts">
const isActive = ref(true)
const hasError = ref(false)
const classObject = reactive({
  'active': true,
  'text-danger': false,
})
</script>

<template>
  <div
    class="static"
    :class="{ 'active': isActive, 'text-danger': hasError }"
  />
  <div :class="classObject" />
</template>
```

```vue [Computed]
<script setup lang="ts">
const isActive = ref(true)
const error = ref(null)

const classObject = computed(() => ({
  'active': isActive.value && !error.value,
  'text-danger': error.value && error.value.type === 'fatal',
}))
</script>

<template>
  <div :class="classObject" />
</template>
```

```vue [Array]
<script setup lang="ts">
const isActive = ref(true)
const errorClass = ref('text-danger')
</script>

<template>
  <div :class="[{ active: isActive }, errorClass]" />
</template>
```

```vue [Style]
<script setup lang="ts">
const activeColor = ref('red')
const fontSize = ref(30)
const styleObject = reactive({ color: 'red', fontSize: '13px' })
</script>

<template>
  <div :style="{ color: activeColor, fontSize: fontSize + 'px' }" />
  <div :style="[baseStyles, overridingStyles]" />
  <div :style="styleObject" />
</template>
```

::

Подробнее — в [Vue docs](https://vuejs.org/guide/essentials/class-and-style).

### Динамические стили с `v-bind`

В блоке `<style>` можно ссылаться на переменные и выражения через `v-bind`.
При изменении значения стиль обновится.

```vue
<script setup lang="ts">
const color = ref('red')
</script>

<template>
  <div class="text">
    hello
  </div>
</template>

<style>
.text {
  color: v-bind(color);
}
</style>
```

### Scoped-стили

Атрибут `scoped` ограничивает стили текущим компонентом.

```vue
<template>
  <div class="example">
    hi
  </div>
</template>

<style scoped>
.example {
  color: red;
}
</style>
```

### CSS Modules

[CSS Modules](https://github.com/css-modules/css-modules) подключаются атрибутом `module`; доступ через `$style`.

```vue
<template>
  <p :class="$style.red">
    This should be red
  </p>
</template>

<style module>
.red {
  color: red;
}
</style>
```

### Поддержка препроцессоров

В `<style>` SFC можно писать на синтаксисе препроцессора. Vite из коробки понимает `.scss`, `.sass`, `.less`, `.styl` и `.stylus` после установки пакетов — в SFC укажите `lang`.

::code-group

```vue [SCSS]
<style lang="scss">
  /* Write scss here */
</style>
```

```vue [Sass]
<style lang="sass">
  /* Write sass here */
</style>
```

```vue [LESS]
<style lang="less">
  /* Write less here */
</style>
```

```vue [Stylus]
<style lang="stylus">
  /* Write stylus here */
</style>
```

::

См. [документацию Vite по CSS](https://vite.dev/guide/features#css) и [@vitejs/plugin-vue](https://github.com/vitejs/vite-plugin-vue/tree/main/packages/plugin-vue).
Для webpack — [vue-loader](https://vue-loader.vuejs.org).

## PostCSS

В Nuxt встроен PostCSS; настройка в `nuxt.config`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  postcss: {
    plugins: {
      'postcss-nested': {},
      'postcss-custom-media': {},
    },
  },
})
```

Для подсветки в SFC используйте `lang="postcss"`.

```vue
<style lang="postcss">
  /* Write postcss here */
</style>
```

По умолчанию уже подключены:

- [postcss-import](https://github.com/postcss/postcss-import): улучшает `@import`
- [postcss-url](https://github.com/postcss/postcss-url): преобразует `url()`
- [autoprefixer](https://github.com/postcss/autoprefixer): вендорные префиксы
- [cssnano](https://cssnano.github.io/cssnano/): минификация и purge

## Разные стили через layouts

Если части приложения должны выглядеть совсем по-разному, используйте layouts и разные стили для разных layout.

```vue
<template>
  <div class="default-layout">
    <h1>Default Layout</h1>
    <slot />
  </div>
</template>

<style>
.default-layout {
  color: red;
}
</style>
```

## Сторонние библиотеки и модули

Nuxt не навязывает один способ стилизации: подойдут [UnoCSS](https://unocss.dev), [Tailwind CSS](https://tailwindcss.com) и другие инструменты.

Сообщество и команда Nuxt выпускают модули для упрощения интеграции.
Каталог — в [разделе модулей](/modules) сайта.
Несколько примеров:

- [UnoCSS](/modules/unocss): атомарный CSS по требованию
- [Tailwind CSS](/modules/tailwindcss): utility-first CSS
- [Fontaine](https://github.com/nuxt-modules/fontaine): метрики шрифтов и fallback
- [Pinceau](https://github.com/Tahul/pinceau): адаптируемая система стилей
- [Nuxt UI](https://ui.nuxt.com): UI-библиотека для современных веб-приложений
- [Panda CSS](https://panda-css.com/docs/installation/nuxt): CSS-in-JS с атомарным CSS на этапе сборки

Модули дают хороший DX из коробки, но отсутствие модуля не значит, что инструмент нельзя использовать — настройте вручную. Иногда нужен [плагин Nuxt](/docs/4.x/directory-structure/app/plugins) и/или [свой модуль](/docs/4.x/guide/modules). Получилось полезное — поделитесь с [сообществом](/modules).

### Оптимизации LCP для CSS

Чтобы ускорить загрузку глобальных CSS:

- CDN — файлы ближе к пользователям
- Сжатие, желательно Brotli
- Доставка по HTTP/2 или HTTP/3
- Тот же домен для ассетов (без лишнего поддомена)

На Cloudflare, Netlify или Vercel часть шагов уже сделана за вас.
Гайд по LCP — на [web.dev](https://web.dev/articles/optimize-lcp).

Если весь CSS инлайнится Nuxt, можно (экспериментально) убрать внешние ссылки на CSS из HTML — через хук в модуле или в `nuxt.config`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  hooks: {
    'build:manifest': (manifest) => {
      // find the app entry, css list
      const css = Object.values(manifest).find(options => options.isEntry)?.css
      if (css) {
        // start from the end of the array and go to the beginning
        for (let i = css.length - 1; i >= 0; i--) {
          // if it starts with 'entry', remove it from the list
          if (css[i].startsWith('entry')) {
            css.splice(i, 1)
          }
        }
      }
    },
  },
})
```
