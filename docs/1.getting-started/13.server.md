## На базе Nitro

![Server engine](/assets/docs/getting-started/server.svg)

Сервер Nuxt — это [Nitro](https://github.com/nitrojs/nitro): изначально для Nuxt, сейчас в [UnJS](https://unjs.io) и доступен другим фреймворкам, в том числе отдельно.

Nitro даёт Nuxt:

- Полный контроль над серверной частью
- Универсальный деплой у любого провайдера (много zero-config)
- Гибридный рендеринг

Внутри Nitro использует [h3](https://github.com/h3js/h3) — минимальный HTTP-фреймворк для производительности и портируемости.

## Эндпоинты и middleware

Серверную часть Nuxt — от API до middleware — задаёте в `server/`.

Эндпоинты и middleware объявляются так:

```ts twoslash [server/api/test.ts]
export default defineEventHandler(async (event) => {
  // ... Do whatever you want here
})
```

Можно возвращать `text`, `json`, `html` или `stream`.

Из коробки — **hot module replacement** и **auto-import**, как в остальной части приложения.

## Гибридный рендеринг

У Nitro есть `routeRules`: правила, как рендерить каждый маршрут Nuxt (и не только).

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  routeRules: {
    // Generated at build time for SEO purpose
    '/': { prerender: true },
    // Cached for 1 hour
    '/api/*': { cache: { maxAge: 60 * 60 } },
    // Redirection to avoid 404
    '/old-page': {
      redirect: { to: '/new-page', statusCode: 302 },
    },
    // ...
  },
})
```

::read-more{to="/docs/4.x/guide/concepts/rendering#hybrid-rendering"}
Все доступные route rules для режима рендеринга маршрутов.
::

Часть правил (`ssr`, `appMiddleware`, `noScripts`) — специфика Nuxt при выдаче HTML.

Некоторые правила (`appMiddleware`, `redirect`, `prerender`) влияют и на клиент.

Nitro собирает приложение для SSR и для пререндера.
