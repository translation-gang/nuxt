Для стилей, шрифтов и изображений Nuxt использует два каталога.

- Содержимое [`public/`](/docs/4.x/directory-structure/public) отдаётся с корня сервера как есть.
- Каталог [`app/assets/`](/docs/4.x/directory-structure/app/assets) по соглашению хранит ресурсы, которые должен обработать сборщик (Vite или webpack).

## Каталог public

[`public/`](/docs/4.x/directory-structure/public) — статика, доступная по фиксированному URL приложения.

Файл из [`public/`](/docs/4.x/directory-structure/public) можно ссылать из кода или открыть в браузере с корня `/`.

### Пример

Картинка в `public/img/` доступна по статическому URL `/img/nuxt.png`:

```vue [app/app.vue]
<template>
  <img
    src="/img/nuxt.png"
    alt="Discover Nuxt"
  >
</template>
```

## Каталог assets

Nuxt использует [Vite](https://vite.dev/guide/assets) (по умолчанию) или [webpack](https://webpack.js.org/guides/asset-management/) для сборки. Основная задача — JavaScript, но через [плагины](https://vite.dev/plugins/) (Vite) или [loaders](https://webpack.js.org/loaders/) (webpack) обрабатываются стили, шрифты, SVG и др. Файлы преобразуются ради производительности и кэширования (минификация CSS, инвалидация кэша браузера).

По соглашению для этого служит [`app/assets/`](/docs/4.x/directory-structure/app/assets), но автосканирования нет — можно назвать каталог иначе.

В коде файлы из [`app/assets/`](/docs/4.x/directory-structure/app/assets) указывают путём `~/assets/`.

### Пример

Изображение, которое обработает сборщик, если настроена обработка расширения:

```vue [app/app.vue]
<template>
  <img
    src="~/assets/img/nuxt.png"
    alt="Discover Nuxt"
  >
</template>
```

::note
Nuxt не отдаёт файлы из [`app/assets/`](/docs/4.x/directory-structure/app/assets) по статическому URL вроде `/assets/my-file.png`. Нужен статический URL — используйте [`public/`](/docs/4.x/getting-started/assets#public-directory).
::
