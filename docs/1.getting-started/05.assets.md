Nuxt использует две директории для ресурсов: стилей, шрифтов и изображений.

- Содержимое [`public/`](/docs/4.x/directory-structure/public) отдаётся с корня сервера как есть.
- В [`app/assets/`](/docs/4.x/directory-structure/app/assets) по соглашению лежат ресурсы, которые должен обрабатывать сборщик (Vite или webpack).

## Директория Public

Директория [`public/`](/docs/4.x/directory-structure/public) используется как публичный каталог статики, доступной по фиксированному URL приложения.

Файлы из [`public/`](/docs/4.x/directory-structure/public) доступны в коде приложения и в браузере по корневому URL `/`.

### Пример

Изображение в `public/img/` доступно по статическому URL `/img/nuxt.png`:

```vue [app/app.vue]
<template>
  <img
    src="/img/nuxt.png"
    alt="Discover Nuxt"
  >
</template>
```

## Директория Assets

Nuxt использует [Vite](https://vite.dev/guide/assets) (по умолчанию) или [webpack](https://webpack.js.org/guides/asset-management/) для сборки приложения. Эти инструменты в первую очередь обрабатывают JavaScript, но через [плагины](https://vite.dev/plugins/) (Vite) или [лоадеры](https://webpack.js.org/loaders/) (webpack) можно обрабатывать и другие ресурсы — стили, шрифты, SVG. На этом этапе файлы преобразуются (минификация, инвалидация кэша и т.п.).

По соглашению для таких файлов используется директория [`app/assets/`](/docs/4.x/directory-structure/app/assets); автоматического сканирования этой папки нет, имя можно изменить.

В коде приложения файлы из [`app/assets/`](/docs/4.x/directory-structure/app/assets) подключаются через путь `~/assets/`.

### Пример

Подключение изображения, которое будет обработано сборщиком, если для данного расширения настроена обработка:

```vue [app/app.vue]
<template>
  <img
    src="~/assets/img/nuxt.png"
    alt="Discover Nuxt"
  >
</template>
```

::note
Nuxt не раздаёт файлы из [`app/assets/`](/docs/4.x/directory-structure/app/assets) по статическому URL вроде `/assets/my-file.png`. Для статического URL используйте директорию [`public/`](/docs/4.x/getting-started/assets#public-directory).
::
