Для реактивного общего состояния с учётом SSR в Nuxt есть композабл [`useState`](/docs/4.x/api/composables/use-state).

[`useState`](/docs/4.x/api/composables/use-state) — SSR-friendly замена [`ref`](https://vuejs.org/api/reactivity-core#ref): значение сохраняется после SSR (при гидратации на клиенте) и разделяется между компонентами по уникальному ключу.

:video-accordion{title="Александр Лихтер: зачем и когда использовать useState" videoId="mv0WcBABcIk"}

::important
Данные из [`useState`](/docs/4.x/api/composables/use-state) сериализуются в JSON — не кладите туда классы, функции, символы и всё, что не сериализуется.
::

::read-more{to="/docs/4.x/api/composables/use-state"}
Подробнее о композабле `useState`.
::

## Рекомендации

::warning
Не объявляйте `const state = ref()` вне `<script setup>` или `setup()`.<br>
Например, `export myState = ref({})` даст общее состояние между запросами на сервере и может привести к утечкам памяти.
::

::tip{icon="i-lucide-circle-check"}
Лучше так: `const useX = () => useState('x')`
::

## Примеры

### Базовый пример

Локальный для приложения счётчик: любой компонент с `useState('counter')` видит одно и то же реактивное состояние.

```vue twoslash [app/app.vue]
<script setup lang="ts">
const counter = useState('counter', () => Math.round(Math.random() * 1000))
</script>

<template>
  <div>
    Counter: {{ counter }}
    <button @click="counter++">
      +
    </button>
    <button @click="counter--">
      -
    </button>
  </div>
</template>
```

:link-example{to="/docs/4.x/examples/features/state-management"}

::note
Глобальная инвалидация кэша состояния — утилита [`clearNuxtState`](/docs/4.x/api/utils/clear-nuxt-state).
::

### Инициализация

Часто состояние нужно заполнить асинхронно. Подойдёт [`app.vue`](/docs/4.x/directory-structure/app/app) и утилита [`callOnce`](/docs/4.x/api/utils/call-once).

```vue twoslash [app/app.vue]
<script setup lang="ts">
const websiteConfig = useState('config')

await callOnce(async () => {
  websiteConfig.value = await $fetch('https://my-cms.com/api/website-config')
})
</script>
```

::tip
Похоже на action [`nuxtServerInit`](https://v2.nuxt.com/docs/directory-structure/store/#the-nuxtserverinit-action) в Nuxt 2 — начальное состояние на сервере до рендера страницы.
::

:read-more{to="/docs/4.x/api/utils/call-once"}

### Использование с Pinia

Пример с [модулем Pinia](/modules/pinia): глобальный store по всему приложению.

::important
Установите модуль: `npx nuxt module add pinia` или шаги из [документации Pinia](https://pinia.vuejs.org/ssr/nuxt.html#Installation).
::

::code-group
```ts [app/stores/website.ts]
export const useWebsiteStore = defineStore('websiteStore', {
  state: () => ({
    name: '',
    description: '',
  }),
  actions: {
    async fetch () {
      const infos = await $fetch('https://api.nuxt.com/modules/pinia')

      this.name = infos.name
      this.description = infos.description
    },
  },
})
```
```vue [app/app.vue]
<script setup lang="ts">
const website = useWebsiteStore()

await callOnce(website.fetch)
</script>

<template>
  <main>
    <h1>{{ website.name }}</h1>
    <p>{{ website.description }}</p>
  </main>
</template>
```
::

## Продвинутые сценарии

::code-group
```ts [app/composables/locale.ts]
import type { Ref } from 'vue'

export const useLocale = () => {
  return useState<string>('locale', () => useDefaultLocale().value)
}

export const useDefaultLocale = (fallback = 'en-US') => {
  const locale = ref(fallback)
  if (import.meta.server) {
    const reqLocale = useRequestHeaders()['accept-language']?.split(',')[0]
    if (reqLocale) {
      locale.value = reqLocale
    }
  } else if (import.meta.client) {
    const navLang = navigator.language
    if (navLang) {
      locale.value = navLang
    }
  }
  return locale
}

export const useLocales = () => {
  const locale = useLocale()
  const locales = ref([
    'en-US',
    'en-GB',
    // ...,
    'ja-JP-u-ca-japanese',
  ])
  if (!locales.value.includes(locale.value)) {
    locales.value.unshift(locale.value)
  }
  return locales
}

export const useLocaleDate = (date: Ref<Date> | Date, locale = useLocale()) => {
  return computed(() => new Intl.DateTimeFormat(locale.value, { dateStyle: 'full' }).format(unref(date)))
}
```

```vue [app/app.vue]
<script setup lang="ts">
const locales = useLocales()
const locale = useLocale()
const date = useLocaleDate(new Date('2016-10-26'))
</script>

<template>
  <div>
    <h1>Nuxt birthday</h1>
    <p>{{ date }}</p>
    <label for="locale-chooser">Preview a different locale</label>
    <select
      id="locale-chooser"
      v-model="locale"
    >
      <option
        v-for="loc of locales"
        :key="loc"
        :value="loc"
      >
        {{ loc }}
      </option>
    </select>
  </div>
</template>
```
::

## Общее состояние

С [автоимпортируемыми композаблами](/docs/4.x/directory-structure/app/composables) можно описать типобезопасное глобальное состояние и подключать его везде.

```ts twoslash [composables/states.ts]
export const useColor = () => useState<string>('color', () => 'pink')
```

```vue [app/app.vue]
<script setup lang="ts">
// ---cut-start---
const useColor = () => useState<string>('color', () => 'pink')
// ---cut-end---
const color = useColor() // Same as useState('color')
</script>

<template>
  <p>Current color: {{ color }}</p>
</template>
```

## Сторонние библиотеки

Раньше Nuxt опирался на Vuex. При миграции с Nuxt 2 см. [руководство по миграции](/docs/4.x/migration/configuration#vuex).

Nuxt не навязывает один стек: выберите подходящее решение. Есть интеграции с популярными библиотеками:

- [Pinia](/modules/pinia) — рекомендация экосистемы Vue
- [Harlem](/modules/harlem) — иммутабельное глобальное состояние
- [XState](/modules/xstate) — state machines, визуализация и тесты логики состояния
