## `app.vue`

![Файл app.vue — точка входа приложения](/assets/docs/getting-started/views/app.svg)

По умолчанию Nuxt считает этот файл **точкой входа** и рендерит его содержимое для каждого маршрута приложения.

```vue [app/app.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
  </div>
</template>
```

::tip
Если вы знакомы с Vue, возможно, ищете `main.js` (файл, который обычно создаёт приложение Vue). Nuxt делает это под капотом.
::

## Компоненты

![Компоненты — переиспользуемые элементы интерфейса](/assets/docs/getting-started/views/components.svg)

Компоненты — это переиспользуемые части интерфейса (кнопки, меню и т.п.). В Nuxt их можно создавать в директории [`app/components/`](/docs/4.x/directory-structure/app/components): они автоматически станут доступны по всему приложению без явного импорта.

::code-group

```vue [app/app.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
    <AppAlert>
      This is an auto-imported component.
    </AppAlert>
  </div>
</template>
```

```vue [app/components/AppAlert.vue]
<template>
  <span>
    <slot />
  </span>
</template>
```

::

## Страницы (Pages)

![Страницы — представления для конкретного маршрута](/assets/docs/getting-started/views/pages.svg)

Страницы — это представления для каждого шаблона маршрута. Каждый файл в [`app/pages/`](/docs/4.x/directory-structure/app/pages) соответствует своему маршруту и отображает его содержимое.

Чтобы использовать страницы, создайте `app/pages/index.vue` и добавьте компонент `<NuxtPage />` в [`app/app.vue`](/docs/4.x/directory-structure/app/app) (или удалите `app/app.vue` для стандартной точки входа). Новые файлы в [`app/pages/`](/docs/4.x/directory-structure/app/pages) создают новые страницы и маршруты.

::code-group

```vue [app/pages/index.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
    <AppAlert>
      This is an auto-imported component
    </AppAlert>
  </div>
</template>
```

```vue [app/pages/about.vue]
<template>
  <section>
    <p>This page will be displayed at the /about route.</p>
  </section>
</template>
```

::

:read-more{title="Раздел о маршрутизации" to="/docs/4.x/getting-started/routing"}

## Макеты (Layouts)

![Макеты — обёртки вокруг страниц](/assets/docs/getting-started/views/layouts.svg)

Макеты — это обёртки вокруг страниц с общим интерфейсом (шапка, подвал и т.д.). Это Vue-файлы с компонентом `<slot />` для вывода содержимого **страницы**. По умолчанию используется `app/layouts/default.vue`. Свой макет можно задать в метаданных страницы.

::note
Если в приложении один макет, лучше использовать [`app/app.vue`](/docs/4.x/directory-structure/app/app) с [`<NuxtPage />`](/docs/4.x/api/components/nuxt-page) вместо отдельного layout.
::

::code-group

```vue [app/app.vue]
<template>
  <div>
    <NuxtLayout>
      <NuxtPage />
    </NuxtLayout>
  </div>
</template>
```

```vue [app/layouts/default.vue]
<template>
  <div>
    <AppHeader />
    <slot />
    <AppFooter />
  </div>
</template>
```

```vue [app/pages/index.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
    <AppAlert>
      This is an auto-imported component
    </AppAlert>
  </div>
</template>
```

```vue [app/pages/about.vue]
<template>
  <section>
    <p>This page will be displayed at the /about route.</p>
  </section>
</template>
```

::

Подробнее о создании и использовании макетов — в [разделе про макеты](/docs/4.x/directory-structure/app/layouts).

## Расширение HTML-шаблона (продвинутое)

::note
Если нужно изменить только `<head>`, см. [раздел SEO и мета](/docs/4.x/getting-started/seo-meta).
::

Полный контроль над HTML-шаблоном даёт Nitro-плагин, регистрирующий хук. В колбэке хука `render:html` можно изменить HTML перед отправкой клиенту.

```ts [server/plugins/extend-html.ts]
export default defineNitroPlugin((nitroApp) => {
  nitroApp.hooks.hook('render:html', (html, { event }) => {
    // Объектное представление HTML-шаблона
    console.log(html)
    html.head.push(`<meta name="description" content="My custom description" />`)
  })
  // Можно перехватить и ответ здесь
  nitroApp.hooks.hook('render:response', (response, { event }) => { console.log(response) })
})
```
