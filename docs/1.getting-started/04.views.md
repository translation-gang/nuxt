## `app.vue`

![Файл app.vue — точка входа приложения](/assets/docs/getting-started/views/app.svg)

По умолчанию Nuxt считает этот файл **точкой входа** и рендерит его содержимое для каждого маршрута.

```vue [app/app.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
  </div>
</template>
```

::tip
Если вы знакомы с Vue, может возникнуть вопрос, где `main.js` (обычно создаёт экземпляр Vue). В Nuxt это делается под капотом.
::

## Компоненты

![Компоненты — переиспользуемые части UI](/assets/docs/getting-started/views/components.svg)

Компоненты — это переиспользуемые части интерфейса: кнопки, меню и т.д. В Nuxt их создают в каталоге [`app/components/`](/docs/4.x/directory-structure/app/components); они автоматически доступны в приложении без явного импорта.

::code-group

```vue [app/app.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
    <AppAlert>
      This is an auto-imported component.
    </AppAlert>
  </div>
</template>
```

```vue [app/components/AppAlert.vue]
<template>
  <span>
    <slot />
  </span>
</template>
```

::

## Страницы

![Страницы — представления, привязанные к маршруту](/assets/docs/getting-started/views/pages.svg)

Страницы — представления для конкретных шаблонов маршрутов. Каждый файл в [`app/pages/`](/docs/4.x/directory-structure/app/pages) — отдельный маршрут с собственным содержимым.

Чтобы использовать страницы, создайте `app/pages/index.vue` и добавьте `<NuxtPage />` в [`app/app.vue`](/docs/4.x/directory-structure/app/app) (или удалите `app/app.vue` для поведения по умолчанию). Новые страницы и маршруты — новыми файлами в [`app/pages/`](/docs/4.x/directory-structure/app/pages).

::code-group

```vue [app/pages/index.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
    <AppAlert>
      This is an auto-imported component
    </AppAlert>
  </div>
</template>
```

```vue [app/pages/about.vue]
<template>
  <section>
    <p>This page will be displayed at the /about route.</p>
  </section>
</template>
```

::

:read-more{title="Раздел «Маршрутизация»" to="/docs/4.x/getting-started/routing"}

## Layouts

![Layouts оборачивают страницы](/assets/docs/getting-started/views/layouts.svg)

Layouts — обёртки вокруг страниц с общим UI (шапка, подвал и т.д.). Это Vue-файлы со `<slot />` для вывода **страницы**. По умолчанию используется `app/layouts/default.vue`. Свой layout задаётся в метаданных страницы.

::note
Если в приложении один layout, удобнее [`app/app.vue`](/docs/4.x/directory-structure/app/app) с [`<NuxtPage />`](/docs/4.x/api/components/nuxt-page).
::

::code-group

```vue [app/app.vue]
<template>
  <div>
    <NuxtLayout>
      <NuxtPage />
    </NuxtLayout>
  </div>
</template>
```

```vue [app/layouts/default.vue]
<template>
  <div>
    <AppHeader />
    <slot />
    <AppFooter />
  </div>
</template>
```

```vue [app/pages/index.vue]
<template>
  <div>
    <h1>Welcome to the homepage</h1>
    <AppAlert>
      This is an auto-imported component
    </AppAlert>
  </div>
</template>
```

```vue [app/pages/about.vue]
<template>
  <section>
    <p>This page will be displayed at the /about route.</p>
  </section>
</template>
```

::

Подробнее о нескольких layouts и их использовании — в [разделе про layouts](/docs/4.x/directory-structure/app/layouts).

## Продвинутое: расширение HTML-шаблона

::note
Если нужно изменить только `<head>`, см. [раздел SEO и meta](/docs/4.x/getting-started/seo-meta).
::

Полный контроль над HTML даёт плагин Nitro с хуком.
Колбэк хука `render:html` позволяет менять HTML перед отправкой клиенту.

<!-- TODO: figure out how to use twoslash to inject types for a different context -->

```ts [server/plugins/extend-html.ts]
export default defineNitroPlugin((nitroApp) => {
  nitroApp.hooks.hook('render:html', (html, { event }) => {
    // This will be an object representation of the html template.
    console.log(html)
    html.head.push(`<meta name="description" content="My custom description" />`)
  })
  // You can also intercept the response here.
  nitroApp.hooks.hook('render:response', (response, { event }) => { console.log(response) })
})
```
