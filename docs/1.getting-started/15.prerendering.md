Выбранные страницы можно отрендерить при сборке: Nuxt отдаёт готовый HTML вместо генерации на лету.

:read-more{title="Режимы рендеринга Nuxt" to="/docs/4.x/guide/concepts/rendering"}

## Пререндер на основе обхода ссылок

Команда [`nuxt generate`](/docs/4.x/api/commands/generate) собирает приложение и пререндерит его краулером [Nitro](/docs/4.x/guide/concepts/server-engine). Это близко к `nuxt build` с `nitro.static: true` или `nuxt build --prerender`.

Собирается сайт, поднимается инстанс Nuxt; по умолчанию пререндерится корень `/`, страницы, на которые есть ссылки, и так далее по графу.

::code-group{sync="pm"}

```bash [npm]
npx nuxt generate
```

```bash [yarn]
yarn nuxt generate
```

```bash [pnpm]
pnpm nuxt generate
```

```bash [bun]
bun x nuxt generate
```

```bash [deno]
deno x nuxt generate
```

::

Деплой каталога `.output/public` на любой static hosting или локальный просмотр: `npx serve .output/public`.

Как работает краулер Nitro:

1. Загружается HTML корня (`/`), нединамические страницы из `~/pages` и маршруты из `nitro.prerender.routes`.
2. HTML и `payload.json` сохраняются в `~/.output/public/` для статической выдачи.
3. Из HTML извлекаются ссылки `<a href="...">`.
4. Шаги 1–3 повторяются для каждой найденной ссылки, пока ссылки не закончатся.

Страницы без ссылок с «обнаружимых» страниц автоматически не попадут в пререндер.

### Извлечение payload

Nuxt создаёт `_payload.json` рядом с HTML для:
- пререндеренных маршрутов (при сборке);
- маршрутов ISR/SWR (при первом запросе).

В payload — сериализованные данные из `useAsyncData` и `useFetch`. При клиентской навигации подгружаются кэшированные payload вместо повторных запросов. Динамические маршруты вроде `pages/[...slug].vue` настраивайте через route rules, например `'/**': { isr: true }`.

::read-more{to="/docs/4.x/api/commands/generate#nuxt-generate"}
Подробнее о команде `nuxt generate`.
::

### Выборочный пререндер

В `nuxt.config` можно явно задать маршруты для пререндера [Nitro](/docs/4.x/guide/concepts/server-engine) или исключить часть (например `/dynamic`):

```ts twoslash [nuxt.config.ts]
// @errors: 2353
export default defineNuxtConfig({
  nitro: {
    prerender: {
      routes: ['/user/1', '/user/2'],
      ignore: ['/dynamic'],
    },
  },
})
```

Вместе с `crawlLinks` — добавить маршруты, которые краулер не найдёт сам (`/sitemap.xml`, `/robots.txt`):

```ts twoslash [nuxt.config.ts]
// @errors: 2353
export default defineNuxtConfig({
  nitro: {
    prerender: {
      crawlLinks: true,
      routes: ['/sitemap.xml', '/robots.txt'],
    },
  },
})
```

`nitro.prerender: true` эквивалентно `nitro.prerender.crawlLinks: true`.

::read-more{to="https://nitro.build/config#prerender"}
Пререндер в документации Nitro.
::

Также можно задать через `routeRules`:

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  routeRules: {
    // Set prerender to true to configure it to be prerendered
    '/rss.xml': { prerender: true },
    // Set it to false to configure it to be skipped for prerendering
    '/this-DOES-NOT-get-prerendered': { prerender: false },
    // Everything under /blog gets prerendered as long as it
    // is linked to from another page
    '/blog/**': { prerender: true },
  },
})
```

::read-more{to="https://nitro.build/config#routerules"}
Конфигурация `routeRules` в Nitro.
::

Краткая форма — в файле страницы через [`defineRouteRules`](/docs/4.x/api/utils/define-route-rules).

::read-more{to="/docs/4.x/guide/going-further/experimental-features#inlinerouterules" icon="i-lucide-star"}
Экспериментально: включите `experimental.inlineRouteRules` в `nuxt.config`.
::

```vue [app/pages/index.vue]
<script setup>
// Or set at the page level
defineRouteRules({
  prerender: true,
})
</script>

<template>
  <div>
    <h1>Homepage</h1>
    <p>Pre-rendered at build time</p>
  </div>
</template>
```

Это превращается в:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  routeRules: {
    '/': { prerender: true },
  },
})
```

## Конфигурация пререндера в runtime

### `prerenderRoutes`

В [контексте Nuxt](/docs/4.x/guide/going-further/nuxt-app#the-nuxt-context) в runtime можно добавить маршруты для пререндера Nitro.

```vue [app/pages/index.vue]
<script setup>
prerenderRoutes(['/some/other/url'])
prerenderRoutes('/api/content/article/my-article')
</script>

<template>
  <div>
    <h1>This will register other routes for prerendering when prerendered</h1>
  </div>
</template>
```

:read-more{title="prerenderRoutes" to="/docs/4.x/api/utils/prerender-routes"}

### Хук Nuxt `prerender:routes`

Вызывается перед пререндером, чтобы зарегистрировать дополнительные маршруты.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  hooks: {
    async 'prerender:routes' (ctx) {
      const { pages } = await fetch('https://api.some-cms.com/pages').then(
        res => res.json(),
      )
      for (const page of pages) {
        ctx.routes.add(`/${page.name}`)
      }
    },
  },
})
```

### Хук Nitro `prerender:generate`

Вызывается для каждого маршрута при пререндере — точечная обработка.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  nitro: {
    hooks: {
      'prerender:generate' (route) {
        if (route.route?.includes('private')) {
          route.skip = true
        }
      },
    },
  },
})
```
