Одна из ключевых возможностей Nuxt — файловый роутер. Каждый Vue-файл в [`app/pages/`](/docs/4.x/directory-structure/app/pages) даёт соответствующий URL (маршрут) с содержимым этого файла. Динамические импорты по страницам дают code splitting — на маршрут уходит минимум JavaScript.

## Страницы

Маршрутизация основана на [vue-router](https://router.vuejs.org): маршруты генерируются из компонентов в [`app/pages/`](/docs/4.x/directory-structure/app/pages) по имени файла.

Соглашения имён задают динамические и вложенные маршруты:

::code-group

```bash [Directory Structure]
-| pages/
---| about.vue
---| index.vue
---| posts/
-----| [id].vue
```

```json [Generated Router File]
{
  "routes": [
    {
      "path": "/about",
      "component": "pages/about.vue"
    },
    {
      "path": "/",
      "component": "pages/index.vue"
    },
    {
      "path": "/posts/:id",
      "component": "pages/posts/[id].vue"
    }
  ]
}
```

::

:read-more{to="/docs/4.x/directory-structure/app/pages"}

## Навигация

Компонент [`<NuxtLink>`](/docs/4.x/api/components/nuxt-link) связывает страницы. Рендерится `<a>` с `href` на маршрут страницы. После гидратации переходы выполняются в JS без полной перезагрузки — можно анимировать.

Когда [`<NuxtLink>`](/docs/4.x/api/components/nuxt-link) попадает во viewport на клиенте, Nuxt префетчит компоненты и payload (сгенерированные страницы) заранее — навигация быстрее.

```vue [app/pages/index.vue]
<template>
  <header>
    <nav>
      <ul>
        <li><NuxtLink to="/about">About</NuxtLink></li>
        <li><NuxtLink to="/posts/1">Post 1</NuxtLink></li>
        <li><NuxtLink to="/posts/2">Post 2</NuxtLink></li>
      </ul>
    </nav>
  </header>
</template>
```

:read-more{to="/docs/4.x/api/components/nuxt-link"}

## Параметры маршрута

Композабл [`useRoute()`](/docs/4.x/api/composables/use-route) в `<script setup>` или `setup()` даёт текущий маршрут.

```vue twoslash [pages/posts/[id\\].vue]
<script setup lang="ts">
const route = useRoute()

// When accessing /posts/1, route.params.id will be 1
console.log(route.params.id)
</script>
```

:read-more{to="/docs/4.x/api/composables/use-route"}

## Route middleware

В Nuxt есть настраиваемый **route middleware** — код, который выполняется перед переходом на маршрут.

::note
Route middleware работает в Vue-части приложения. По названию похож на server middleware в Nitro, но это разные вещи.
::

::important
Route middleware **не** выполняется для server routes (например `/api/*`) и прочих серверных запросов. Для них используйте [server middleware](/docs/4.x/directory-structure/server#server-middleware).
::

Три вида:

1. Анонимный (inline) — прямо в странице, где используется.
2. Именованный — файлы в [`app/middleware/`](/docs/4.x/directory-structure/app/middleware), подгружаются асинхронно при указании на странице. (**Имя** нормализуется в kebab-case: `someMiddleware` → `some-middleware`.)
3. Глобальный — в [`app/middleware/`](/docs/4.x/directory-structure/app/middleware) с суффиксом `.global`, выполняется при каждой смене маршрута.

Пример middleware `auth`, защищающего `/dashboard`:

::code-group

```ts twoslash [middleware/auth.ts]
function isAuthenticated (): boolean { return false }
// ---cut---
export default defineNuxtRouteMiddleware((to, from) => {
  // isAuthenticated() is an example method verifying if a user is authenticated
  if (isAuthenticated() === false) {
    return navigateTo('/login')
  }
})
```

```vue twoslash [pages/dashboard.vue]
<script setup lang="ts">
definePageMeta({
  middleware: 'auth',
})
</script>

<template>
  <h1>Welcome to your dashboard</h1>
</template>
```

::

:read-more{to="/docs/4.x/directory-structure/app/middleware"}

## Валидация маршрута

Валидация через свойство `validate` в [`definePageMeta()`](/docs/4.x/api/utils/define-page-meta) на нужной странице.

`validate` получает `route`. Верните `boolean` — валиден ли маршрут для этой страницы. `false` даёт 404. Можно вернуть объект с `status`/`statusText` для своей ошибки.

Сложные случаи — анонимный route middleware.

```vue twoslash [pages/posts/[id\\].vue]
<script setup lang="ts">
definePageMeta({
  validate (route) {
    // Check if the id is made up of digits
    return typeof route.params.id === 'string' && /^\d+$/.test(route.params.id)
  },
})
</script>
```
