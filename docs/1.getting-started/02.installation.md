## Играйте онлайн

Если хотите попробовать Nuxt в браузере без создания проекта, используйте одну из онлайн-песочниц:

::card-group
  :card{title="Открыть в StackBlitz" icon="i-simple-icons-stackblitz" to="https://nuxt.new/s/v4" target="_blank"}
  :card{title="Открыть в CodeSandbox" icon="i-simple-icons-codesandbox" to="https://nuxt.new/c/v4" target="_blank"}
::

Или следуйте шагам ниже, чтобы создать новый проект Nuxt на компьютере.

## Новый проект

### Требования

- **Node.js** — [`20.x`](https://nodejs.org/en) или новее (рекомендуем [активный LTS](https://github.com/nodejs/release#release-schedule))
- **Редактор кода** — IDE не навязана; рекомендуем [Visual Studio Code](https://code.visualstudio.com/) с [официальным расширением Vue](https://marketplace.visualstudio.com/items?itemName=Vue.volar) (раньше Volar) или [WebStorm](https://www.jetbrains.com/webstorm/) — вместе с [другими IDE JetBrains](https://www.jetbrains.com/ides/) даёт хорошую поддержку Nuxt из коробки. В другом редакторе (например Neovim) можно настроить [Vue Language Server](https://github.com/vuejs/language-tools) по [инструкциям Vue Language Tools](https://github.com/vuejs/language-tools/wiki).
- **Терминал** — чтобы запускать команды Nuxt

::note
  ::details
  :summary[Дополнительные заметки для комфортной настройки:]
  - **Node.js**: используйте чётную версию (20, 22 и т.д.)
  - **Neovim**: при настройке Vue TypeScript-плагина `location` должен указывать на каталог пакета `@vue/language-server`, а не на бинарник. См. [гайд по Neovim](https://github.com/vuejs/language-tools/wiki/Neovim).
  - **WSL**: на Windows при медленном HMR попробуйте [WSL](https://learn.microsoft.com/en-us/windows/wsl/install) — часто помогает с производительностью.
  - **Медленный DNS на Windows**: для локального dev-сервера вместо `localhost:3000` используйте `127.0.0.1` — в браузере загрузка заметно быстрее.
  ::
::

Откройте терминал (в [Visual Studio Code](https://code.visualstudio.com) — [встроенный терминал](https://code.visualstudio.com/docs/terminal/basics)) и выполните команду создания стартового проекта:

::code-group{sync="pm"}

```bash [npm]
npm create nuxt@latest <project-name>
```

```bash [yarn]
yarn create nuxt <project-name>
```

```bash [pnpm]
pnpm create nuxt@latest <project-name>
```

```bash [bun]
bun create nuxt@latest <project-name>
```

```bash [deno]
deno -A npm:create-nuxt@latest <project-name>
```

::

::tip
Другие стартеры и темы — на [nuxt.new](https://nuxt.new) по инструкциям на сайте.
::

Откройте папку проекта в Visual Studio Code:

```bash [Terminal]
code <project-name>
```

Или перейдите в каталог проекта в терминале:

```bash
cd <project-name>
```

## Сервер разработки

Запустите приложение Nuxt в режиме разработки:

::code-group{sync="pm"}

```bash [npm]
npm run dev -- -o
```

```bash [yarn]
yarn dev --open
```

```bash [pnpm]
pnpm dev -o
```

```bash [bun]
bun run dev -o

# Чтобы использовать runtime Bun в разработке
# bun --bun run dev -o
```

```bash [deno]
deno run dev -o
```
::

::tip{icon="i-lucide-circle-check"}
Готово! Браузер должен сам открыть <http://localhost:3000>.
::
