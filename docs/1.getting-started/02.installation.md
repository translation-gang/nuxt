## Играть онлайн

Если хотите попробовать Nuxt в браузере без настройки проекта, используйте один из наших онлайн-sandbox:

::card-group
  :card{title="Open on StackBlitz" icon="i-simple-icons-stackblitz" to="https://nuxt.new/s/v4" target="_blank"}
  :card{title="Open on CodeSandbox" icon="i-simple-icons-codesandbox" to="https://nuxt.new/c/v4" target="_blank"}
::

Либо выполните шаги ниже, чтобы создать новый проект Nuxt на своём компьютере.

## Новый проект

### Требования

- **Node.js** — [`20.x`](https://nodejs.org/en) или новее (рекомендуется [активный LTS](https://github.com/nodejs/release#release-schedule))
- **Текстовый редактор** — жёстких требований к IDE нет; рекомендуем [Visual Studio Code](https://code.visualstudio.com/) с [официальным расширением Vue](https://marketplace.visualstudio.com/items?itemName=Vue.volar) (ранее Volar) или [WebStorm](https://www.jetbrains.com/webstorm/), который вместе с [другими IDE JetBrains](https://www.jetbrains.com/ides/) хорошо поддерживает Nuxt из коробки. Для другого редактора (например Neovim) можно настроить [Vue Language Server](https://github.com/vuejs/language-tools) по [гайдам Vue Language Tools](https://github.com/vuejs/language-tools/wiki).
- **Терминал** — для запуска команд Nuxt

::note
  ::details
  :summary[Дополнительные замечания для оптимальной настройки:]
  - **Node.js**: используйте чётную версию (20, 22 и т.д.)
  - **Neovim**: при настройке плагина Vue TypeScript укажите в `location` директорию пакета `@vue/language-server`, а не бинарник. См. [гайд по Neovim](https://github.com/vuejs/language-tools/wiki/Neovim).
  - **WSL**: при медленном HMR на Windows можно попробовать [WSL](https://learn.microsoft.com/en-us/windows/wsl/install) — это может устранить проблемы с производительностью.
  - **Медленный DNS на Windows**: вместо `localhost:3000` для локального dev-сервера используйте `127.0.0.1` для более быстрой загрузки в браузере.
  ::
::

Откройте терминал (в [Visual Studio Code](https://code.visualstudio.com) можно открыть [встроенный терминал](https://code.visualstudio.com/docs/terminal/basics)) и выполните команду для создания нового стартового проекта:

::code-group{sync="pm"}

```bash [npm]
npm create nuxt@latest <project-name>
```

```bash [yarn]
yarn create nuxt <project-name>
```

```bash [pnpm]
pnpm create nuxt@latest <project-name>
```

```bash [bun]
bun create nuxt@latest <project-name>
```

```bash [deno]
deno -A npm:create-nuxt@latest <project-name>
```

::

::tip
Другие стартеры и темы можно найти на [nuxt.new](https://nuxt.new), следуя инструкциям там.
::

Откройте папку проекта в Visual Studio Code:

```bash [Terminal]
code <project-name>
```

Или перейдите в каталог проекта в терминале:

```bash
cd <project-name>
```

## Сервер разработки

Запустите приложение Nuxt в режиме разработки:

::code-group{sync="pm"}

```bash [npm]
npm run dev -- -o
```

```bash [yarn]
yarn dev --open
```

```bash [pnpm]
pnpm dev -o
```

```bash [bun]
bun run dev -o

# Чтобы использовать рантайм Bun при разработке
# bun --bun run dev -o
```

```bash [deno]
deno run dev -o
```
::

::tip{icon="i-lucide-circle-check"}
Готово! Окно браузера должно открыться на <http://localhost:3000>.
::
