## Попробовать онлайн

Если вы хотите попробовать Nuxt в браузере без локальной установки, воспользуйтесь нашими онлайн-песочницами:

::card-group
  :card{title="Открыть в StackBlitz" icon="i-simple-icons-stackblitz" to="https://nuxt.new/s/v4" target="_blank"}
  :card{title="Открыть в CodeSandbox" icon="i-simple-icons-codesandbox" to="https://nuxt.new/c/v4" target="_blank"}
::

Или следуйте шагам ниже, чтобы запустить проект на своём компьютере.

## Новый проект

### Требования

- **Node.js** - [`20.x`](https://nodejs.org/en) или новее (рекомендуем [активный LTS-релиз](https://github.com/nodejs/release#release-schedule))
- **Редактор кода** — жёстких требований к IDE нет; удобны [Visual Studio Code](https://code.visualstudio.com/) с [официальным расширением Vue](https://marketplace.visualstudio.com/items?itemName=Vue.volar) (ранее Volar) или [WebStorm](https://www.jetbrains.com/webstorm/) и [другие IDE JetBrains](https://www.jetbrains.com/ides/) с поддержкой Nuxt из коробки. В других редакторах (например Neovim) можно настроить [Vue Language Server](https://github.com/vuejs/language-tools) по [инструкциям Vue Language Tools](https://github.com/vuejs/language-tools/wiki).
- **Терминал** — чтобы запускать команды Nuxt

::note
  ::details
  :summary[Дополнительные заметки для комфортной настройки:]
  - **Node.js**: используйте чётную версию (20, 22 и т. д.)
  - **Neovim**: при настройке Vue TypeScript-плагина укажите `location` на каталог пакета `@vue/language-server`, а не на его бинарник. См. [руководство по Neovim](https://github.com/vuejs/language-tools/wiki/Neovim).
  - **WSL**: на Windows при медленном HMR попробуйте [WSL (Windows Subsystem for Linux)](https://learn.microsoft.com/en-us/windows/wsl/install) — это часто улучшает производительность.
  - **Медленное разрешение DNS в Windows**: для локального dev-сервера вместо `localhost:3000` используйте `127.0.0.1` — в браузере загрузка может быть заметно быстрее.
  ::
::

Откройте терминал (в [Visual Studio Code](https://code.visualstudio.com) можно открыть [встроенный терминал](https://code.visualstudio.com/docs/terminal/basics)) и создайте стартовый проект командой:

::code-group{sync="pm"}

```bash [npm]
npm create nuxt@latest <project-name> -- -t v3
```

```bash [yarn]
yarn create nuxt <project-name> -t v3
```

```bash [pnpm]
pnpm create nuxt@latest <project-name> -t v3
```

```bash [bun]
bun create nuxt@latest <project-name> -t v3
```

```bash [deno]
deno -A npm:create-nuxt@latest <project-name> -t v3
```

::

::tip
Также можно выбрать другие стартеры или темы на [nuxt.new](https://nuxt.new) и следовать инструкциям там.
::

Откройте папку проекта в Visual Studio Code:

```bash [Terminal]
code <project-name>
```

Или измените текущую директорию на директорию вашего проекта:

```bash
cd <project-name>
```

## Сервер разработки

Теперь можно запустить приложение Nuxt в режиме разработки:

::code-group{sync="pm"}

```bash [npm]
npm run dev -- -o
```

```bash [yarn]
yarn dev --open
```

```bash [pnpm]
pnpm dev -o
```

```bash [bun]
bun run dev -o

# To use the Bun runtime during development
# bun --bun run dev -o
```

```bash [deno]
deno run dev -o
```
::

::tip{icon="i-lucide-circle-check"}
Отлично! Теперь в окне браузера должно автоматически открыться <http://localhost:3000>.
::
