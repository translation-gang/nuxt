::

## Переходы страниц (page transitions)

Можно включить переходы страниц — автоматический transition для всех [страниц](/docs/4.x/directory-structure/app/pages).

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  app: {
    pageTransition: { name: 'page', mode: 'out-in' },
  },
})
```

::note
Если одновременно меняются и layout, и страница, настроенный здесь page transition не выполнится. Нужен [layout transition](/docs/4.x/getting-started/transitions#layout-transitions).
::

Чтобы добавить переход между страницами, подключите CSS в [`app.vue`](/docs/4.x/directory-structure/app/app):

::code-group

```vue [app/app.vue]
<template>
  <NuxtPage />
</template>

<style>
.page-enter-active,
.page-leave-active {
  transition: all 0.4s;
}
.page-enter-from,
.page-leave-to {
  opacity: 0;
  filter: blur(1rem);
}
</style>
```

```vue [app/pages/index.vue]
<template>
  <div>
    <h1>Home page</h1>
    <NuxtLink to="/about">About page</NuxtLink>
  </div>
</template>
```

```vue [app/pages/about.vue]
<template>
  <div>
    <h1>About page</h1>
    <NuxtLink to="/">Home page</NuxtLink>
  </div>
</template>
```

::

Другой transition для страницы задаётся ключом `pageTransition` в [`definePageMeta`](/docs/4.x/api/utils/define-page-meta):

::code-group

```vue twoslash [pages/about.vue]
<script setup lang="ts">

  },

```

```vue [app/app.vue]

</template>

<style>

  transition: all 0.4s;
}
.rotate-enter-from,
.rotate-leave-to {
  opacity: 0;
  transform: rotate3d(1, 1, 1, 15deg);
}
</style>
```

## Переходы layout

Можно включить layout transitions — автоматический переход для всех [layouts](/docs/4.x/directory-structure/app/layouts).

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  app: {
    layoutTransition: { name: 'layout', mode: 'out-in' },
  },
})
```

Добавьте CSS в [`app.vue`](/docs/4.x/directory-structure/app/app) для переходов между страницами и layouts:

::code-group

```vue [app/app.vue]
<template>
  <NuxtLayout>
    <NuxtPage />
  </NuxtLayout>
</template>

<style>
.layout-enter-active,
.layout-leave-active {
  transition: all 0.4s;
}
.layout-enter-from,
.layout-leave-to {
  filter: grayscale(1);
}
</style>
```

```vue [app/layouts/default.vue]
<template>
  <div>
    <pre>default layout</pre>
    <slot />
  </div>
</template>

<style scoped>
div {
  background-color: lightgreen;
}
</style>
```

```vue [app/layouts/orange.vue]
<template>
  <div>
    <pre>orange layout</pre>
    <slot />
  </div>
</template>

<style scoped>
div {
  background-color: #eebb90;
  padding: 20px;
  height: 100vh;
}
</style>
```

```vue [app/pages/index.vue]
<template>
  <div>
    <h1>Home page</h1>
    <NuxtLink to="/about">About page</NuxtLink>
  </div>
</template>
```

```vue [app/pages/about.vue]
<script setup lang="ts">
definePageMeta({
  layout: 'orange',
})
</script>

<template>
  <div>
    <h1>About page</h1>
    <NuxtLink to="/">Home page</NuxtLink>
  </div>
</template>
```

## Глобальные настройки

Имена и параметры переходов по умолчанию задаются в `nuxt.config`.

Ключи `pageTransition` и `layoutTransition` принимают [`TransitionProps`](https://vuejs.org/api/built-in-components#transition) как JSON-сериализуемые значения: `name`, `mode` и другие валидные props transition.

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  app: {
    pageTransition: {
      name: 'fade',
      mode: 'out-in', // default
    },
    layoutTransition: {
      name: 'slide',
      mode: 'out-in', // default
    },
  },
})
```

::warning
Если меняете `name`, переименуйте и соответствующие CSS-классы.
::

Чтобы переопределить глобальные настройки, в `definePageMeta` задайте `pageTransition` или `layoutTransition` для конкретной страницы.

```vue twoslash [pages/some-page.vue]
<script setup lang="ts">
definePageMeta({
  pageTransition: {
    name: 'bounce',
    mode: 'out-in', // default
  },
})
</script>
```

## Отключение переходов

`pageTransition` и `layoutTransition` можно отключить для конкретного маршрута:

```vue twoslash [pages/some-page.vue]
<script setup lang="ts">
definePageMeta({
  pageTransition: false,
  layoutTransition: false,
})
</script>
```

Или глобально в `nuxt.config`:

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  app: {
    pageTransition: false,
    layoutTransition: false,
  },
})
```

## JavaScript-хуки

Для сложных сценариев используйте JS-хуки transition — динамичные кастомные переходы.

Удобно с библиотеками вроде [GSAP](https://gsap.com).

```vue twoslash [pages/some-page.vue]
<script setup lang="ts">
definePageMeta({
  pageTransition: {
    name: 'custom-flip',
    mode: 'out-in',
    onBeforeEnter: (el) => {
      console.log('Before enter...')
    },
    onEnter: (el, done) => {},
    onAfterEnter: (el) => {},
  },
})
</script>
```

::tip
Дополнительные [JS-хуки](https://vuejs.org/guide/built-ins/transition#javascript-hooks) компонента `Transition`.
::

## Динамические переходы

Условная логика: через inline-[middleware](/docs/4.x/directory-structure/app/middleware) задайте разное имя перехода в `to.meta.pageTransition`.

::code-group

```vue twoslash [pages/[id\\].vue]
<script setup lang="ts">
definePageMeta({
  pageTransition: {
    name: 'slide-right',
    mode: 'out-in',
  },
  middleware (to, from) {
    if (to.meta.pageTransition && typeof to.meta.pageTransition !== 'boolean') {
      to.meta.pageTransition.name = +to.params.id! > +from.params.id! ? 'slide-left' : 'slide-right'
    }
  },
})
</script>

<template>
  <h1>#{{ $route.params.id }}</h1>
</template>

<style>
.slide-left-enter-active,
.slide-left-leave-active,
.slide-right-enter-active,
.slide-right-leave-active {
  transition: all 0.2s;
}
.slide-left-enter-from {
  opacity: 0;
  transform: translate(50px, 0);
}
.slide-left-leave-to {
  opacity: 0;
  transform: translate(-50px, 0);
}
.slide-right-enter-from {
  opacity: 0;
  transform: translate(-50px, 0);
}
.slide-right-leave-to {
  opacity: 0;
  transform: translate(50px, 0);
}
</style>
```

```vue [app/layouts/default.vue]
<script setup lang="ts">
const route = useRoute()
const id = computed(() => Number(route.params.id || 1))
const prev = computed(() => '/' + (id.value - 1))
const next = computed(() => '/' + (id.value + 1))
</script>

<template>
  <div>
    <slot />
    <div v-if="$route.params.id">
      <NuxtLink :to="prev">⬅️</NuxtLink> |
      <NuxtLink :to="next">➡️</NuxtLink>
    </div>
  </div>
</template>
```

::

Страница использует `slide-left` при переходе к следующему id и `slide-right` к предыдущему:

## Transition у NuxtPage

Если в `app.vue` стоит `<NuxtPage />`, переходы можно задать пропом `transition` глобально.

```vue [app/app.vue]
<template>
  <div>
    <NuxtLayout>
      <NuxtPage
        :transition="{
          name: 'bounce',
          mode: 'out-in',
        }"
      />
    </NuxtLayout>
  </div>
</template>
```

::note
Такой page transition нельзя переопределить через `definePageMeta` на отдельных страницах.
::

## View Transitions API (экспериментально)

В Nuxt есть экспериментальная поддержка [**View Transitions API**](https://developer.chrome.com/docs/web-platform/view-transitions) ([MDN](https://developer.mozilla.org/en-US/docs/Web/API/View_Transition_API)) — нативные переходы в браузере, в том числе между несвязанными элементами на разных страницах.

Демо [на StackBlitz](https://stackblitz.com/edit/nuxt-view-transitions).

Включение в конфиге — опция `experimental.viewTransition`:

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  experimental: {
    viewTransition: true,
  },
})
```

Допустимые значения: `false`, `true` или `'always'`.

При `true` Nuxt не применяет переходы, если в браузере `prefers-reduced-motion: reduce` (рекомендуется). При `always` переходы всегда включены — учитывать пожелания пользователя нужно самостоятельно.

По умолчанию view transitions включены для всех [страниц](/docs/4.x/directory-structure/app/pages); глобально можно задать иное значение по умолчанию.

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  app: {
    // Disable view transitions globally, and opt-in on a per page basis
    viewTransition: false,
  },
})
```

Для страницы значение по умолчанию переопределяется ключом `viewTransition` в [`definePageMeta`](/docs/4.x/api/utils/define-page-meta):

```vue twoslash [pages/about.vue]
<script setup lang="ts">
definePageMeta({
  viewTransition: false,
})
</script>
```

::warning
Переопределение view transitions на уровне страницы сработает только если включён `experimental.viewTransition`.
::

### Типы view transition

[Типы view transition](https://developer.chrome.com/blog/view-transitions-update-io24#view-transition-types) задают разные CSS-анимации в зависимости от навигации — удобно для асимметрии (вперёд / назад).

Типы задаются на объекте [`ViewTransition`](https://developer.mozilla.org/en-US/docs/Web/API/ViewTransition) и стилизуются через псевдокласс [`:active-view-transition-type()`](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Selectors/:active-view-transition-type).

Глобально по умолчанию — в `nuxt.config.ts`:

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  app: {
    viewTransition: {
      enabled: true,
      types: ['slide'],
    },
  },
})
```

Или типы на странице через `definePageMeta` — статические массивы или функции:

```vue twoslash [pages/detail.vue]
<script setup lang="ts">
definePageMeta({
  viewTransition: {
    enabled: true,
    // Types applied to any transition involving this page
    types: ['slide'],
    // Types applied only when navigating TO this page
    toTypes: ['slide-in'],
    // Types applied only when navigating FROM this page
    fromTypes: ['slide-out'],
  },
})
</script>
```

Функции для `types`, `toTypes` и `fromTypes` в `definePageMeta` вычисляют типы по маршруту:

```vue twoslash [pages/[id].vue]
<script setup lang="ts">
definePageMeta({
  viewTransition: {
    enabled: true,
    toTypes: (to, from) => {
      // Slide left when going to a higher ID, right otherwise
      return Number(to.params.id) > Number(from.params.id)
        ? ['slide-left']
        : ['slide-right']
    },
  },
})
</script>
```

В CSS нацельтесь на эти типы:

```css
/* Default crossfade */
::view-transition-old(root),
::view-transition-new(root) {
  animation-duration: 0.3s;
}

/* Slide left animation */
html:active-view-transition-type(slide-left) {
  &::view-transition-old(root) {
    animation: slide-out-left 0.3s ease-in-out;
  }
  &::view-transition-new(root) {
    animation: slide-in-right 0.3s ease-in-out;
  }
}

/* Slide right animation */
html:active-view-transition-type(slide-right) {
  &::view-transition-old(root) {
    animation: slide-out-right 0.3s ease-in-out;
  }
  &::view-transition-new(root) {
    animation: slide-in-left 0.3s ease-in-out;
  }
}
```

::note
Функции для `types`, `toTypes` и `fromTypes` работают только в `definePageMeta`, не в `nuxt.config.ts` (там поддерживается лишь статический `string[]`).
::

Хук `page:view-transition:start` даёт доступ к объекту [`ViewTransition`](https://developer.mozilla.org/en-US/docs/Web/API/ViewTransition) и свойству [`types`](https://developer.mozilla.org/en-US/docs/Web/API/ViewTransition/types) (`ViewTransitionTypeSet`) — чтение и изменение в runtime:

```ts [plugins/view-transition.client.ts]
export default defineNuxtPlugin((nuxtApp) => {
  nuxtApp.hook('page:view-transition:start', (transition) => {
    // Read or modify types at runtime
    console.log([...transition.types])
  })
})
```

Если параллельно используются Vue-переходы `pageTransition` и `layoutTransition` для того же эффекта, что даёт View Transitions API, имеет смысл **отключить** Vue-переходы в браузерах с нативной поддержкой. Создайте `~/middleware/disable-vue-transitions.global.ts`:

```ts
export default defineNuxtRouteMiddleware((to) => {
  if (import.meta.server || !document.startViewTransition) {
    return
  }

  // Disable built-in Vue transitions
  to.meta.pageTransition = false
  to.meta.layoutTransition = false
})
```

### Известные ограничения

- Если в `setup` страницы выполняется загрузка данных, эту функцию пока стоит использовать осторожно: View Transitions по задумке **замораживают** обновления DOM на время анимации. Планируется ограничить transition финальными моментами до разрешения `<Suspense>`; пока оцените риски, если ваш сценарий такой.
