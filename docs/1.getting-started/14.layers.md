Одна из ключевых возможностей Nuxt — layers и расширение. Можно переиспользовать компоненты, утилиты и конфигурацию. Структура слоя почти как у обычного Nuxt-приложения — проще писать и поддерживать.

## Сценарии

- Общие пресеты конфигурации в `nuxt.config` и `app.config`
- Библиотека компонентов в [`app/components/`](/docs/4.x/directory-structure/app/components)
- Библиотека композаблов и утилит в [`app/composables/`](/docs/4.x/directory-structure/app/composables) и [`app/utils/`](/docs/4.x/directory-structure/app/utils)
- Пресеты Nuxt-модулей
- Единая база для нескольких проектов
- Темы для Nuxt
- Модульная архитектура и DDD в крупных проектах

## Использование

По умолчанию каталоги в `~~/layers` автоматически регистрируются как слои.

::note
Авторегистрация слоёв — с Nuxt v3.12.0.
::

Для каждого слоя создаётся именованный алиас к `srcDir`, например `~~/layers/test` → `#layers/test`.

::note
Именованные алиасы слоёв — с Nuxt v3.16.0.
::

Слой можно подключить через свойство [extends](/docs/4.x/api/nuxt-config#extends) в [`nuxt.config`](/docs/4.x/directory-structure/nuxt-config).

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: [
    // Extend from a local layer
    '../base',
    // Extend from an installed npm package
    '@my-themes/awesome',
    // Extend from a git repository
    'github:my-themes/awesome#v1',
  ],
})
```

Для приватного репозитория GitHub можно передать токен:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: [
    // per layer configuration
    ['github:my-themes/private-awesome', { auth: process.env.GITHUB_TOKEN }],
  ],
})
```

::note
Если ветка не указана, клонируется `main`.
::

::tip
Алиас слоя можно переопределить в опциях рядом с источником.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: [
    [
      'github:my-themes/awesome',
      {
        meta: {
          name: 'my-awesome-theme',
        },
      },
    ],
  ],
})
```

::

Удалённые слои подключаются через [unjs/c12](https://github.com/unjs/c12) и [unjs/giget](https://github.com/unjs/giget) — см. их документацию по опциям.

## Приоритет слоёв

При нескольких слоях важен порядок переопределения: **больший приоритет** перекрывает меньший при совпадении файлов или компонентов.

### Порядок приоритета

От большего к меньшему:

1. **Файлы вашего проекта** — всегда выше всех
2. **Автоподхваченные слои** из `~~/layers` — по алфавиту (Z выше A)
3. **Слои в `extends`** — первый в списке важнее второго

### Пример

Несколько слоёв с одним и тем же компонентом:

```bash [Directory structure]
layers/
  1.base/
    app/components/Button.vue    # Base button style
  2.theme/
    app/components/Button.vue    # Themed button (overrides base)
app/
  components/Button.vue          # Project button (overrides all layers)
```

Итог:
- Только слои — берётся `2.theme/Button.vue` (выше по алфавиту среди слоёв)
- Если есть `app/components/Button.vue` в проекте — он перекрывает все слои

### Управление порядком

Префиксы с номерами в имени каталога слоя:

```bash [Directory structure]
layers/
  1.base/        # Lowest priority
  2.features/    # Medium priority
  3.admin/       # Highest priority (among layers)
```

::tip
Удобно для базового слоя с дефолтами и последовательного переопределения более специфичными слоями.
::

### Что когда использовать

- **`~~/layers`** — локальные слои внутри репозитория
- **`extends`** — npm, удалённые репозитории или слои вне каталога проекта

### Полный пример с `extends`

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: [
    '../base', // Local layer outside project
    '@my-themes/awesome', // NPM package
    'github:my-themes/awesome#v1', // Remote repository
  ],
})
```

Если ещё есть `~~/layers/custom`, порядок такой:
- файлы проекта (выше всех)
- `~~/layers/custom`
- `../base`
- `@my-themes/awesome`
- `github:my-themes/awesome#v1` (ниже всех)
