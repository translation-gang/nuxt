В этой главе — обзор частей фреймворка, порядка их выполнения и того, как они связаны.

## Жизненный цикл на сервере

На сервере для каждого первого запроса выполняются такие шаги:

::steps
### Плагины сервера :badge[once]{color="info" class="align-middle"}

Nuxt работает на [Nitro](https://nitro.build/).

При старте Nitro инициализирует и выполняет плагины из [`/server/plugins`](/docs/4.x/directory-structure/server#server-plugins). Они могут:
- Перехватывать и обрабатывать ошибки приложения.
- Регистрировать хуки при остановке Nitro.
- Регистрировать хуки жизненного цикла запроса, например менять ответ.

::callout{icon="i-lucide-lightbulb"}
Плагины Nitro выполняются один раз при старте сервера. В serverless окружении сервер поднимается на каждый запрос — поднимаются и плагины, но они не ожидаются (`await`).
::

:read-more{to="/docs/4.x/directory-structure/server#server-plugins"}

### Middleware сервера

После инициализации Nitro для каждого запроса выполняется middleware из `server/middleware/`: авторизация, логирование, преобразование запроса.

::warning
Возврат значения из middleware завершает запрос и отправляет это значение как ответ. Так обычно лучше не делать, чтобы не ломать обработку запросов!
::

:read-more{to="/docs/4.x/directory-structure/server#server-middleware"}

### Плагины приложения

Сначала создаются экземпляры Vue и Nuxt, затем выполняются плагины приложения:
- Встроенные (Vue Router, `unhead` и др.).
- Свои из `app/plugins/` — без суффикса (`myPlugin.ts`) и с `.server` (`myServerPlugin.server.ts`).

Порядок выполнения фиксирован, возможны зависимости. Подробнее — [документация по плагинам](/docs/4.x/directory-structure/app/plugins).

::callout{icon="i-lucide-lightbulb"}
После этого Nuxt вызывает хук [`app:created`](/docs/4.x/api/advanced/hooks#app-hooks-runtime) для дополнительной логики.
::

:read-more{to="/docs/4.x/directory-structure/app/plugins"}

### Валидация маршрута

После плагинов и до middleware Nuxt вызывает `validate` из `definePageMeta`, если он задан. Функция может быть синхронной или асинхронной и проверяет параметры динамического маршрута.

- При успехе вернуть `true`.
- При ошибке — `false` или объект с `status` и/или `statusText`, чтобы завершить запрос.

Подробнее — [валидация маршрутов](/docs/4.x/getting-started/routing#route-validation).

:read-more{to="/docs/4.x/getting-started/routing#route-validation"}

### Middleware приложения

Middleware выполняется до перехода на маршрут: авторизация, редиректы, логирование.

Типы:
- **Глобальный route middleware**
- **Именованный route middleware**
- **Анонимный (inline) route middleware**

Глобальный выполняется при первой загрузке (сервер и клиент) и снова перед каждой клиентской навигацией. Именованный и анонимный — только на маршрутах, указанных в `middleware` в метаданных страницы.

Подробнее — [документация по middleware](/docs/4.x/directory-structure/app/middleware).

Редирект на сервере отдаёт заголовок `Location:`; браузер делает новый запрос. Состояние приложения сбрасывается, если его не сохранить в cookie.

:read-more{to="/docs/4.x/directory-structure/app/middleware"}

### Страница и компоненты

Nuxt рендерит страницу и компоненты, подгружает данные через `useFetch` и `useAsyncData`. На сервере нет динамических обновлений DOM, поэтому хуки Vue `onBeforeMount`, `onMounted` и дальше **не** выполняются при SSR.

По умолчанию Vue на SSR приостанавливает трекинг зависимостей ради производительности.

::callout{icon="i-lucide-lightbulb"}
На сервере нет реактивности в привычном смысле: SSR идёт сверху вниз в статический HTML, уже отрисованное нельзя «переделать».
::

::important
Избегайте побочных эффектов в корне `<script setup>`, которые нужно снимать. Пример: `setInterval` в SSR не вызовет `onBeforeUnmount`/`onUnmounted` — таймеры останутся. Выносите такой код в `onMounted`.
::

::tip{icon="i-lucide-video" to="https://youtu.be/dZSNW07sO-A" target="_blank"}
Видео Daniel Roe: SSR и глобальное состояние.
::

### Вывод HTML

Когда данные получены и компоненты отрисованы, Nuxt собирает HTML с учётом `unhead` и отправляет клиенту вместе с payload — SSR завершён.

::callout{icon="i-lucide-lightbulb"}
После рендера Vue в HTML вызывается хук [`app:rendered`](/docs/4.x/api/advanced/hooks#app-hooks-runtime).
::

::callout{icon="i-lucide-lightbulb"}
Перед финальной отправкой HTML Nitro вызывает [`render:html`](/docs/4.x/api/advanced/hooks#nitro-app-hooks-runtime-server-side) — можно править HTML, инжектить скрипты, менять мета-теги.
::

::

## Жизненный цикл на клиенте

Выполняется целиком в браузере, независимо от выбранного режима Nuxt.

::steps

### Плагины приложения

Аналогично серверу: встроенные и пользовательские плагины.

Из `app/plugins/` на клиенте — без суффикса и с `.client` (`myClientPlugin.client.ts`).

::callout{icon="i-lucide-lightbulb"}
После шага снова вызывается [`app:created`](/docs/4.x/api/advanced/hooks#app-hooks-runtime).
::

:read-more{to="/docs/4.x/directory-structure/app/plugins"}

### Валидация маршрута

Как на сервере: `validate` из `definePageMeta`, если задан.

### Middleware приложения

Middleware Nuxt работает и на сервере, и на клиенте. Для разделения по средам используйте `import.meta.client` и `import.meta.server`.

:read-more{to="/docs/4.x/directory-structure/app/middleware#when-middleware-runs"}

### Монтирование Vue и гидратация

`app.mount('#__nuxt')` монтирует приложение в DOM. При SSR/SSG Vue выполняет гидратацию для интерактивности: воссоздаёт приложение (кроме [Server Components](/docs/4.x/directory-structure/app/components#server-components)), сопоставляет компоненты с узлами DOM, вешает обработчики.

Данные на сервере и клиенте должны совпадать. Для API используйте `useAsyncData`, `useFetch` и другие SSR-friendly композаблы — данные с сервера переиспользуются при гидратации без лишних запросов. Новые запросы — после гидратации, чтобы не было ошибок гидратации.

::callout{icon="i-lucide-lightbulb"}
Перед монтированием вызывается [`app:beforeMount`](/docs/4.x/api/advanced/hooks#app-hooks-runtime).
::

::callout{icon="i-lucide-lightbulb"}
После монтирования — [`app:mounted`](/docs/4.x/api/advanced/hooks#app-hooks-runtime).
::

### Жизненный цикл Vue

В браузере выполняется полный [жизненный цикл Vue](https://vuejs.org/guide/essentials/lifecycle).

::
