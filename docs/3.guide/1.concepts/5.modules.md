## Зачем нужны модули Nuxt

При разработке приложений на Nuxt возможностей ядра может не хватать. Nuxt можно расширять через конфиг и плагины, но поддерживать такие настройки в нескольких проектах бывает утомительно. С другой стороны, встраивать все возможные сценарии в ядро сделало бы Nuxt слишком сложным.

Поэтому в Nuxt есть система модулей для расширения ядра. Модули Nuxt — это асинхронные функции, которые по очереди выполняются при запуске Nuxt в режиме разработки ([`nuxt dev`](/docs/4.x/api/commands/dev)) или при сборке ([`nuxt build`](/docs/4.x/api/commands/build)). Они могут переопределять шаблоны, настраивать загрузчики, подключать CSS-библиотеки и выполнять другие задачи.

Модули можно распространять как npm-пакеты — их можно переиспользовать в проектах и делиться ими с сообществом.

## Подключение модулей

После установки модули подключаются в файле [`nuxt.config.ts`](/docs/4.x/directory-structure/nuxt-config) в свойстве `modules`. Обычно в документации модуля указаны дополнительные шаги и опции.

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    // Имя пакета (рекомендуемый способ)
    '@nuxtjs/example',

    // Локальный модуль
    './modules/example',

    // Модуль с опциями
    ['./modules/example', { token: '123' }],

    // Инлайн-определение модуля
    async (inlineOptions, nuxt) => { },
  ],
})
```

::warning
В Nuxt 3+ модули работают только на этапе сборки; свойство `buildModules` из Nuxt 2 устарело, используйте `modules`.
::

## Отключение модулей

Модуль можно отключить, задав его конфигурационный ключ в `false` в nuxt.config. Это удобно, когда нужно отключить модуль, унаследованный от слоя.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  // Отключить модуль @nuxt/image
  image: false,
})
```
