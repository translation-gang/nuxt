## Знакомство с модулями Nuxt

Разрабатывая production-приложения на Nuxt, вы можете столкнуться с тем, что возможностей ядра недостаточно. Nuxt расширяется настройками и плагинами, но поддерживать такие доработки в нескольких проектах утомительно, однообразно и долго. С другой стороны, закрывать все сценарии из коробки сделало бы Nuxt слишком сложным.

Поэтому в Nuxt есть система модулей, позволяющая расширять ядро. Модули Nuxt — это асинхронные функции, которые последовательно выполняются при запуске `nuxt dev` в режиме разработки или при сборке production командой `nuxt build`. Они могут переопределять шаблоны, настраивать загрузчики webpack, подключать CSS-библиотеки и решать множество других задач.

Модули можно публиковать в npm. Их можно переиспользовать в проектах и делиться с сообществом, формируя экосистему качественных дополнений.

## Подключение модулей Nuxt

После установки добавьте модули в [`nuxt.config.ts`](/docs/4.x/directory-structure/nuxt-config) в свойство `modules`. Авторы модулей обычно дают отдельные шаги и детали использования.

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  modules: [
    // По имени пакета (рекомендуется)
    '@nuxtjs/example',

    // Локальный модуль
    './modules/example',

    // Модуль с опциями в массиве
    ['./modules/example', { token: '123' }],

    // Inline-определение модуля
    async (inlineOptions, nuxt) => { },
  ],
})
```

::warning
Модули Nuxt теперь только на этапе сборки; свойство `buildModules` из Nuxt 2 устарело в пользу `modules`.
::

## Отключение модулей

Модуль можно отключить, задав для его ключа конфигурации значение `false` в конфиге Nuxt. Это удобно, чтобы отключать модули, унаследованные из слоёв.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  // Отключить модуль `@nuxt/image`
  image: false,
})
```
