Nuxt интегрирует Vue 3 — мажорную версию с новыми возможностями для разработчиков Nuxt.

::note
Глубокое знание Vue не обязательно, но мы рекомендуем документацию и примеры на [vuejs.org](https://vuejs.org).
::

Nuxt всегда строился на Vue как на UI-фреймворке.

Причины:

- Реактивная модель: изменение данных обновляет интерфейс.
- Компонентный шаблон с HTML как общим языком веба — предсказуемый UI.
- Vue хорошо масштабируется от маленьких проектов до крупных приложений.

## Vue в Nuxt

### Однофайловые компоненты

[Однофайловые компоненты Vue](https://vuejs.org/guide/scaling-up/sfc) (SFC, `*.vue`) объединяют разметку (`<template>`), логику (`<script>`) и стили (`<style>`). В Nuxt для SFC нулевая настройка и [HMR](https://vite.dev/guide/features#hot-module-replacement) от Vite.

### Автоимпорт

Компоненты в [`app/components/`](/docs/4.x/directory-structure/app/components) доступны без импорта. Неиспользуемые не попадают в production-бандл.

:read-more{to="/docs/4.x/guide/concepts/auto-imports"}

### Vue Router

**Маршрутизация** — несколько страниц и переходы между ними. Nuxt использует [`app/pages/`](/docs/4.x/directory-structure/app/pages) и соглашения об именах, создавая маршруты через [Vue Router](https://router.vuejs.org).

:read-more{to="/docs/4.x/getting-started/routing"}

:link-example{to="/docs/4.x/examples/features/auto-imports"}

## Отличия от Nuxt 2 / Vue 2

Nuxt 3+ на Vue 3. В мажорной версии Vue появилось то, на чём строится Nuxt:

- Лучшая производительность
- Composition API
- Поддержка TypeScript

### Быстрее рендеринг

Virtual DOM Vue переписан и даёт лучшую производительность рендера. Компилятор SFC на этапе сборки отделяет статическую и динамическую разметку.

Быстрее первый рендер и обновления, меньше памяти; в Nuxt 3 — быстрее и SSR.

### Меньший бандл

В Vue 3 и Nuxt 3 упор на размер бандла: большая часть функциональности Vue, директивы и встроенные компоненты — tree-shakable. В production не попадёт неиспользуемое.

Минимальное Vue 3-приложение может ужиматься до ~12 КБ gzip.

### Composition API

В Vue 2 данные и логика в шаблон передавались через Options API (`data`, `methods`):

```vue twoslash
<script>
export default {
  data () {
    return {
      count: 0,
    }
  },
  methods: {
    increment () {
      this.count++
    },
  },
}
</script>
```

[Composition API](https://vuejs.org/guide/extras/composition-api-faq) в Vue 3 не заменяет Options API, но упрощает переиспользование логики и группировку по смыслу в сложных компонентах.

С `setup` в `<script>` тот же пример с Composition API и автоимпортом реактивности в Nuxt 3:

```vue twoslash [components/Counter.vue]
<script setup lang="ts">
const count = ref(0)
const increment = () => count.value++
</script>
```

Nuxt заточен под удобную работу с Composition API:

- Автоимпорт [реактивных функций](https://vuejs.org/api/reactivity-core) Vue и [встроенных композаблов](/docs/4.x/api/composables/use-async-data) Nuxt.
- Свои переиспользуемые функции в [`app/composables/`](/docs/4.x/directory-structure/app/composables).

### TypeScript

Vue 3 и Nuxt 3+ написаны на TypeScript. Строгая типизация снижает ошибки и документирует API. Писать приложение на TS не обязательно: переименуйте `.js` в `.ts` или добавьте `<script setup lang="ts">`.
