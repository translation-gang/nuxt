Nuxt строится на Vue 3 — новом мажорном релизе Vue с новыми возможностями для пользователей Nuxt.

::note
Глубокое знание Vue не обязательно для работы с Nuxt, но мы рекомендуем прочитать документацию и примеры на [vuejs.org](https://vuejs.org).
::

Nuxt изначально использует Vue как фронтенд-фреймворк.

Мы выбрали Vue по нескольким причинам:

- Реактивная модель Vue: изменение данных автоматически отражается в интерфейсе.
- Компонентная модель на основе шаблонов с привычным HTML сохраняет понятные паттерны и даёт гибкость.
- Vue хорошо масштабируется — от небольших проектов до крупных приложений.

## Vue в Nuxt

### Однофайловые компоненты (SFC)

[Однофайловые компоненты Vue](https://vuejs.org/guide/scaling-up/sfc) (SFC, файлы `*.vue`) объединяют разметку (`<template>`), логику (`<script>`) и стили (`<style>`). В Nuxt работа с SFC настроена из коробки с [Hot Module Replacement](https://vite.dev/guide/features#hot-module-replacement) для удобной разработки.

### Автоимпорты

Каждый компонент Vue в директории [`app/components/`](/docs/4.x/directory-structure/app/components) доступен в проекте без импорта. Неиспользуемые компоненты не попадают в продакшен-сборку.

:read-more{to="/docs/4.x/guide/concepts/auto-imports"}

### Vue Router

Большинству приложений нужны несколько страниц и навигация между ними — **маршрутизация**. В Nuxt маршруты создаются по структуре директории [`app/pages/`](/docs/4.x/directory-structure/app/pages) и соглашениям об именовании с помощью [Vue Router](https://router.vuejs.org).

:read-more{to="/docs/4.x/getting-started/routing"}

:link-example{to="/docs/4.x/examples/features/auto-imports"}

## Отличия от Nuxt 2 / Vue 2

Nuxt 3+ основан на Vue 3. В новой мажорной версии Vue есть изменения, которые использует Nuxt:

- Лучшая производительность
- Composition API
- Поддержка TypeScript

### Быстрый рендеринг

Виртуальный DOM (VDOM) во Vue переписан и даёт более быстрый рендеринг. Плюс при использовании скомпилированных SFC компилятор Vue может дополнительно оптимизировать их при сборке, разделяя статическую и динамическую разметку.

В итоге быстрее первая отрисовка и обновления, меньше потребление памяти. В Nuxt 3 это также ускоряет серверный рендеринг.

### Меньший бандл

Во Vue 3 и Nuxt 3 много внимания уделено уменьшению размера бандла. Большая часть возможностей Vue, включая директивы шаблонов и встроенные компоненты, поддерживает tree-shaking. В продакшен они не попадут, если не используются.

Минимальное Vue 3-приложение может весить около 12 КБ в gzip.

### Composition API

В Vue 2 данные и логику в компоненты можно было передавать только через Options API — через предопределённые свойства вроде `data` и `methods`:

```vue twoslash
<script>
export default {
  data () {
    return {
      count: 0,
    }
  },
  methods: {
    increment () {
      this.count++
    },
  },
}
</script>
```

[Composition API](https://vuejs.org/guide/extras/composition-api-faq), появившийся во Vue 3, не заменяет Options API, но упрощает переиспользование логики и группировку кода по смыслу в сложных компонентах.

С ключевым словом `setup` в `<script>` тот же компонент на Composition API и с автоимпортируемыми API реактивности Nuxt выглядит так:

```vue twoslash [components/Counter.vue]
<script setup lang="ts">
const count = ref(0)
const increment = () => count.value++
</script>
```

Цель Nuxt — удобная работа с Composition API:

- использовать автоимпортируемые [функции реактивности](https://vuejs.org/api/reactivity-core) Vue и [встроенные композаблы](/docs/4.x/api/composables/use-async-data) Nuxt;
- писать свои переиспользуемые функции в [директории `app/composables/`](/docs/4.x/directory-structure/app/composables).

### Поддержка TypeScript

Vue 3 и Nuxt 3+ написаны на TypeScript. Полная типизация уменьшает ошибки и документирует использование API. Писать приложение на TypeScript не обязательно: можно подключить типы, переименовав файл с `.js` на `.ts` или добавив `<script setup lang="ts">` в компонент.
