Nuxt автоматически импортирует компоненты, композаблы и [API Vue.js](https://vuejs.org/api/) — без явных `import` в коде.

```vue twoslash [app/app.vue]
<script setup lang="ts">
const count = ref(1) // ref подключается автоматически
</script>
```

За счёт соглашений по каталогам Nuxt подхватывает [`app/components/`](/docs/4.x/directory-structure/app/components), [`app/composables/`](/docs/4.x/directory-structure/app/composables) и [`app/utils/`](/docs/4.x/directory-structure/app/utils).

В отличие от глобальных объявлений, Nuxt сохраняет типы, подсказки IDE и **в бандл попадает только то, что реально используется в production-коде**.

::note
В документации функции без явного импорта — автоимпортируемые Nuxt. Справочник — в разделе [API](/docs/4.x/api).
::

::note
В каталоге [`server`](/docs/4.x/directory-structure/server) Nuxt автоимпортирует экспорты из `server/utils/`.
::

::note
Свои каталоги и сторонние пакеты настраиваются в [`imports`](/docs/4.x/api/nuxt-config#imports) в `nuxt.config`.
::

## Встроенный автоимпорт

Nuxt подключает функции и композаблы для [загрузки данных](/docs/4.x/getting-started/data-fetching), [контекста приложения](/docs/4.x/api/composables/use-nuxt-app), [runtime config](/docs/4.x/guide/going-further/runtime-config), [состояния](/docs/4.x/getting-started/state-management), а также для компонентов и плагинов.

```vue twoslash
<script setup lang="ts">
/* useFetch() автоимпортируется */
const { data, refresh, status } = await useFetch('/api/hello')
</script>
```

Vue отдаёт реактивные API вроде `ref` и `computed`, хуки жизненного цикла и хелперы — тоже через автоимпорт Nuxt.

```vue twoslash
<script setup lang="ts">
/* ref() и computed() автоимпортируются */
const count = ref(1)
const double = computed(() => count.value * 2)
</script>
```

### Композаблы Vue и Nuxt

<!-- TODO: move to separate page with https://github.com/nuxt/nuxt/issues/14723 and add more information -->

Встроенные композаблы Composition API из Vue и Nuxt нужно вызывать в правильном _контексте_.

Во время жизни компонента Vue временно привязывает текущий экземпляр (и Nuxt — `nuxtApp`) к глобальной переменной и сбрасывает её в том же тике. Это важно для SSR: нельзя смешивать состояние между запросами и между компонентами.

Поэтому (с редкими исключениями) их нельзя вызывать вне плагина Nuxt, route middleware Nuxt или `setup` компонента. Плюс вызовы должны быть синхронными — нельзя делать `await` перед композаблом, кроме `<script setup>`, `setup` в `defineNuxtComponent`, `defineNuxtPlugin` и `defineNuxtRouteMiddleware`, где выполняется трансформация для сохранения контекста после `await`.

Ошибка `Nuxt instance is unavailable` обычно значит, что композабл Nuxt вызван не в том месте жизненного цикла.

:video-accordion{title="Видео Александра Лихтера: ошибка «Nuxt instance is unavailable»" videoId="ofuKRZLtOdY"}

::tip
Если композаблу нужен контекст Nuxt вне SFC, оберните компонент в `defineNuxtComponent`, а не в `defineComponent`.
::

::read-more{to="/docs/4.x/guide/going-further/experimental-features#asynccontext" icon="i-lucide-star"}
Эксперимент `asyncContext` — композаблы Nuxt в async-функциях.
::

::read-more{to="https://github.com/nuxt/nuxt/issues/14269#issuecomment-1397352832" target="_blank"}
Полное объяснение в комментарии на GitHub.
::

**Пример некорректного кода:**

```ts twoslash [composables/example.ts]
// обращение к runtime config вне композабла
const config = useRuntimeConfig()

export const useMyComposable = () => {
  // config здесь
}
```

**Пример корректного кода:**

```ts twoslash [composables/example.ts]
export const useMyComposable = () => {
  // Композабл вызывается в нужном месте жизненного цикла —
  // useRuntimeConfig здесь работает
  const config = useRuntimeConfig()

  // ...
}
```

## Автоимпорт по каталогам

Файлы из стандартных каталогов подхватываются напрямую:

- `app/components/` — [компоненты Vue](/docs/4.x/directory-structure/app/components).
- `app/composables/` — [композаблы](/docs/4.x/directory-structure/app/composables).
- `app/utils/` — утилиты.

:link-example{to="/docs/4.x/examples/features/auto-imports"}

::warning
**Автоимпортированные `ref` и `computed` не разворачиваются в `<template>` компонента.** :br
Так устроен Vue для ref не верхнего уровня шаблона. Подробнее — [документация Vue](https://vuejs.org/guide/essentials/reactivity-fundamentals#caveat-when-unwrapping-in-templates).
::

### Явные импорты

Все автоимпорты доступны через алиас `#imports`:

<!-- TODO:twoslash: Twoslash does not support tsconfig paths yet -->

```vue
<script setup lang="ts">
import { computed, ref } from '#imports'

const count = ref(1)
const double = computed(() => count.value * 2)
</script>
```

### Отключение автоимпорта

Чтобы отключить автоимпорт композаблов и утилит, задайте `imports.autoImport: false` в `nuxt.config`.

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  imports: {
    autoImport: false,
  },
})
```

После этого автоимпорта нет, но можно использовать [явные импорты](/docs/4.x/guide/concepts/auto-imports#explicit-imports) из `#imports`.

### Частичное отключение

Чтобы оставить автоимпорт фреймворка (`ref` и т.д.), но отключить сканирование своего кода, задайте `imports.scan: false`:

```ts
export default defineNuxtConfig({
  imports: {
    scan: false,
  },
})
```

Тогда:
- `ref`, `computed`, `watch` и т.п. по-прежнему без ручного импорта.
- Свои композаблы нужно импортировать вручную.

::warning
**Ограничения:** со слоями придётся явно импортировать композаблы из каждого слоя. Ломается переопределение через слои. Включая `imports.scan: false`, учитывайте это.
::

## Автоимпорт компонентов

Компоненты из `~/components` подключаются отдельно от композаблов и утилит.

:read-more{to="/docs/4.x/directory-structure/app/components"}

Чтобы отключить автоимпорт из своего `~/components`, задайте `components.dirs: []` (модули могут всё равно добавлять компоненты).

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  components: {
    dirs: [],
  },
})
```

## Автоимпорт из npm-пакетов

Nuxt умеет автоимпортировать и из сторонних пакетов.

::tip
Если есть официальный модуль пакета, он часто уже настроил автоимпорт.
::

Пример: `useI18n` из `vue-i18n`:

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  imports: {
    presets: [
      {
        from: 'vue-i18n',
        imports: ['useI18n'],
      },
    ],
  },
})
```
