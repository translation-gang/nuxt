## Что такое MCP?

MCP (Model Context Protocol) — стандартный протокол доступа ИИ-ассистентов к внешним данным и инструментам. У Nuxt есть MCP-сервер: Claude Code, Cursor, Windsurf и др. могут читать документацию, посты блога и гайды по деплою.

Сервер отдаёт структурированный доступ к документации Nuxt.

## Ресурсы

Для обзора доступны:

- **`resource://nuxt-com/documentation-pages`** — все страницы документации (по умолчанию v4.x)
- **`resource://nuxt-com/blog-posts`** — посты блога, релизы, туториалы
- **`resource://nuxt-com/deploy-providers`** — провайдеры деплоя и хостинг

В Claude Code к ресурсам обращаются через `@`.

## Инструменты

### Документация

- **`list_documentation_pages`** — список страниц с категориями; фильтр версий (3.x, 4.x, all)
- **`get_documentation_page`** — содержимое страницы по пути
- **`get_getting_started_guide`** — гайд «с чего начать» для версии

### Блог

- **`list_blog_posts`** — список постов с датами, категориями, тегами
- **`get_blog_post`** — пост по пути

### Деплой

- **`list_deploy_providers`** — провайдеры и платформы хостинга
- **`get_deploy_provider`** — детали и инструкции по пути

## Подсказки (prompts)

- **`find_documentation_for_topic`** — найти документацию по теме
- **`deployment_guide`** — инструкции по хостингу
- **`migration_help`** — миграция между версиями Nuxt

В Claude Code — через `/`.

## Настройка

MCP-сервер Nuxt использует HTTP-транспорт.

### ChatGPT

::note{icon="i-lucide-info"}
**Кастомные коннекторы MCP в ChatGPT** — для Pro и Plus в веб-версии.
::

1. **Режим разработчика:** Настройки → Connectors → Advanced settings → Developer mode
2. Откройте настройки ChatGPT
3. **Connectors → Create:** имя `Nuxt`, URL `https://nuxt.com/mcp`, аутентификация `None`
4. **Create**

Коннектор появится в композере в режиме разработчика.

### Claude Code

::note{icon="i-lucide-info"}
**Нужен установленный Claude Code** — [документация Anthropic](https://code.claude.com/docs/en/quickstart).
::

```bash
claude mcp add --transport http nuxt-remote https://nuxt.com/mcp
```

### Claude Desktop

1. Настройки → Developer → Edit Config
2. В `claude_desktop_config.json`:

```json [claude_desktop_config.json]
{
  "mcpServers": {
    "nuxt": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "https://nuxt.com/mcp"
      ]
    }
  }
}
```

3. Перезапустите Claude Desktop.

### Visual Studio Code

::note{icon="i-lucide-info"}
Нужны [GitHub Copilot](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot) и [Copilot Chat](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat).
::

Command Palette → Preferences: Open Workspace Settings (JSON) → `.vscode/mcp.json`:

```json [.vscode/mcp.json]
{
  "servers": {
    "nuxt": {
      "type": "http",
      "url": "https://nuxt.com/mcp"
    }
  }
}
```

### GitHub Copilot Agent

::note{icon="i-lucide-info"}
Нужны права администратора репозитория для MCP у coding agent.
::

Settings → Code & automation → Copilot → Coding agent → MCP:

```json
{
  "mcpServers": {
    "nuxt": {
      "type": "http",
      "url": "https://nuxt.com/mcp",
      "tools": ["*"]
    }
  }
}
```

Проверка: issue → Copilot → PR → View session → … → Copilot → Start MCP Servers.

[Extend coding agent with MCP](https://docs.github.com/en/copilot/how-tos/use-copilot-agents/coding-agent/extend-coding-agent-with-mcp)

### Windsurf

{
  "mcpServers": {
    "nuxt": {
      "type": "http",
      "url": "https://nuxt.com/mcp"

### Zed

Settings → JSON → `context_servers`:

```json [.config/zed/settings.json]
{
  "context_servers": {
    "nuxt": {
      "source": "custom",
      "command": "npx",
      "args": ["mcp-remote", "https://nuxt.com/mcp"],
      "env": {}
    }
  }
}
```

### Opencode

В корне `opencode.json`:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "nuxt": {
      "type": "remote",
      "url": "https://nuxt.com/mcp",
      "enabled": true
    }
  }
}
```
