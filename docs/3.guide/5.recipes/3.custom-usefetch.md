Часто фронт на Nuxt ходит во внешний API и хочется общие настройки для всех запросов.

[`$fetch`](/docs/4.x/api/utils/dollarfetch) (им пользуется [`useFetch`](/docs/4.x/api/composables/use-fetch)) **намеренно** не настраивается глобально: поведение запросов остаётся предсказуемым, модули могут опираться на ядро.

Зато можно завести **свой** инстанс для своего API (или несколько — для разных API).

## Свой `$fetch`

Создадим инстанс в [плагине Nuxt](/docs/4.x/directory-structure/app/plugins).

::note
`$fetch` — настроенный [ofetch](https://github.com/unjs/ofetch): базовый URL сервера Nuxt и прямые вызовы при SSR без лишнего HTTP.
::

Допустим:
- API: https://api.nuxt.com
- JWT в сессии через [nuxt-auth-utils](https://github.com/atinux/nuxt-auth-utils)
- при ответе `401` — редирект на `/login`

```ts [app/plugins/api.ts]
export default defineNuxtPlugin((nuxtApp) => {
  const { session } = useUserSession()

  const api = $fetch.create({
    baseURL: 'https://api.nuxt.com',
    onRequest ({ request, options, error }) {
      if (session.value?.token) {
        // note that this relies on ofetch >= 1.4.0 - you may need to refresh your lockfile
        options.headers.set('Authorization', `Bearer ${session.value?.token}`)
      }
    },
    async onResponseError ({ response }) {
      if (response.status === 401) {
        await nuxtApp.runWithContext(() => navigateTo('/login'))
      }
    },
  })

  // Expose to useNuxtApp().$api
  return {
    provide: {
      api,
    },
  }
})
```

`$api` доступен из `useNuxtApp()`:

```vue [app/app.vue]
<script setup>
const { $api } = useNuxtApp()
const { data: modules } = await useAsyncData('modules', () => $api('/modules'))
</script>
```

::callout
Обёртка в [`useAsyncData`](/docs/4.x/api/composables/use-async-data) **избегает двойной загрузки** при SSR (сервер + клиент при гидратации).
::

## Свой `useFetch` / `useAsyncData`

Композабл `useAPI` поверх `useAsyncData` + `$api`:

```ts [app/composables/useAPI.ts]
import type { UseFetchOptions } from 'nuxt/app'

export function useAPI<T> (
  url: string | (() => string),
  options?: UseFetchOptions<T>,
) {
  return useFetch(url, {
    ...options,
    $fetch: useNuxtApp().$api as typeof $fetch,
  })
}
```

Использование:

```vue [app/app.vue]
<script setup>
const { data: modules } = await useAPI('/modules')
</script>
```

Тип ошибки:

```ts
import type { FetchError } from 'ofetch'
import type { UseFetchOptions } from 'nuxt/app'

interface CustomError {
  message: string
  status: number
}

export function useAPI<T> (
  url: string | (() => string),
  options?: UseFetchOptions<T>,
) {
  return useFetch<T, FetchError<CustomError>>(url, {
    ...options,
    $fetch: useNuxtApp().$api,
  })
}
```

::note
Тот же подход подходит для кастомного `useAsyncData`.
