## Добавление маршрутов

Структура маршрутов следует за файлами в [app/pages](/docs/4.x/directory-structure/app/pages). Поскольку используется [vue-router](https://router.vuejs.org), есть несколько способов задать свои маршруты.

### Конфигурация роутера

Через [опции роутера](/docs/4.x/guide/recipes/custom-routing#router-options) можно заменить или дополнить список маршрутов функцией `(routes) => customRoutes`.

Если вернуть `null` или `undefined`, Nuxt оставит маршруты по умолчанию (удобно, если нужно только менять входной массив).

```ts [router.options.ts]
import type { RouterConfig } from '@nuxt/schema'

export default {
  // https://router.vuejs.org/api/interfaces/routeroptions#routes
  routes: _routes => [
    {
      name: 'home',
      path: '/',
      component: () => import('~/pages/home.vue'),
    },
  ],
} satisfies RouterConfig
```

::note
Маршруты, возвращённые из `routes`, Nuxt **не** обогащает метаданными из `definePageMeta` указанного компонента. Нужны метаданные — используйте хук `pages:extend` ([время сборки](/docs/4.x/api/advanced/hooks#nuxt-hooks-build-time)).
::

### Хук Pages

Хук `pages:extend` позволяет добавлять, менять и удалять страницы из отсканированных маршрутов.

Например, убрать маршруты для `.ts` файлов:

```ts [nuxt.config.ts]
import type { NuxtPage } from '@nuxt/schema'

export default defineNuxtConfig({
  hooks: {
    'pages:extend' (pages) {
      // add a route
      pages.push({
        name: 'profile',
        path: '/profile',
        file: '~/extra-pages/profile.vue',
      })

      // remove routes
      function removePagesMatching (pattern: RegExp, pages: NuxtPage[] = []) {
        const pagesToRemove: NuxtPage[] = []
        for (const page of pages) {
          if (page.file && pattern.test(page.file)) {
            pagesToRemove.push(page)
          } else {
            removePagesMatching(pattern, page.children)
          }
        }
        for (const page of pagesToRemove) {
          pages.splice(pages.indexOf(page), 1)
        }
      }
      removePagesMatching(/\.ts$/, pages)
    },
  },
})
```

### Модуль Nuxt

Набор страниц под одну фичу часто удобнее оформить [модулем Nuxt](/modules).

В [Nuxt Kit](/docs/4.x/guide/going-further/kit) для маршрутов есть:
- [`extendPages`](/docs/4.x/api/kit/pages#extendpages) `(pages) => void`
- [`extendRouteRules`](/docs/4.x/api/kit/pages#extendrouterules) `(route, rule, options)`

## Опции роутера

Помимо настроек [`vue-router`](https://router.vuejs.org/api/interfaces/routeroptions), Nuxt добавляет [свои опции](/docs/4.x/api/nuxt-config#router).

### Файл `router.options`

Рекомендуемый способ задать [опции роутера](/docs/4.x/api/nuxt-config#router).

```ts [app/router.options.ts]
import type { RouterConfig } from '@nuxt/schema'

export default {
} satisfies RouterConfig
```

Дополнительные файлы опций подключаются хуком `pages:routerOptions`; поздние элементы массива перекрывают ранние.

::callout
Файл из этого хука включает файловую маршрутизацию, если не указан `optional` — тогда файл учитывается только при уже включённой файловой маршрутизации.
::

```ts [nuxt.config.ts]
import { createResolver } from '@nuxt/kit'

export default defineNuxtConfig({
  hooks: {
    'pages:routerOptions' ({ files }) {
      const resolver = createResolver(import.meta.url)
      // add a route
      files.push({
        path: resolver.resolve('./runtime/router-options'),
        optional: true,
      })
    },
  },
})
```

### Через `nuxt.config`

**Важно:** из JSON-сериализуемых [полей](/docs/4.x/api/nuxt-config#router) доступны только:

- `linkActiveClass`
- `linkExactActiveClass`
- `end`
- `sensitive`
- `strict`
- `hashMode`
- `scrollBehaviorType`

```ts [nuxt.config]
export default defineNuxtConfig({
  router: {
    options: {},
  },
})
```

### Hash mode (SPA)

История с хешем в SPA: опция `hashMode` в [router](/docs/4.x/api/nuxt-config#router). URL после `#` **не уходит на сервер**, **SSR не используется**.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  ssr: false,
  router: {
    options: {
      hashMode: true,
    },
  },
})
```

### Прокрутка к якорям

При `scrollBehaviorType: 'smooth'` переход на URL с хешем (например `https://example.com/blog/my-article#comments`) плавно прокручивает к якорю.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  router: {
    options: {
      scrollBehaviorType: 'smooth',
    },
  },
})
```

#### Свой history (продвинуто)

Режим history можно задать функцией `(base) => history | null`. `null`/`undefined` — стандартный history Nuxt.

```ts [router.options.ts]
import type { RouterConfig } from '@nuxt/schema'
import { createMemoryHistory } from 'vue-router'

export default {
  // https://router.vuejs.org/api/interfaces/routeroptions
  history: base => import.meta.client ? createMemoryHistory(base) : null, /* default */
} satisfies RouterConfig
```
