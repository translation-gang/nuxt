---

Модули Nuxt закрывают большинство задач, но иногда удобнее подключить именно плагин Vite.

Установите плагин — в примере ниже это `@rollup/plugin-yaml`:

::code-group{sync="pm"}

  ```bash [npm]
  npm install @rollup/plugin-yaml
  ```

  ```bash [yarn]
  yarn add @rollup/plugin-yaml
  ```

  ```bash [pnpm]
  pnpm add @rollup/plugin-yaml
  ```

  ```bash [bun]
  bun add @rollup/plugin-yaml
  ```

::

Импортируйте плагин и добавьте его в [`nuxt.config.ts`](/docs/3.x/directory-structure/nuxt-config):

```ts [nuxt.config.ts]
import yaml from '@rollup/plugin-yaml'

export default defineNuxtConfig({
  vite: {
    plugins: [
      yaml()
    ]
  }
})
```

После установки и настройки плагина YAML можно импортировать в коде как модуль.

Например, в `data/hello.yaml` лежит приветствие — его можно подключить в компоненте:

::code-group

```yaml [data/hello.yaml]
greeting: "Привет, Nuxt и Vite!"
```

```vue [components/Hello.vue]
<script setup>
import config from '~/data/hello.yaml'
</script>

<template>
  <h1>{{ config.greeting }}</h1>
</template>
```

::

::tip
Для типовых задач Nuxt сначала проверьте [модули](/docs/3.x/guide/modules/getting-started): они задают соглашения проекта и обновляются вместе с экосистемой. Плагин Vite подключают, когда нужен низкоуровневый контроль над сборкой.
::
