Модули Nuxt покрывают много сценариев, но иногда проще взять конкретный плагин Vite.

Установим плагин, в примере — `@rollup/plugin-yaml`:

::code-group{sync="pm"}

  ```bash [npm]
  npm install @rollup/plugin-yaml
  ```

  ```bash [yarn]
  yarn add @rollup/plugin-yaml
  ```

  ```bash [pnpm]
  pnpm add @rollup/plugin-yaml
  ```

  ```bash [bun]
  bun add @rollup/plugin-yaml
  ```

  ```bash [deno]
  deno add npm:@rollup/plugin-yaml
  ```

::

Импортируйте и добавьте в [`nuxt.config.ts`](/docs/4.x/directory-structure/nuxt-config):

```ts [nuxt.config.ts]
import yaml from '@rollup/plugin-yaml'

export default defineNuxtConfig({
  vite: {
    plugins: [
      yaml(),
    ],
  },
})
```

После этого YAML можно импортировать в проекте.

Например, `config.yaml` и компонент:

::code-group

```yaml [data/hello.yaml]
greeting: "Hello, Nuxt with Vite!"
```

```vue [app/components/Hello.vue]
<script setup>
import config from '~/data/hello.yaml'
</script>

<template>
  <h1>{{ config.greeting }}</h1>
</template>
```

::

## Плагины Vite в модулях Nuxt

В модуле используйте [`addVitePlugin`](/docs/4.x/api/kit/builder#addviteplugin):

```ts [modules/my-module.ts]
import { addVitePlugin, defineNuxtModule } from '@nuxt/kit'
import yaml from '@rollup/plugin-yaml'

export default defineNuxtModule({
  setup () {
    addVitePlugin(yaml())
  },
})
```

В Nuxt 5+ для привязки к окружению — `applyToEnvironment()`:

```ts [modules/my-module.ts]
import { addVitePlugin, defineNuxtModule } from '@nuxt/kit'

export default defineNuxtModule({
  setup () {
    addVitePlugin(() => ({
      name: 'my-client-plugin',
      applyToEnvironment (environment) {
        return environment.name === 'client'
      },
      // Plugin configuration
    }))
  },
})
```

::important
Если нужен уже смерженный конфиг Vite, используйте хуки `config` / `configResolved` **внутри** плагина Vite, а не `vite:extend`, `vite:extendConfig` и `vite:configResolved` в Nuxt.
::
