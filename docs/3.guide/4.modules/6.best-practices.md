Модули дают большую силу и ответственность. Ниже — практики, чтобы приложения оставались быстрыми, а DX — приятным.

## Асинхронный setup

Модули могут быть асинхронными (запрос к API и т.д.). Имейте в виду: Nuxt ждёт завершения setup модуля перед следующим модулем и перед dev-сервером/сборкой. Тяжёлую работу лучше откладывать на хуки.

::warning
Если setup модуля длится дольше **1 секунды**, Nuxt выводит предупреждение.
::

## Префиксы в API

Явный префикс для конфигурации, плагинов, API, композаблов, компонентов и серверных маршрутов снижает конфликты с другими модулями, внутренностями Nuxt и кодом пользователя.

Логично брать имя модуля, например `nuxt-foo`:

| Тип | ❌ Так не надо | ✅ Лучше |
| --- | --- | --- |
| Компоненты | `<Button>`, `<Modal>` | `<FooButton>`, `<FooModal>` |
| Композаблы | `useData()`, `useModal()` | `useFooData()`, `useFooModal()` |
| Серверные маршруты | `/api/track`, `/api/data` | `/api/_foo/track`, `/api/_foo/data` |

### Серверные маршруты

Особенно важно: пути вроде `/api/auth`, `/api/login`, `/api/user` часто заняты приложением.

Уникальный префикс по имени модуля:

- `/api/_foo/...` (подчёркивание в префиксе)
- `/_foo/...` для не-API маршрутов

## Хуки жизненного цикла

Разовые задачи (генерация конфигов, БД, установка зависимостей) лучше вешать на lifecycle-хуки модуля, а не выполнять целиком в `setup`.

```ts
import { addServerHandler, defineNuxtModule } from 'nuxt/kit'
import semver from 'semver'

export default defineNuxtModule({
  meta: {
    name: 'my-database-module',
    version: '1.0.0',
  },
  async onInstall (nuxt) {
    // One-time setup: create database schema, generate config files, etc.
    await generateDatabaseConfig(nuxt.options.rootDir)
  },
  async onUpgrade (nuxt, options, previousVersion) {
    // Handle version-specific migrations
    if (semver.lt(previousVersion, '1.0.0')) {
      await migrateLegacyData()
    }
  },
  setup (options, nuxt) {
    // Regular setup logic that runs on every build
    addServerHandler({ /* ... */ })
  },
})
```

Так меньше лишней работы на каждой сборке. Подробнее — [хуки установки и обновления модуля](/docs/4.x/api/kit/modules#using-lifecycle-hooks-for-module-installation-and-upgrade).

## Версии Nuxt

Nuxt, Kit и связанный стек рассчитаны на совместимость вперёд и назад.

Пишите «X for Nuxt», а не «X for Nuxt 3», и задавайте ограничения через `meta.compatibility`.
