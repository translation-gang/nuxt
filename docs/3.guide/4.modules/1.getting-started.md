## Создать модуль

Рекомендуем начать с [стартового шаблона](https://github.com/nuxt/starter/tree/module):

::code-group{sync="pm"}

```bash [npm]
npm create nuxt -- -t module my-module
```

```bash [yarn]
yarn create nuxt -t module my-module
```

```bash [pnpm]
pnpm create nuxt -t module my-module
```

```bash [bun]
bun create nuxt --template=module my-module
```
::

Появится проект `my-module` с заготовкой для разработки и публикации модуля.

**Дальше:**

1. Откройте `my-module` в IDE
2. Установите зависимости любимым менеджером пакетов
3. Подготовьте локальные файлы: `npm run dev:prepare`
4. Продолжайте по этой странице и остальным разделам про модули

### Разработка

Исходники модуля — в каталоге `src`; для проверки нужно приложение Nuxt — это `playground`, уже подключённый к модулю.

С playground можно работать как с обычным Nuxt:

- `npm run dev` — dev-сервер с перезагрузкой при изменениях в `src`
- `npm run dev:build` — сборка

::note
Любые команды `nuxt` можно запускать для `playground` (например `nuxt <COMMAND> playground`). Дополнительные скрипты `dev:*` в `package.json` по желанию.
::

### Тесты

В шаблоне уже есть:

- линтер [ESLint](https://eslint.org) — `npm run lint`
- [Vitest](https://vitest.dev) — `npm run test` или `npm run test:watch`

::tip
Стратегию тестов можно расширять под задачи модуля.
::

### Сборка модуля

Сборщик — [`@nuxt/module-builder`](https://github.com/nuxt/module-builder#readme): без настройки с вашей стороны, с TypeScript и корректной упаковкой для других приложений Nuxt.

Команда: `npm run prepack`.

::tip
Отдельная сборка нужна не всегда: в разработке собирает playground, при релизе — скрипт публикации.
::

### Публикация в npm

::important
Перед публикацией заведите аккаунт на [npmjs.com](https://www.npmjs.com) и выполните локально `npm login`.
::

Можно публиковать вручную через версию и `npm publish`, но в шаблоне есть скрипт релиза, который проверяет проект перед выкладкой.

Закоммитьте изменения (удобно [Conventional Commits](https://www.conventionalcommits.org) для авто-версии и changelog), затем `npm run release`.

Скрипт релиза:

- прогоняет проверки:
  - линтер (`npm run lint`)
  - тесты (`npm run test`)
  - сборку (`npm run prepack`)
- при успехе:
  - поднимает версию и обновляет changelog по Conventional Commits
  - публикует в npm (модуль собирается ещё раз с новой версией)
  - пушит git-тег в `origin`

::tip
Скрипт `release` в `package.json` при необходимости подстройте под процесс команды.
::
