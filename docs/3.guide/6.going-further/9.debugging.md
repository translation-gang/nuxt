## Sourcemaps

Sourcemaps для серверной сборки включены по умолчанию; для клиента — в dev. Можно настроить явно:

```ts
export default defineNuxtConfig({
  sourcemap: {
    server: true,
    client: true,
  },
})
```

## Отладка через Node Inspector

Для отладки серверной части Nuxt используйте [Node inspector](https://nodejs.org/en/learn/getting-started/debugging):

```bash
nuxt dev --inspect
```

Nuxt запустится в dev с включённым отладчиком. В Chrome DevTools появится иконка Node.js для подключения.

::important
Процессы Node.js и Chrome должны работать на одной платформе. В Docker так отлаживать нельзя.
::

## Отладка в IDE

Приложение Nuxt можно отлаживать прямо в IDE. В VS Code добавьте конфигурацию запуска (Chrome / Node) с путём к `nuxt dev` и при необходимости — к браузеру. Подробнее — в [документации VS Code по отладке](https://code.visualstudio.com/docs/debugtest/debugging#_launch-configurations).
