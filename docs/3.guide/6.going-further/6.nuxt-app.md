## Контекст Nuxt

Многим композаблам и утилитам нужен экземпляр Nuxt; он есть не везде — на каждый запрос создаётся новый контекст.

Сейчас контекст Nuxt доступен в [плагинах](/docs/4.x/directory-structure/app/plugins), [хуках Nuxt](/docs/4.x/guide/going-further/hooks), [middleware Nuxt](/docs/4.x/directory-structure/app/middleware) (если обёрнуты в `defineNuxtRouteMiddleware`) и в [функциях setup](https://vuejs.org/api/composition-api-setup) страниц и компонентов.

Если композабл вызван без контекста, возможна ошибка вида «A composable that requires access to the Nuxt instance was called outside of a plugin, Nuxt hook, Nuxt middleware, or Vue setup function». Тогда можно явно выполнить код в контексте через [`nuxtApp.runWithContext`](/docs/4.x/api/composables/use-nuxt-app#runwithcontext).

## Доступ к NuxtApp

В композаблах, плагинах и компонентах используйте [`useNuxtApp()`](/docs/4.x/api/composables/use-nuxt-app):

```ts [app/composables/useMyComposable.ts]
export function useMyComposable () {
  const nuxtApp = useNuxtApp()
  // access runtime nuxt app instance
}
```

Если `nuxtApp` не всегда нужен или нужна проверка наличия контекста, вместо [`useNuxtApp`](/docs/4.x/api/composables/use-nuxt-app) (который бросает исключение) вызывайте [`tryUseNuxtApp`](/docs/4.x/api/composables/use-nuxt-app#tryusenuxtapp).

В плагинах `nuxtApp` передаётся первым аргументом.

:read-more{title="Плагины" to="/docs/4.x/directory-structure/app/plugins"}

## provide и хелперы

Хелперы для всего приложения обычно регистрируют в плагине Nuxt.

```ts
const nuxtApp = useNuxtApp()
nuxtApp.provide('hello', name => `Hello ${name}!`)

console.log(nuxtApp.$hello('name')) // Prints "Hello name!"
```

::read-more{to="/docs/4.x/directory-structure/app/plugins#providing-helpers"}
Хелперы можно вернуть из плагина в поле `provide`.
::

::read-more{to="https://v2.nuxt.com/docs/directory-structure/plugins/#inject-in-root--context" target="_blank"}
В плагинах Nuxt 2 это соответствовало функции **inject**.
::
