Часть возможностей Nuxt подключается явно или отключается по необходимости.

## `features`

### devLogs

Стриминг серверных логов на клиент в разработке. Обрабатываются в хуке `dev:ssr-logs`.

По умолчанию включено в dev (если не активен test mode).

При `silent` логи не выводятся в консоль браузера.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  features: {
    devLogs: true,
  },
})
```

### inlineStyles

Встраивание стилей в HTML при рендере. Сейчас только со сборщиком Vite.

Можно передать функцию `(path) => boolean`, решающую, инлайнить ли стили для данного компонента.

По умолчанию `(id) => id.includes('.vue')`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  features: {
    inlineStyles: false, // or a function to determine inlining
  },
})
```

### noScripts

Отключить вывод скриптов Nuxt и подсказок к JS-ресурсам. Дополнительно настраивается в `routeRules`.

Точечно — тоже через `routeRules`.

`true` или `'production'` — без JS только в production; `'all'` — и в dev, и в production.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  features: {
    noScripts: true, // or 'production' | 'all' | false
  },
})
```

## `future`

Пространство имён `future` — ранний opt-in к тем настройкам, которые станут умолчанию в будущей (возможно major) версии.

### compatibilityVersion

Ранний доступ к поведению и флагам следующих версий Nuxt.

`compatibilityVersion: 5` переключает ряд значений по умолчанию на поведение Nuxt v5, в том числе [Vite Environment API](/docs/4.x/guide/going-further/experimental-features#viteenvironmentapi).

```ts
export default defineNuxtConfig({
  future: {
    compatibilityVersion: 5,
  },
})
```

### multiApp

Экспериментальная поддержка нескольких приложений. Статус — в [issue #21635](https://github.com/nuxt/nuxt/issues/21635).

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  future: {
    multiApp: true,
  },
})
```

### typescriptBundlerResolution

Режим разрешения модулей TypeScript **Bundler** — рекомендуемый для Nuxt и [Vite](https://vite.dev/guide/performance#reduce-resolve-operations).

Лучше типы у библиотек с полем `exports` в `package.json`.

История — в [PR TypeScript](https://github.com/microsoft/TypeScript/pull/51669).

`false` возвращает старый режим **Node** (прежнее умолчание TypeScript).

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  future: {
    typescriptBundlerResolution: false,
  },
})
```
