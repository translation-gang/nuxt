События помогают слабо связать части приложения: разные подписчики не зависят друг от друга. Например, при отправке заказа можно не вызывать почтовый код из обработчика заказа, а породить событие — отдельный слушатель отправит письмо.

События Nuxt построены на [unjs/hookable](https://github.com/unjs/hookable) — той же библиотеке, что и встроенные хуки Nuxt.

## События и слушатели

Свой слушатель регистрируется методом `hook`:

```ts
const nuxtApp = useNuxtApp()

nuxtApp.hook('app:user:registered', (payload) => {
  console.log('A new user has registered!', payload)
})
```

Чтобы вызвать событие и оповестить слушателей, используйте `callHook`:

```ts
const nuxtApp = useNuxtApp()

await nuxtApp.callHook('app:user:registered', {
  id: 1,
  name: 'John Doe',
})
```

Объект payload можно использовать для двусторонней связи: он передаётся по ссылке, слушатель может изменить поля, и эмиттер увидит результат.

```ts
const nuxtApp = useNuxtApp()

nuxtApp.hook('app:user:registered', (payload) => {
  payload.message = 'Welcome to our app!'
})

const payload = {
  id: 1,
  name: 'John Doe',
}

await nuxtApp.callHook('app:user:registered', {
  id: 1,
  name: 'John Doe',
})

// payload.message will be 'Welcome to our app!'
```

::tip
Все события можно смотреть на панели Hooks в **Nuxt DevTools**.
