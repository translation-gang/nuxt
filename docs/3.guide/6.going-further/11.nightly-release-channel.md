---
title: "Ночной канал релизов (Nightly)"
description: "Nightly-канал позволяет использовать Nuxt, собранный из последних коммитов репозитория."
---

В Nuxt каждый день попадают коммиты, доработки и исправления. Можно подключать их раньше следующего стабильного релиза.

После мержа коммита в ветку `main` [nuxt/nuxt](https://github.com/nuxt/nuxt) и **успешного прохождения тестов** через GitHub Actions публикуется ночная сборка в npm.

Эти «nightly»-релизы подходят для бета-теста новых возможностей. Метод сборки и качество те же, что у стабильных; отличие в том, что обновления нужно следить по репозиторию, и есть небольшой риск регрессий. Команда Nuxt использует этот канал для проверки перед каждым релизом.

::note
Возможности, доступные только в nightly, в документации помечены специальным значком.
::

::warning
Канал `latest` nightly сейчас следует за веткой Nuxt v4 — возможны breaking changes. Для nightly ветки 3.x используйте `"nuxt": "npm:nuxt-nightly@3x"`.
::

## Подключение

В `package.json` измените зависимость:

```diff [package.json]
{
  "devDependencies": {
--    "nuxt": "^4.0.0"
++    "nuxt": "npm:nuxt-nightly@latest"
  }
}
```

Удалите lockfile и переустановите зависимости.

## Отключение

Верните в `package.json` стабильную версию (например `"nuxt": "^4.0.0"`), удалите lockfile и переустановите зависимости.

## Nightly @nuxt/cli

Чтобы использовать последнюю версию [nuxt/cli](https://github.com/nuxt/cli):

```bash [Terminal]
npx @nuxt/cli-nightly@latest [command]
```
