---
title: "Канал ночных сборок"
description: "Ночной канал — сборки Nuxt из последних коммитов в репозитории."
---

В Nuxt каждый день появляются коммиты, улучшения и исправления. Ночной канал позволяет проверять их до очередного релиза.

После мержа в ветку `main` [nuxt/nuxt](https://github.com/nuxt/nuxt) и **успешного прохождения тестов** автоматически публикуется пакет на npm через GitHub Actions.

Такие «ночные» релизы удобны для бета-тестирования.

Сборка и качество такие же, как у стабильных релизов. Отличие — нужно чаще смотреть репозиторий на GitHub: редко регрессии могут проскочить мимо ревью и тестов. Внутри команды этот канал используют для финальной проверки перед релизом.

::note
Возможности, доступные только на ночном канале, в документации помечены предупреждением.
::

::warning
Канал `latest` сейчас следует за веткой Nuxt v4 — **высокий** риск breaking changes; действуйте осторожно. Для ночных сборок 3.x укажите `"nuxt": "npm:nuxt-nightly@3x"`.
::

## Подключение

Обновите зависимость `nuxt` в `package.json`:

```diff [package.json]
{
  "devDependencies": {
--    "nuxt": "^4.0.0"
++    "nuxt": "npm:nuxt-nightly@latest"
  }
}
```

Удалите lockfile (`package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, `bun.lock` или `bun.lockb`) и переустановите зависимости.

## Отключение

Верните стабильную версию в `package.json`:

```diff [package.json]
{
  "devDependencies": {
--    "nuxt": "npm:nuxt-nightly@latest"
++    "nuxt": "^4.0.0"
  }
}
```

Снова удалите lockfile и переустановите зависимости.

## Ночной `@nuxt/cli`

Последняя версия [nuxt/cli](https://github.com/nuxt/cli):

```bash [Terminal]
npx @nuxt/cli-nightly@latest [command]
```
