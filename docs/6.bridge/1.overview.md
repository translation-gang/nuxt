Если вы начинаете новый проект на Nuxt 3, пропустите этот раздел и перейдите к [Установке Nuxt 3](/docs/4.x/getting-started/introduction).
::

::warning
Nuxt Bridge предоставляет те же возможности, что и Nuxt 3 ([документация](/docs/4.x/guide/concepts/auto-imports)), но есть ограничения: в частности, недоступны композаблы [`useAsyncData`](/docs/4.x/api/composables/use-async-data) и [`useFetch`](/docs/4.x/api/composables/use-fetch). Подробности — далее на этой странице.
::

Bridge — слой совместимости с будущим Nuxt 3: многие новые возможности Nuxt 3 можно включить, просто установив и активировав модуль Nuxt.

С Nuxt Bridge можно подготовить проект к Nuxt 3 (почти полностью) и постепенно перейти на Nuxt 3.

## Первый шаг

### Обновите Nuxt 2

Убедитесь, что dev-сервер (`nuxt dev`) не запущен, удалите lock-файлы (`package-lock.json` и `yarn.lock`) и установите последнюю версию Nuxt 2:

```diff [package.json]
- "nuxt": "^2.16.3"
+ "nuxt": "^2.17.3"
```

Затем переустановите зависимости:

::code-group{sync="pm"}

```bash [npm]
npm install
```

```bash [yarn]
yarn install
```

```bash [pnpm]
pnpm install
```

```bash [bun]
bun install
```

```bash [deno]
deno install
```

::

::note
После установки убедитесь, что сборки для разработки и продакшена работают как ожидается, прежде чем продолжать.
::

### Установите Nuxt Bridge

Установите `@nuxt/bridge` и `nuxi` как dev-зависимости:

::code-group{sync="pm"}

```bash [npm]
npm install -D @nuxt/bridge nuxi
```

```bash [yarn]
yarn add --dev @nuxt/bridge nuxi
```

```bash [pnpm]
pnpm add -D @nuxt/bridge nuxi
```

```bash [bun]
bun add -D @nuxt/bridge nuxi
```

```bash [deno]
deno add -D npm:@nuxt/bridge npm:nuxi
```

::

### Обновите `nuxt.config`

Избегайте синтаксиса CommonJS (`module.exports`, `require`, `require.resolve`) в конфиге — он скоро будет устаревшим и неподдерживаемым.

Вместо этого используйте статический `import`, динамический `import()` и `export default`. Рекомендуется TypeScript — переименуйте файл в [`nuxt.config.ts`](/docs/4.x/directory-structure/nuxt-config).

```ts [nuxt.config.ts]
import { defineNuxtConfig } from '@nuxt/bridge'

export default defineNuxtConfig({
  bridge: false,
})
```

### Обновите команды

Команду `nuxt` нужно заменить на `nuxt2`.

```diff
{
  "scripts": {
-   "dev": "nuxt",
+   "dev": "nuxt2",
-   "build": "nuxt build",
+   "build": "nuxt2 build",
-   "start": "nuxt start",
+   "start": "nuxt2 start"
  }
}
```

Запустите здесь `nuxt2` один раз — приложение должно работать как раньше.

(Если `bridge` установлен в `false`, приложение будет вести себя без изменений, как прежде.)
