## Удалите модули

- Удалите `@nuxt/typescript-build`: Bridge даёт ту же функциональность
- Удалите `@nuxt/typescript-runtime` и `nuxt-ts`: во встроенной поддержке Nuxt 2 достаточно runtime

### Установите `bridge.typescript`

```ts
import { defineNuxtConfig } from '@nuxt/bridge'

export default defineNuxtConfig({
  bridge: {
    typescript: true,
    nitro: false, // If migration to Nitro is complete, set to true
  },
})
```

## Обновите `tsconfig.json`

При использовании TypeScript отредактируйте `tsconfig.json`, чтобы подключить автогенерируемые типы Nuxt:

```diff [tsconfig.json]
{
+ "extends": "./.nuxt/tsconfig.json",
  "compilerOptions": {
    ...
  }
}
```

::note
Файл `.nuxt/tsconfig.json` генерируется и не попадает в систему контроля версий — его нужно сгенерировать перед запуском тестов. Добавьте шаг `nuxi prepare` перед тестами, иначе возможна ошибка `TS5083: Cannot read file '~/.nuxt/tsconfig.json'`.

В современных проектах Nuxt рекомендуем [ссылки проектов TypeScript](/docs/4.x/directory-structure/tsconfig) вместо прямого `extends` из `.nuxt/tsconfig.json`.
::

::note
Помните: все опции, унаследованные из `./.nuxt/tsconfig.json`, будут перезаписаны опциями в вашем `tsconfig.json`.
Перезапись, например, `"compilerOptions.paths"` своей конфигурацией приведёт к тому, что TypeScript не учтёт разрешение модулей из `./.nuxt/tsconfig.json`. Из-за этого могут не распознаваться, например, импорты `#imports`.

Если нужно расширить опции из `./.nuxt/tsconfig.json`, используйте свойство `alias` в `nuxt.config` — `nuxi` подхватит их и дополнит `./.nuxt/tsconfig.json`.
::
