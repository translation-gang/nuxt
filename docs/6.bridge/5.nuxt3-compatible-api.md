---
title: Новый Composition API
description: Nuxt Bridge реализует композаблы, совместимые с Nuxt 3.
---

Переход с `@nuxtjs/composition-api` на API, совместимый с Nuxt 3, уменьшит объём переписывания при полной миграции на Nuxt 3.

## `ssrRef` и `shallowSsrRef`

Их заменил композабл `useState` с похожей внутренней логикой.

Отличия: нужно задать **ключ** состояния (для `ssrRef` и `shallowSsrRef` его генерировал Nuxt), и вызывать можно только внутри плагина Nuxt 3 (`defineNuxtPlugin`) или экземпляра компонента. ([`useState`](/docs/4.x/api/composables/use-state) нельзя вызывать в глобальном/окружающем контексте из‑за риска общего состояния между запросами.)

```diff
- import { ssrRef } from '@nuxtjs/composition-api'

- const ref1 = ssrRef('initialData')
- const ref2 = ssrRef(() => 'factory function')
+ const ref1 = useState('ref1-key', () => 'initialData')
+ const ref2 = useState('ref2-key', () => 'factory function')
  // accessing the state
  console.log(ref1.value)
```

Состояние ключуется — к нему можно обращаться из разных мест с тем же ключом.

Подробнее — в [документации Nuxt 3](/docs/4.x/api/composables/use-state).

## `ssrPromise`

Функция удалена; при использовании нужен другой подход. Если кейс важен — напишите в discussion.

## `onGlobalSetup`

Функция удалена; сценарии закрываются [`useNuxtApp`](/docs/4.x/api/composables/use-nuxt-app) или [`useState`](/docs/4.x/api/composables/use-state) внутри `defineNuxtPlugin`, либо произвольным кодом в `setup()` лейаута.

```diff
- import { onGlobalSetup } from '@nuxtjs/composition-api'

- export default () => {
-   onGlobalSetup(() => {
+ export default defineNuxtPlugin((nuxtApp) => {
+   nuxtApp.hook('vue:setup', () => {
      // ...
    })
- }
+ })
```

## `useStore`

Экземпляр Vuex: `useNuxtApp().$store`.

```diff
- import { useStore } from '@nuxtjs/composition-api`
+ const { $store } = useNuxtApp()
```

## `useContext` и `withContext`

Инжектированные хелперы — через `useNuxtApp`.

```diff
- import { useContext } from '@nuxtjs/composition-api`
+ const { $axios } = useNuxtApp()
```

::note
`useNuxtApp()` также отдаёт `nuxt2Context` с теми же полями, что и контекст Nuxt 2, но **не** опирайтесь на это напрямую — в Nuxt 3 этого не будет. Ищите другой способ получить нужное (или заведите feature request / discussion).
::

## `wrapProperty`

Хелпер убран; можно заменить так:

```ts
import { computed, getCurrentInstance } from 'vue'

const wrapProperty = (property: string, makeComputed = true) => () => {
  const vm = getCurrentInstance().proxy
  return makeComputed ? computed(() => vm[property]) : vm[property]
}
```

## `useAsync` и `useFetch`

Их заменяют `useLazyAsyncData` и `useLazyFetch` — см. [документацию Nuxt 3](/docs/4.x/getting-started/data-fetching). Как и старые композаблы `@nuxtjs/composition-api`, они **не** блокируют навигацию на клиенте (отсюда «lazy» в названии).

::important
API полностью другое, несмотря на похожие имена. Не пытайтесь менять значения переменных снаружи композабла (как могли делать со старым `useFetch`).
::

::warning
Для `useLazyFetch` должен быть настроен [Nitro](/docs/4.x/bridge/nitro).
::

Миграция с `useAsync`:

```diff
<script setup>
- import { useAsync } from '@nuxtjs/composition-api'
- const posts = useAsync(() => $fetch('/api/posts'))
+ const { data: posts } = useLazyAsyncData('posts', () => $fetch('/api/posts'))
+ // or, more simply!
+ const { data: posts } = useLazyFetch('/api/posts')
</script>
```

Миграция с `useFetch`:

```diff
<script setup>
- import { useFetch } from '@nuxtjs/composition-api'
- const posts = ref([])
- const { fetch } = useFetch(() => { posts.value = await $fetch('/api/posts') })
+ const { data: posts, refresh } = useLazyAsyncData('posts', () => $fetch('/api/posts'))
+ // or, more simply!
+ const { data: posts, refresh } = useLazyFetch('/api/posts')
  function updatePosts() {
-   return fetch()
+   return refresh()
  }
</script>
```

### `useMeta`

Для работы с `vue-meta` используйте `useNuxt2Meta` — работает в Nuxt Bridge (не в Nuxt 3) и даёт управление мета-тегами в духе `vue-meta`.

```diff
<script setup>
- import { useMeta } from '@nuxtjs/composition-api'
  useNuxt2Meta({
    title: 'My Nuxt App',
  })
</script>
```

Можно передавать computed или ref — значения обновятся реактивно:

```vue
<script setup>
const title = ref('my title')
useNuxt2Meta({
  title,
})
title.value = 'new title'
</script>
```

::note
Не смешивайте `useNuxt2Meta()` и Options API `head()` в одном компоненте — поведение может быть непредсказуемым.
::

В Nuxt Bridge есть реализация меты в стиле Nuxt 3 — композабл [`useHead`](/docs/4.x/api/composables/use-head).

```diff
<script setup>
- import { useMeta } from '@nuxtjs/composition-api'
  useHead({
    title: 'My Nuxt App',
  })
</script>
```

Нужно явно включить в `nuxt.config`:

```js
import { defineNuxtConfig } from '@nuxt/bridge'

export default defineNuxtConfig({
  bridge: {
    meta: true,
  },
})
```

[`useHead`](/docs/4.x/api/composables/use-head) использует `@unhead/vue` (не `vue-meta`) для `<head>`. Не смешивайте нативные свойства Nuxt 2 `head()` и [`useHead`](/docs/4.x/api/composables/use-head) — возможны конфликты.

Подробнее — [документация Nuxt 3](/docs/4.x/getting-started/seo-meta).

### Явные импорты

Nuxt экспортирует каждый автоимпорт через алиас `#imports` для явного импорта при необходимости:

```vue
<script setup lang="ts">
import { computed, ref } from '#imports'

const count = ref(1)
const double = computed(() => count.value * 2)
</script>
```

### Отключение автоимпортов

Чтобы отключить автоимпорт композаблов и утилит, в `nuxt.config` установите `imports.autoImport` в `false`.

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  imports: {
    autoImport: false,
  },
})
```

Автоимпорты отключатся полностью, но останутся [явные импорты](/docs/4.x/bridge/nuxt3-compatible-api#explicit-imports) из `#imports`.
