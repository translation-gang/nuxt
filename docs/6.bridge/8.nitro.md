title: Nitro
description: 'Подключите Nitro к приложению Nuxt 2 через Nuxt Bridge.'
---

## Удалите модули

- Удалите `@nuxt/nitro`: ту же функциональность добавляет Bridge

## Обновите конфиг

```ts [nuxt.config.ts]
import { defineNuxtConfig } from '@nuxt/bridge'

export default defineNuxtConfig({
  bridge: {
    nitro: true,
  },
})
```

## Обновите скрипты

В `package.json` обновите скрипты: сборка Nuxt теперь выдаёт сервер Nitro.

### Установите Nuxi

Установите `nuxi` как dev-зависимость:

::code-group{sync="pm"}

```bash [npm]
npm install -D nuxi
```

```bash [yarn]
yarn add --dev nuxi
```

```bash [pnpm]
pnpm add -D nuxi
```

```bash [bun]
bun add -D nuxi
```

```bash [deno]
deno add -D npm:nuxi
```

::

### Nuxi

В Nuxt 3 появилась CLI [`nuxi`](/docs/4.x/api/commands/add). Обновите скрипты для лучшей поддержки Bridge:

```diff
{
  "scripts": {
-   "dev": "nuxt",
+   "dev": "nuxi dev",
-   "build": "nuxt build",
+   "build": "nuxi build",
-   "start": "nuxt start",
+   "start": "nuxi preview"
  }
}
```

::tip
Если `nitro: false`, используйте команду `nuxt2`.
::

### Режим static

Если в `nuxt.config` задано `target: 'static'`, скрипт сборки должен быть `nuxi generate`.

```json [package.json]
{
  "scripts": {
    "build": "nuxi generate"
  }
}
```

### Серверный target

В остальных случаях используйте `nuxi build`.

```json [package.json]
{
  "scripts": {
    "build": "nuxi build",
    "start": "nuxi preview"
  }
}
```

## Исключите папку сборки Nitro из Git

Добавьте каталог `.output` в `.gitignore`.

## Проверка

✔️ Запустите `nuxi dev` и `nuxi build` (или `nuxi generate`) и убедитесь, что всё проходит успешно.
