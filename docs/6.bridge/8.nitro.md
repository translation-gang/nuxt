title: Nitro
description: 'Активируйте Nitro для вашего приложения Nuxt 2 с помощью Nuxt Bridge.'
---

После Nitro логично перейти к [Vite](/docs/bridge/vite) или сразу к [настройке флагов Bridge](/docs/bridge/configuration).

## Удалите модули

- Удалите модуль `@nuxt/nitro`: Bridge имеет ту же функциональность

## Обновите конфигурацию

```ts [nuxt.config.ts]
import { defineNuxtConfig } from '@nuxt/bridge'

export default defineNuxtConfig({
  bridge: {
    nitro: true
  }
})
```

## Обновите ваши скрипты

Вам также потребуется обновить скрипты в файле `package.json`, чтобы отразить тот факт, что Nuxt теперь будет создавать сервер Nitro в качестве результата сборки.

### Установите Nuxi

Установите `nuxi` в качестве зависимости для разработки:

::code-group{sync="pm"}

```bash [npm]
npm install -D nuxi
```

```bash [yarn]
yarn add --dev nuxi
```

```bash [pnpm]
pnpm add -D nuxi
```

```bash [bun]
bun add -D nuxi
```

::

### Nuxi

Nuxt 3 ввёл интерфейс командной строки [`nuxi`](/docs/3.x/api/commands/dev). Обновите скрипты в `package.json`, чтобы использовать поддержку Nuxt Bridge:

```diff
{
  "scripts": {
-   "dev": "nuxt",
+   "dev": "nuxi dev",
-   "build": "nuxt build",
+   "build": "nuxi build",
-   "start": "nuxt start",
+   "start": "nuxi preview"
  }
}
```

::tip
Если `nitro: false`, используйте команды `nuxt2`.
::

### Режим `static`

Если в `nuxt.config` задано `target: 'static'`, в скриптах сборки используйте `nuxi generate`.

```json [package.json]
{
  "scripts": {
    "build": "nuxi generate"
  }
}
```

### Режим `server`

Во всех других случаях вы можете использовать команду `nuxi build`.

```json [package.json]
{
  "scripts": {
    "build": "nuxi build",
    "start": "nuxi preview"
  }
}
```

## Исключите папку сборки Nitro из Git

Добавьте папку `.output` в файл `.gitignore`.

## Проверка

Запустите `nuxi dev` и `nuxi build` (или `nuxi generate`) и убедитесь, что всё ведёт себя ожидаемо.
